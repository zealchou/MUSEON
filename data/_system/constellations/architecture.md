# 星座系統——設計哲學與根原則

> 此文件不屬於任何單一星座，而是整個星系的底層操作邏輯。
> 所有星座的設計、追蹤、探針都隱含這些原則。

## 三位一體：荒謬 × X-Model × PDEIF

MUSEON 星座系統的底層是三個不可分割的面向：

```
荒謬六芒星              X-Model（M1→M6）         PDEIF 逆熵流程設計
─────────              ──────────────          ─────────────────
WHAT（問題是什麼）      HOW（怎麼解決）          DIRECTION（設計方向）
診斷層                  路徑層                   方向層

不知道自己是誰    →     M1 情境盤點        →    從「你想成為誰」逆推
沒有方向         →     M2 邊界確認        →    從「終點狀態」逆推
不掌握 GAP      →     M3 槓桿掃描        →    從「差距」逆推資源
未盤點累積       →     M4 交換分析        →    從「你有什麼」逆推價值
不懂人脈槓桿     →     M5 實驗設計        →    從「最小可行交換」逆推
缺整合佈局       →     M6 審計規劃        →    從「系統完整態」逆推
```

### 核心公理

> **只要講得出來的問題，就一定有解決方案。**

- 荒謬能被命名 → X-Model 就能找路徑 → PDEIF 就能逆推設計
- 講不出來的問題才真的無解——所以荒謬六芒星的第一步就是「讓你講得出來」
- 這是顧問心法的底層邏輯，也是 MUSEON 所有診斷和介入的起點

### 三者的立體關係

不是 A→B→C 的線性流程，而是**立體、鏈狀的對照**：
- 荒謬六芒星是**橫切面**（你現在在哪個維度卡住）
- X-Model 是**縱切面**（從問題到解方的層次推演）
- PDEIF 是**方向箭頭**（永遠從終點往回推）

三者同時存在於每一個決策瞬間。

## PDEIF 逆熵流程設計的操作邏輯

### 逆推工具：OKR × MECE

PDEIF 的逆推不是靠直覺，而是靠兩套邏輯工具：

**OKR（組織邏輯）**——提供層次結構：
- **O（Objective）** = Transformation 終點狀態（Before → After 的 After）
- **KR（Key Results）** = 可量化的里程碑（逆推：到達 O 之前必須經過哪些關卡？）
- **Tasks** = 具體行動（逆推：每個 KR 需要哪些最小可執行的步驟？）

**MECE（拆解邏輯）**——確保不重不漏：
- 每一層的拆解必須**互斥**（Mutually Exclusive）且**窮盡**（Collectively Exhaustive）
- 拆解出的子項之間不重疊，合起來覆蓋全部可能性
- 防止「做了很多但漏了關鍵」或「同一件事做了兩遍」

### 逆推→順向驗證

```
Step 1: 定義終點（O = Transformation After State）
Step 2: 逆推 KR（到達終點前必須達成什麼？MECE 拆解）
Step 3: 逆推 Tasks（每個 KR 需要什麼行動？MECE 拆解）
Step 4: 順向驗證（從 Task 1 開始走一遍，看是否真的能到達終點）
Step 5: 如果順向走不通 → 回到 Step 2 調整 KR 的拆解
```

逆推設計保證「方向正確」，順向驗證保證「路徑可行」。

## 三位一體在星座系統中的應用

### 每個星座的診斷都隱含三步

1. **命名問題**（荒謬）：使用者在這個星座的哪個維度有缺口？
2. **找路徑**（X-Model）：缺口對應 M1-M6 的哪個層？有什麼槓桿可用？
3. **逆推設計**（PDEIF）：理想狀態是什麼？從那裡往回拆解（OKR×MECE）

### 探針的三階段邏輯

探針不只是「問一個問題」，而是引導使用者走過三位一體：
- **Phase 1 探針**（荒謬層）：「你知道問題在哪嗎？」→ 幫助命名
- **Phase 2 探針**（X-Model 層）：「如果問題能說清楚，你覺得從哪個方向開始？」→ 引導找路徑
- **Phase 3 探針**（PDEIF 層）：「如果已經解決了，那個狀態長什麼樣？從那裡往回看。」→ 引導逆推

CPU 閘門判斷使用者在哪個階段，只推下一步。

### 與其他星座的關聯

- **商模十二力**的產品力缺口 → 荒謬 #3（GAP）→ X-Model M3 槓桿掃描 → PDEIF 逆推產品設計
- **品牌七芒星**的品牌靈魂缺口 → 荒謬 #1（自我認知）→ X-Model M1 情境盤點 → PDEIF 逆推品牌 Why
- **轉化漏斗三角星**的產品漏斗 → 起點是品牌價值主張（Transformation）→ PDEIF 逆推整條漏斗
- **戰略三稜鏡**的路徑設計 → 荒謬 #6（整合佈局）→ X-Model M6 審計 → PDEIF 逆推戰略路徑

## 設計禁忌

1. **禁止從入口往上堆**——永遠從終點逆推
2. **禁止跳過命名**——問題沒被講出來就直接給解方 = 解錯問題
3. **禁止只逆推不驗證**——設計完必須順向走一遍
4. **禁止打破 MECE**——拆解如果有遺漏，順向驗證會看起來通但實際有洞
