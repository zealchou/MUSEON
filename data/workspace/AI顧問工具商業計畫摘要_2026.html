產品定位

模組解決的問題使用者感受

現實檢查：100 客戶 × $2,999 = 月營收 30 萬。扣掉兩人團隊的基本薪資和 API 成本，淨利率約 23%。

這個數字夠活但不算好。 真正的商業模型不靠月費撐——月費是「門票」，建立信任後的顧問專案（5-15 萬/案）才是利潤引擎。100 家客戶中若 10% 轉為顧問專案客戶，額外年營收 60-180 萬。

風險與代價

風險嚴重度發生機率防禦
流失率過高（>10%/月） 致命 中高 前 3 個月 1 對 1 onboarding，讓客戶「用出價值」再放手
獲客成本過高 嚴重 中 先用內容行銷 + 口碑，不燒廣告；CAC 超過 $5,000 就暫停
大廠進場（Google/微軟出中文版） 嚴重 中 他們做通用，我們做「台灣中小企業老闆的專屬顧問」——深度打贏廣度
API 成本暴漲 中等 低中 多模型架構（Claude + Gemini + 本地模型），不綁死單一供應商
兩人團隊負荷不了 100 客戶 嚴重 高 50 客戶時就開始招人或建自動化，不等到撐不住

Go-to-Market 路線

第一年四階段

階段時間目標關鍵動作
驗證期 M1-M3 15 個付費客戶 找 5 個行業的種子客戶免費試用 → 驗證產品能解決什麼問題 → 收集 3 個成功案例
牽引期 M4-M6 40 個付費客戶 用成功案例做行業白皮書 → LinkedIn / FB 社群內容行銷 → 每月 2 場線上小型 workshop
加速期 M7-M9 75 個付費客戶 啟動推薦計畫（現有客戶推薦免一個月） → 異業合作（會計師事務所、商圈組織）
穩態期 M10-M12 100+ 個付費客戶 開始顧問專案 upsell → 建立自動化 onboarding → 準備第二年擴編

目標行業優先序

第一波（M1-M6）：美業 / 餐飲 / 零售 — 老闆自己管社群、自己做行銷，對「省時間」的痛感最強。
第二波（M7-M12）：專業服務（設計 / 顧問 / 工程行）— 需要報告、提案、客戶溝通，對「提升產出品質」的需求高。
