二、核心系統七層架構

從 身分 → 綁定 → 收費 → 授權 → 監控，完整的設備管理 SaaS 價值鏈。

┌─────────────────────────────────────────────┐ │ Layer 1：帳號認證與身分 │ │ ├─ Account Registration & Login │ │ ├─ JWT Token 簽發 │ │ └─ Role-Based Access Control (RBAC) │ └─────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────┐ │ Layer 2：序號管理（License） │ │ ├─ Admin 建立/啟用/停用序號 │ │ ├─ User 查詢自己的序號 │ │ └─ 序號作為計費與設備綁定單位 │ └─────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────┐ │ Layer 3：設備配對與令牌 │ │ ├─ User 產生配對碼（5 分鐘有效） │ │ ├─ Agent 輸入碼完成首次綁定 │ │ ├─ Platform 發行 Device Token │ │ └─ 明文只回傳一次，之後存 Hash │ └─────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────┐ │ Layer 4：訂閱管理（Subscription） │ │ ├─ ACTIVE / GRACE / SUSPENDED │ │ ├─ Reconciler 每 5 分鐘自動轉狀態 │ │ └─ 收款後立即恢復 ACTIVE │ └─────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────┐ │ Layer 5：付款與計費（Payment） │ │ ├─ Admin 建立付款單（DRAFT） │ │ ├─ Admin 確認收款（CONFIRMED） │ │ ├─ 確認後自動激活 Subscription │ │ └─ 支援作廢，但已確認不可逆 │ └─────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────┐ │ Layer 6：授權檢查（Entitlement） │ │ ├─ Agent 定期輪詢授權狀態 │ │ ├─ Response 用 Ed25519 簽章（5 分鐘有效） │ │ ├─ Agent 驗章成功才認定有效 │ │ └─ 接新任務前再檢查一次（pre-job check） │ └─────────────────────────────────────────────┘ ↓ ┌─────────────────────────────────────────────┐ │ Layer 7：監控、稽審、告警 │ │ ├─ Audit Log：所有操作記錄 │ │ ├─ Alert：告警事件分級 │ │ ├─ Notification：Email 通知 │ │ └─ Admin Dashboard：統計 + 監控 │ └─────────────────────────────────────────────┘

五、安全基線清單

Phase 1 已實現所有防破解基線，Phase 2 將進階到自動金流與推薦分潤。

項目 實作方式 狀態
序號不作為直接授權憑證 用 Device Token 代替 ✅
配對碼一次性、短時效 5 分鐘內、Hash 存儲 ✅
Device Token 高熵亂數 32 hex (16 bytes) ✅
Token 綁 device_id + fingerprint 必填驗證欄位 ✅
Entitlement 簽章 Ed25519 簽章驗證 ✅
Agent 驗章 內建公鑰，本地驗證 ✅
接新任務前檢查 pre-job check API ✅
離線超過 15 分鐘停止接任務 Agent SDK 實現 ✅

六、系統邊界與外部 API

設備側 API（Agent 呼叫）

POST /api/device/pair → 配對 + 取得 Token
GET /api/device/entitlement → 查詢授權狀態（驗章）
POST /api/device/check-before-job → 接新任務前檢查

管理側 API（Admin 呼叫）

License 管理：CRUD、啟用/停用/作廢
Payment 管理：建立草稿、確認、作廢
Subscription 管理：查詢、手動調整
Device 管理：列表、Revoke
Alert 管理：列表、ACK、RESOLVE

後台排程（無需人工操作）

Reconciler (5分鐘)：ACTIVE→GRACE→SUSPENDED 狀態轉換
Notification Worker：Email 發送

Phase 1 不包含（待 Phase 15+）

❌ 自動金流串接（須人工確認收款）
❌ 自動退款
❌ 推薦/分潤系統
❌ 代理商管理
❌ 深度反逆向方案
