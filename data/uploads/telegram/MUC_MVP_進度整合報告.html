# SYSTEM_SCOPE ## 包含範圍（In Scope） 本文件定義 MUC MVP 第一階段必須交付的系統範圍。 本階段範圍以「收費閉環 + 設備授權閉環 + 最低防破解基線」為主，不追求完整 SaaS 功能，而是先確保產品能上線、能收費、能控管設備合法使用權。 ### 1. 帳號與身分驗證 - 使用者註冊 - 使用者登入 / 登出 - 基本帳號狀態管理 - 使用者可登入前台查看自己的授權與設備資訊 ### 2. Billing Profile / 付款資料 - 使用者建立與維護付款資料 - 平台可查閱付款資料以進行人工對帳與開通 - 作為人工收款、續期、通知與後續發票作業的基礎資料 ### 3. License / Serial 管理 - 平台建立序號（可指定方案） - 使用者查詢自己持有的序號 - 平台查看序號綁定設備 - 平台可啟用 / 停用 / 作廢序號 - 明確以序號作為計費單位，而非以帳號作為計費單位 - License 綁定 Plan（方案決定金額、期限、信任點配額） ### 3.5 方案管理（Plan） - 方案分兩類：**訂閱**（含設定費 NT$24,000）與**續約**（純月費 NT$6,000/月） - 訂閱方案：A(設定+2月) / B(設定+6月) / C(設定+12月) / D(客製) - 續約方案：1月 / 6月 / 12月 - 每個方案可設定一種優惠（送月 / 送點 / 折扣，三選一不可疊加） - 客製方案（訂閱 D）金額和期限由 Admin 每次指定 - Portal 顯示邏輯：新用戶看訂閱方案，既有用戶看續約方案 - 信任點：300 點/月（所有標準方案），超額 USD 0.1/點 ### 4. Subscription 週期與狀態管理 - 每個序號各自擁有獨立訂閱週期 - 週期起算以平台確認收款日（Asia/Taipei）為準 - 週期長度依方案 duration_months 決定（+1/+2/+6/+12 個月） - 到期後進入 GRACE - 寬限結束後進入 SUSPENDED - 平台確認收款後可由任意狀態恢復為 ACTIVE - 狀態需支援 ACTIVE / GRACE / SUSPENDED ### 5. 人工收款與續期 - 平台建立人工收款紀錄 - 平台確認收款後立即延長對應 Subscription 週期 - 支援 payment_status：DRAFT / CONFIRMED / VOID - 收款確認後可觸發通知與後續營運動作 ### 6. Device Pairing / 設備綁定 - 使用者在前台產生一次性配對碼 - Agent 以配對碼完成首次綁定 - 平台建立正式 device record - 成功配對後僅回傳一次 device token 明文 - 一個序號同時只允許一台 ACTIVE 設備 - 若已有 ACTIVE 設備，需先 revoke 舊設備或明確執行取代流程 ### 7. Device Token / Entitlement - Agent 以 device token 定期輪詢授權 - Token 只存 hash，不存明文 - Token 可被 revoke - Entitlement 必須回傳可驗章之授權結果 - Agent 端必須驗證簽章、device_id、一致性與有效時間 - Entitlement 結果需回傳 allow_new_jobs、subscription_status、grace_until 等關鍵資訊 ### 8. Pre-Job Check / 接單前檢查 - Agent 每次接新任務前必須再次檢查授權 - 可透過 pre-job check API 或短效 job grant 實現 - 若授權失效、簽章無效、device mismatch 或 response 過期，則不得接新任務 ### 9.
