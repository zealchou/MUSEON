# 三角驗證方法論完整操作指南

> 來源：DSE 第一性原理拆解 → McKinsey / Forrester / CB Insights 付費報告結構逆向工程

## 核心原則

三角驗證（Triangulation）是付費級報告最值錢的部分。它回答了一個問題：「你怎麼知道你說的是對的？」

ChatGPT 只有一個資料源（訓練資料）、一個方法（語言模型推理）、一個研究者（自己）。
付費報告至少有三個驗證維度交叉確認。

---

## 四種三角驗證形式

### 1. 資料三角驗證（Data Triangulation）

**原則**：同一個數據點用至少兩種來源驗證。

**操作步驟**：

**Step 1：Top-down 推算**
```
產業總產值 × 目標客群佔比 × 相關市場比例 = 可觸及市場規模
```

搜尋策略：
- 搜「[產業] market size [國家] [年份]」
- 搜「[產業] 產值 市場規模 台灣 [年份]」
- 取 2-3 個不同來源的數字

**Step 2：Bottom-up 推算**
```
（平均門店數 or 企業數）× 平均營收 = 市場規模估算
```

搜尋策略：
- 搜「[產業] 門店數 台灣」或「[產業] 企業家數 經濟部統計」
- 搜「[產業] 平均營收 中小企業」

**Step 3：收斂檢查**
- 兩數差距 < 20% → 數據可信，取平均值
- 兩數差距 20-50% → 標記「數據有分歧」，解釋可能原因
- 兩數差距 > 50% → 標記「數據高度不確定」，需要額外來源

---

### 2. 方法論三角驗證（Methodological Triangulation）

**原則**：量化數據 × 質性分析交叉印證。

**操作步驟**：

**Step 1：量化維度蒐集**
- 市場規模、成長率
- 毛利率、淨利率
- 客戶回購率、客單價
- 員工產值

搜尋策略：
- 搜「[產業] 毛利率 平均 台灣」
- 搜「[產業] 經營指標 benchmark」

**Step 2：質性維度蒐集**
- 經營者訪談摘要
- 產業媒體報導
- 成功/失敗案例
- 論壇討論（PTT、商業論壇）

搜尋策略：
- 搜「[產業] 老闆 心得 經驗分享」
- 搜「[產業] 失敗 倒閉 原因」
- 搜「[產業] PTT 討論」

**Step 3：交叉比對**
- 量化說「市場在成長」但質性說「大家都在虧」→ 可能是「新進者太多，餅變大但分的人更多」
- 量化說「客單價上升」但質性說「客人減少」→ 可能是「劣質客戶被淘汰，留下的消費力更高」
- 矛盾本身就是洞見——報告必須呈現並解釋矛盾

---

### 3. 研究者三角驗證（Investigator Triangulation）

**原則**：同一份資料從三個角色視角獨立解讀。

**三個角色的問題清單**：

**老闆/創辦人視角**：
- 這個數據對我的市場定位意味著什麼？
- 我應該擴張還是收縮？
- 競爭者在做什麼，我要怎麼應對？
- 品牌的長期方向要不要調整？

**財務長/會計視角**：
- 這個數據對毛利率的影響是什麼？
- 現金流能支撐多久？
- ROI 在哪裡？回收期多長？
- 哪些成本可以優化？

**營運長/店長視角**：
- 這意味著我要調整什麼流程？
- 人力配置需要怎麼變？
- 客戶體驗會受到什麼影響？
- 這個改變可以規模化嗎？

**輸出格式**：
```markdown
### [數據/發現] 的多視角解讀

| 視角 | 解讀 | 建議行動 |
|------|------|---------|
| 老闆 | [戰略解讀] | [方向建議] |
| 財務 | [財務解讀] | [成本/收益建議] |
| 營運 | [執行解讀] | [流程建議] |

⚠️ 視角衝突：[如果三個角色的建議矛盾，在這裡說明]
```

---

### 4. 理論三角驗證（Theory Triangulation）

**原則**：用至少兩種分析框架看同一產業。

**常用框架組合**：

| 產業類型 | 推薦框架組合 |
|---------|------------|
| 服務業 | Porter's Five Forces + 價值鏈分析 |
| 零售業 | 4P 分析 + 客戶旅程地圖 |
| 製造業 | 價值鏈分析 + 成本結構拆解 |
| 新創/科技 | Business Model Canvas + SWOT |
| 通用 | business-12 框架（12 力診斷） |

**操作步驟**：

**Step 1：選擇兩個框架**
根據產業類型選擇。至少一個是 business-12 的 12 力框架。

**Step 2：分別分析**
用各自的框架獨立分析同一產業。

**Step 3：比對結論**
- 結論一致 → 信心度高
- 結論不一致 → 找出分歧原因，呈現兩種觀點
- 分歧本身是報告的價值——免費 AI 永遠只給你一個框架的結論

## 品質控制檢查表

每份報告完成後，逐項檢查：

### 資料三角驗證
- [ ] 市場規模是否有 top-down + bottom-up 雙重驗證？
- [ ] 關鍵數據是否有 2+ 獨立來源？
- [ ] 數據分歧是否有標記和解釋？

### 方法論三角驗證
- [ ] 是否同時有量化數據和質性分析？
- [ ] 量化和質性的矛盾是否有呈現和解釋？

### 研究者三角驗證
- [ ] 是否從老闆/財務/營運三個視角分別解讀？
- [ ] 視角衝突是否有呈現？

### 理論三角驗證
- [ ] 是否使用了 2+ 分析框架？
- [ ] 框架間的結論差異是否有說明？
