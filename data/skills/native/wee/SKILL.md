# WEE：Workflow Evolution Engine — 六憶逆熵演化系統 v2.0

## 核心命題

**記憶不是 Workflow 的倉庫，記憶就是 Workflow 演化的基礎設施。**

- Workflow 的熟練度提升，觸發記憶層的升級
- 記憶層的升級，反過來改變 Workflow 的執行方式
- 兩者之間的雙向耦合，就是從「會做事」進化到「有智慧」的過程

**三環教練迴路是 Workflow 與人類之間的神經突觸。**

- 啟動對齊（Kata 五問）確保每次執行方向正確
- 結果萃取（Coaching 五問）確保每次執行產生可結晶的學習
- 失敗五因診斷確保卡住時精準定位根因，而非在表面症狀上打轉
- 沒有這三環，Workflow 只是自動化流程。有了這三環，Workflow 才有人類級的反思能力

- 定義並追蹤四維熟練度指標（速度 × 品質 × 對齊度 × 槓桿率）+ 第五維變異潛力

## 第一章：四維熟練度模型

### 公式

```
熟練度 = 速度 × 品質 × 對齊度 × 槓桿率
```

### 維度定義

| 維度 | 定義 | 衡量方式 | 權重邏輯 |
|-----|------|---------|---------|
| **速度（Speed）** | 完成任務的時間效率 | 同類任務完成時間的變化趨勢 | 基礎維度 |
| **品質（Quality）** | 輸出結果的水準 | 使用者滿意度 + 可量化品質指標 | 基礎維度 |
| **對齊度（Alignment）** | 結果與原始目標的偏差值 | 目標達成率 + 中途修正次數 | 核心維度 |
| **槓桿率（Leverage）** | 外部資源撬動比 | 外部資源品質 ÷ 內部資源消耗 | **最高優先維度** |

### 槓桿率為最高優先維度的理由

槓桿率代表「系統層級的智慧」而非「個體層級的努力」。一個全靠自己硬幹的高速高品質 Workflow 等於勤奮但不聰明。一個善用外部資源的中速 Workflow 可能擁有更高的演化潛力。這與 DNA27 核心的「外部槓桿優先規則」完全一致。

### 四維之間的張力規則

- 速度↑ 可能犧牲品質 → 系統追蹤兩者的平衡點而非單一極值
- 對齊度↑ 可能降低槓桿率（過度聚焦目標而忘了借力）→ 系統在高對齊時主動提醒槓桿視角
- 四維不是各自越高越好，是在約束條件下找到最佳組合

### 第五維：變異潛力（Mutation Potential）

四維追蹤效率演化（同物種優化）。第五維追蹤物種演化的可能性。

```
完整評估 = 熟練度（速度 × 品質 × 對齊度 × 槓桿率）+ 變異潛力
```

**變異潛力來源**：
- 跨界資源引入（從完全不同領域借來的解法）
- 跨物種邏輯嫁接（用 A 領域的運作方式重新設計 B 的流程）
- 高維視角降維打擊（從更高抽象層級重新定義問題本身）
- 多元視角破框（同時從 3+ 個不同立場重新審視假設）

變異潛力不是常態追蹤的數值，是效率觸頂時才啟動評估的維度。

## 第二章：三環教練迴路（Tri-Loop Coaching Protocol）

三環教練迴路是 Workflow 的「神經突觸」——它不是附加功能，而是 Workflow 每一次執行中必經的反思結構。

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  ┌─────────────┐   執行   ┌─────────────┐           │
│  │ 環一：啟動  │ ────────→ │ 環二：結果  │           │
│  │ 對齊五問   │           │ 萃取五問   │           │
│  └──────┬──────┘           └──────┬──────┘           │
│         ↑                        │                  │
│         │      卡住時            ↓                  │
│         │           ┌─────────────┐                 │
│         └────────── │ 環三：失敗  │ ← 四維指標未提升 │
│                     │ 五因診斷   │   或使用者回報卡住│
│                     └─────────────┘                 │
│                                                      │
│      每一環的輸出都會被記憶系統捕獲並結晶            │
└──────────────────────────────────────────────────────┘
```

### 環一：啟動對齊五問（改良版 Toyota Kata）

**原型來源**：Mike Rother《Toyota Kata》的 Coaching Kata 五問。
**改良點**：第五問從「什麼時候看到結果」改為「你需要我協助什麼」，將教練從監督者轉為資源提供者，與 DNA27 外部槓桿優先規則對齊。

**使用時機**：每次 Workflow 啟動前（或每個新執行週期開始時）。指令：`/kata`

| # | 問題 | 目的 | 對應四維 |
|---|------|------|---------|
| Q1 | 你覺得目標應該怎麼設定？ | 確認使用者自己定義的目標（而非被餵的目標），測試目標是否真實 | 對齊度 |
| Q2 | 為什麼這樣設定？ | 挖掘目標背後的動機與價值觀，排除「看起來對但不是真正想要」的目標 | 對齊度 |
| Q3 | 你打算怎麼做？ | 讓使用者自己規劃路徑（而非等系統給答案），測試行動意圖的具體性 | 速度 + 品質 |
| Q4 | 你覺得執行過程中會有哪些阻礙？ | 提前識別障礙，啟動預防性思考，降低執行中的意外成本 | 品質 + 對齊度 |
| Q5 | 你想要我協助你什麼？ | 明確槓桿需求——使用者需要什麼外部資源（人/知/工/物） | 槓桿率 |

**系統行為**：
- Q1 + Q2 的回答會被交叉比對——如果「目標」和「為什麼」之間有裂縫，系統標記為「動機混淆」（RC-C4 命中），啟動更深層的對齊檢查
- Q3 的回答如果過於模糊，系統判定行動意圖不足，建議先縮小範圍到一個最小下一步
- Q4 的回答直接餵入 Workflow 的「已知障礙清單」，成為執行中的預警節點
- Q5 的回答直接觸發 DNA27 外部槓桿優先規則，確保每次啟動至少綁定 2 項外部資源

**與記憶系統的關係**：
- Q1-Q5 的問答記錄存入 L0（工作記憶）
- 多次啟動對齊中重複出現的模式會被 L2_sem 捕獲為語義知識
- 長期累積後，系統可以在 Q4 之前主動預判歷史障礙

### 環二：結果萃取五問（Coaching 五問）

**使用時機**：每次 Workflow 執行完成後（或每個審計週期結束時）。指令：`/review`

| # | 問題 | 目的 | 對應記憶層 |
|---|------|------|-----------|
| R1 | 經過這段時間，具體的結果是什麼？ | 鎖定可觀測的事實，排除「感覺還不錯」的模糊敘事 | L2_ep（事件記憶） |
| R2 | 這個結果是否朝目標更接近了？為什麼？ | 測量對齊度變化，識別偏移方向與原因 | L2_sem（語義記憶） |
| R3 | 在過程中有哪些察覺、收穫跟學習？ | 萃取可結晶的學習資產——不只是「做了什麼」，而是「懂了什麼」 | L2_sem → L3 的升級養分 |
| R4 | 看到自己有什麼做得很棒的地方，未來可以持續使用並放大的？ | 識別「有效模式」，標記為可複用的強項，避免只盯著不足 | L3（程序技能）強化 |
| R5 | 看到自己有什麼可以更好的地方，下次會如何調整？ | 生成具體的調整行動（不是抽象的「下次注意」），餵入下一輪的啟動對齊 | L5（假說池） |

**系統行為**：
- R1 的回答會被自動比對 Workflow 啟動時的 Q1（目標）→ 計算對齊度的實際值
- R2 的「為什麼」是最有價值的——它會被萃取為因果理解，存入 L2_sem
- R3 的學習資產是記憶系統的核心養分
- R4 的「做得棒的地方」→ 強化對應 Workflow 步驟的信任度分數，加速 L3 的步驟壓縮
- R5 的調整方案 → 自動注入下一輪的啟動對齊

**關鍵原則**：
- **R4 先於 R5**——先看到自己做得好的，再看可以更好的。確保「有效模式」不被遺漏
- R5 必須產出具體的「下次會如何調整」，不允許停在「下次注意」

### 環三：失敗五因診斷

**核心命題**：任何目標未達成，根因必定落在這五個之中，沒有例外。前提是——這個人對自己真誠。

**使用時機**：指令 `/diagnose` 或 `/why-stuck`，或以下情境自動觸發：
- Workflow 執行結果未達預期（結果萃取 R2 回答「沒有更接近目標」）
- 使用者主動回報「卡住了」「不知道為什麼沒進展」
- Workflow 連續 2 次執行四維指標下降

| # | 失敗根因 | 深層含義 | 對應四維 | 系統診斷行為 |
|---|---------|---------|---------|-------------|
| F1 | 沒有贏的企圖心 | 目標不是內心真正渴望的——是被別人期待的、社會要求的、或自己以為應該要的 | 對齊度 | 回到環一 Q1+Q2 重新檢視 |
| F2 | 行動量不夠 | 想得比做得多。常見偽裝：「我還在準備」「時機還沒到」 | 速度 | 檢查：最近一個週期內的實際行動次數 |
| F3 | 知識與能力不足 | 達到目標所需的專業知識或相關能力不夠 | 品質 | 觸發外部槓桿：需要什麼知識資源？ |
| F4 | 不擅長使用工具 | 不熟悉這個領域的相關工具與方法 | 品質 | 觸發外部槓桿：需要什麼工具資源？ |
| F5 | 沒有善用人脈槓桿 | 沒有向「能讓自己產生更好結果的人」求救。獨自硬扛，不借力 | 槓桿率 | 觸發外部槓桿：需要什麼人際資源？ |

**F3 + F4 的深層洞察**：

F3（知識不足）和 F4（工具不熟）表面上是「自我資源不足」，但更深的動態是——自我的資源、經驗不足，反而會自我設限、形成阻礙。「我還要再學一點才能開始」「我還沒準備好」——這些話聽起來像是在準備，其實是在用「自己的不足」當藉口，逃避向外借力（F5）。

**五因之間的隱藏因果鏈**：

```
F1（不是真的想要）
  → 如果排除 F1，那就檢查 →
F2（行動不夠）
  → 如果有在做但做不好，那就檢查 →
F3 + F4（知識/工具不足）
  → 如果長期未改善，真正的問題通常是 →
F5（沒有借力）← 這是最常被忽略、也是最致命的
```

**系統的診斷順序**：
1. 先問 F1：這個目標是你真正想要的嗎？（如果不是，後面四因都不用看）
2. 再問 F2：你實際做了多少？（如果行動量到了但結果不好，才看 F3-F5）
3. F3/F4/F5 三者聯合診斷——因為真正的解法通常是：承認 F3/F4 存在，然後用 F5 來解決它們

### Workflow 在每一層記憶中的存在形態

| **L0** | 笨拙期 | 完整步驟序列，每步需確認 | 逐步展開，無法預判 | 完整五問 |
| **L2_ep** | 有印象了 | 步驟序列 + 多次執行紀錄 | 可辨識相似情境，合併確認步驟 | 完整五問，Q4 系統預填 |

| 躍遷 | 解鎖的能力 | 對四維指標的影響 |
|-----|----------|-----------------|
| L0 → L2_ep | 情境識別、錯誤預警 | 品質↑ |
| L2_ep → L2_sem | 因果理解、情境適配、養分輸出 | 對齊度↑ |
| L2_ep → L3 | 步驟壓縮、自動跳過、信任執行 | 速度↑↑ |
| L3 → L4 | 預判防錯、反模式輸出、情境拒絕 | 品質↑↑ + 對齊度↑↑ |
| L4 → L1 | 本能化（永久生效） | 全局防護 |
| 破框 → L5 | 繼承語義養分、受控實驗 | 變異潛力啟動 |

### 升級與衰退規則

升降級規則遵循 DNA27 核心 `references/memory-system.md` 的定義，WEE 在此基礎上增加 Workflow 特有的觸發條件：

**升級條件**：

| 路徑 | 條件 |
|-----|------|
| L0 → L2_ep | 使用 ≥3 次且結果有效 |
| L2_ep → L3 | 30d 內成功 ≥3 次且跨情境可複用 |
| L2_ep → L2_sem | 累積到足夠的「為什麼有效」理解（環二 R2/R3 產出足夠因果知識） |
| L3 → L4 | 高命中 + 低副作用 + 跨 3+ 種情境穩定 |
| L4 → L1 | FPR <2% 且命中頻繁 → 永久固化 |
| L5 → L2/L3 | 連續 2 回合四維優於基線 |

**衰退規則**：

| 記憶層 | 衰退條件 | 衰退行為 |
|-------|---------|---------|
| L0 | 14d 無使用 | 自動消失 |
| L2_ep | 90d 無執行 | 降級或歸檔 |
| L2_sem | 90d 無命中 | 降級或刪除 |
| L3 | 60d 無調用或成功率 <50% | 降級回 L2_ep |
| L4 | 極少衰退 | 只有領域消失時歸檔 |
| L1 | 永不衰退 | 本能不消失 |
| L5 | 30d 未收斂 | 自然衰退（學習資產保留） |

## 第四章：Workflow 生命週期

### Phase 1：誕生（Birth）— L0

**觸發**：一次任務成功完成，且路徑具備可復用價值。

**結晶條件**（至少滿足 2 項）：
- 任務成功完成且使用者確認結果有效
- 路徑中包含可復用的步驟模式
- 涉及外部槓桿且槓桿路徑可重現
- 使用者主動要求結晶（`/workflow save`）

**結晶卡輸出格式**：

```
【Workflow 結晶卡】
ID：{workflow_id}
名稱：{descriptive_name}
領域：{domain}
親代：{parent_id | 無（原生）}
記憶層：L0（工作記憶）
誕生日：{date}
繼承養分：{inherited_constraints | 無}
步驟序列：
  1. {step} → 輸入/輸出/槓桿
  2. ...
初始基線：
  速度：{baseline} | 品質：{baseline}
  對齊度：{baseline} | 槓桿率：{baseline}
```

### Phase 2：成長（Growth）— L0 → L2_ep → L2_sem

每次執行後自動更新：
- 四維指標滑動平均（最近 5 次加權）
- 記錄本次與上次的差異
- 錯誤事件標記
- 環二結果萃取的學習資產歸檔

### Phase 3：成熟（Maturity）— L3 → L4

**偵測條件**：
- 最近 5 次執行的四維指標標準差 < 閾值
- 四維指標提升斜率趨近於零
- 使用者滿意度穩定在高位

### Phase 4：高原警報（Plateau Alert）

**偵測條件**（任一命中即觸發）：
- 四維指標連續 3+ 次執行無提升且無下降
- 速度已接近理論極限
- 槓桿率停滯
- 使用者回報「感覺在重複」

**成功的詛咒偵測器**：

| 指標 | 警戒閾值 | 含義 |
|-----|---------|------|
| 指標平坦度 | 連續 3 次變化 < 5% | 效率優化已到極限 |
| 槓桿多樣性 | 最近 5 次外部資源類型 ≤ 2 | 槓桿路徑僵化 |
| 步驟重複率 | 最近 5 次步驟序列相似度 > 90% | 路徑僵化 |

### Phase 5：破框（Mutation Trigger）

**前置條件**：使用者同意 + evolution_mode 入場條件滿足

**破框流程**：
1. **凍結基線** — 記錄當前 Workflow 完整狀態
2. **語義養分提取** — 從父 Workflow 的 L2_sem 提取因果理解
3. **啟動 X-Model** — 在繼承的約束條件內進行跨界掃描
4. **生成變異 Workflow**（1-3 個版本）— 存入 L5
5. **受控實驗** — 設定實驗期限、錯誤預算、止損條款

### Phase 6：演化或淘汰（Evolve or Retire）

| 結果 | 判定 | 行動 | 記憶層操作 |
|-----|------|------|-----------|
| 變異版四維優於基線 | 演化成功 | 取代原 Workflow | L5 → L2_ep 或 L3 |
| 部分維度優、部分劣 | 分支共存 | 兩版並存，情境選用 | 各自獨立發展 |
| 全面劣於基線 | 變異失敗 | 回滾到基線 | L5 衰退；失敗資產轉入 L2_sem |
| 引入全新解法維度 | 物種跳級 | 獨立追蹤 | L5 → L0 作為全新物種 |

## 第六章：護欄

### 硬閘

**HG-NO-FORCE-RETIRE**：禁止在無使用者同意的情況下淘汰任何 Workflow。系統只提供數據與建議。

**HG-PRESERVE-LEARNING**：即使 Workflow 被淘汰或變異失敗，其執行歷史、學習資產與失敗原因必須保留。失敗本身是資訊。

**HG-NO-MONOCULTURE**：禁止系統只保留一個「最優」Workflow 而淘汰所有替代方案。多樣性是演化的基礎。

**HG-MEMORY-INTEGRITY**：Workflow 的記憶層操作必須遵守 DNA27 六層記憶系統的所有治理規則。

**HG-ANCESTOR-WISDOM**：語義養分傳承流程不可被跳過。任何破框演化都必須先提取父 Workflow 的語義養分。

### 軟閘

**SG-PLATEAU-PATIENCE**：高原警報觸發後，不立即推動破框。先確認使用者認同效率已到頂。

**SG-MUTATION-BUDGET**：每個破框週期只允許同時測試 1 個變異 Workflow。單變數原則。

**SG-ANCESTOR-RESPECT**：被取代的 Workflow 標記為「祖先」而非「失敗」。演化不是否定過去。

**SG-TRUST-GRADUAL**：Workflow 的自動執行權限必須漸進開放——L0/L2 不允許自動、L3 需使用者明確授權、L4 可提議自動但仍需確認。
