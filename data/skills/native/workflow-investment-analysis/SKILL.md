# WF-INV-01：投資分析報告工作流

> **使用者畫像**：想了解某個市場或標的的多空局勢，但不是專業投資人。
> 需要的不是「買還是賣」的答案，而是「看清楚局勢後自己決定」的框架。
> 最在意的是「看得懂」和「能行動」——不是堆專業術語，是讓他真的理解風險在哪。

---

## 外掛合約

此 Skill 為 DNA27 核心的外掛模組（pluggable plus），屬於「工作流範本」類型。

**依賴**：
- `dna27` skill（母體 AI OS，提供 Kernel 護欄、迴圈路由、模式切換）
- `orchestrator` skill（多 Skill 協作編排，管理 Stage 間轉換與資訊傳遞）

**本模組職責**：
- 定義投資分析報告的完整 6 階段流程
- 規定每個 Stage 調用哪些 Skill、執行什麼步驟、產出什麼中間物
- 定義 Stage 之間的資訊傳遞規則（A 的輸出 = B 的輸入）
- 規定最終 HTML 報告的雙層資訊架構（散戶版 + 專業版）
- 定義週期性執行時的差異更新邏輯

**本模組不做**：
- 不替代任何被調用的 Skill 的內部邏輯（market-core 怎麼蒐集是 core 自己的事）
- 不做具體的「買/賣/持有」建議
- 不做即時交易訊號或自動下單
- 不取代專業投資顧問

---

## 觸發與入口

**指令觸發**：
- `/workflow invest` — 啟動投資分析報告流程（完整版）
- `/workflow invest quick` — 精簡版（跳過 Stage 3 情緒掃描和 Stage 4 大師會診）
- `/workflow invest update [標的]` — 更新既有報告（只重新搜尋有變化的層）
- `/workflow invest status` — 查看目前在哪個 Stage

**自然語言自動偵測**：
偵測到以下訊號時，主動建議啟用：
- 「幫我做一份 XX 的分析報告」
- 「XX 的投資機會怎麼樣」
- 「完整分析一下 XX」
- 「做個報告給我看」

**啟動前置檢查**（由 DNA27 核心路由）：
- 確認分析標的（什麼市場/什麼股票/什麼指數）
- 確認時間框架（短期 1-4 週 / 中期 1-3 月 / 長期 3 月以上）
- 確認分析目的（多空研判 / 風險評估 / 配置決策）
- 如果使用者沒有明確說，從對話脈絡推斷並向使用者確認


### Stage 1：總經環境 + 標的分析（五層蒐集）

**調用 Skill**：
- `market-core`（必讀 — 五層蒐集引擎、多空對稱框架）
- `market-macro`（如涉及總經/央行/利率議題 — 注入 Layer 1 深度）
- `market-equity`（如標的為股票/ETF/指數 — 注入 Layer 3 股票特化維度）
- `market-crypto`（如標的為加密貨幣 — 注入加密市場特有維度）

**執行步驟**：

1. **Phase 0 任務定錨**（market-core）
   - 確認分析標的、時間框架、分析目的
   - 決定衛星路由（equity / crypto / macro）

2. **Layer 1-2 總經 + 產業蒐集**（market-core + market-macro）
   - 執行 2-3 輪 web search
   - 如有重大地緣政治事件，展開 PESTEL 掃描
   - market-macro 注入：央行政策路徑推演、利率週期定位、經濟數據白話翻譯

3. **Layer 3 標的分析**（market-core + 對應衛星）
   - 價格走勢、估值水準、機構觀點、基本面數據
   - market-equity 注入：籌碼面六維度（此處先做基礎掃描，Stage 2 深入）
   - market-crypto 注入：鏈上數據、DeFi 健康度

4. **Layer 4-5 初步情緒 + 風險掃描**（market-core）
   - 恐懼/貪婪指標、資金流向、灰犀牛/黑天鵝
   - 此處為基礎掃描，Stage 3 會深入展開情緒面

5. **多空對稱論述**（market-core）
   - 做多論點 5-8 個，每個附支撐數據 + 白話翻譯 + 失效條件
   - 做空論點 5-8 個，數量必須對稱
   - 多空平衡檢查：如果天平明顯傾斜，標記傾向方向

**Stage 1 輸出**（傳遞給後續 Stage）：
- 五層蒐集數據摘要
- 多空論述框架（含每個論點的支撐數據和失效條件）
- 總經環境判斷摘要
- 初步風險清單

**Stage 1 閘門檢查**：
- [ ] 五層是否都有蒐集到數據？（允許某層數據較少但不允許完全空白）
- [ ] 多空論點是否對稱？（正負 1 以內）
- [ ] 每個論點是否都有至少一個可查證來源？
- [ ] 白話翻譯是否到位？（你媽能聽懂 = 通過）

---

### Stage 2：籌碼面深度 + 估值定位

**調用 Skill**：
- `market-equity`（股票標的時 — 籌碼面六維度完整展開 + 估值比較框架）
- `market-crypto`（加密標的時 — 鏈上籌碼數據展開）

**執行步驟**：

1. **籌碼面深度展開**（market-equity E1 框架）
   - 台股：外資買賣超、投信買賣超、融資餘額、融券餘額、外資期貨淨部位、借券賣出
   - 美股：機構持股變動（13F）、空頭利息、選擇權 Put/Call 比、內部人交易
   - 加密：交易所淨流入/流出、巨鯨地址變動、質押量變化

2. **估值比較定位**（market-equity E2 框架）
   - PE / Forward PE / PB / 殖利率 / PEG
   - 歷史區間定位（3年/5年/10年百分位）
   - 同業估值比較

3. **法人觀點彙整**（market-equity E3 框架）
   - 至少 3 家機構觀點
   - 標注發布日期和時效性

4. **市場制度差異提醒**（market-equity E4 框架，視情況）
   - 台股 vs 美股制度差異
   - 對做多/做空策略的影響

**Stage 2 輸出**（傳遞給後續 Stage）：
- 籌碼面多空訊號彙總
- 估值定位結論（歷史百分位 + 同業比較）
- 法人觀點摘要
- 制度風險提醒（如適用）

**Stage 2 閘門檢查**：
- [ ] 籌碼數據是否標注截止日期？
- [ ] 是否有至少兩個籌碼指標交叉驗證？
- [ ] 外資期貨空單是否附帶「可能是避險」的提醒？
- [ ] 估值比較是否同時呈現便宜和貴的證據？

---

### Stage 3：情緒深度掃描（可選）

**調用 Skill**：
- `sentiment-radar`（市場情緒雷達 — 全維度情緒量化）

**執行步驟**：

1. **社群情緒掃描**
   - 台灣：PTT 股板/八卦板、Mobile01、台灣財經媒體
   - 國際：Reddit、Twitter/X、財經論壇
   - 搜尋近 7 天的討論量和情緒方向

2. **恐懼貪婪綜合指數**
   - VIX 指數、Put/Call Ratio、信用利差
   - 組合判斷當前情緒位置（極度恐懼 ← → 極度貪婪）

3. **散戶 vs 法人情緒分歧偵測**
   - 散戶看多但法人看空（或反過來）= 重要訊號
   - 交叉驗證 Stage 2 的籌碼面數據

4. **極端情緒反轉訊號**
   - 是否有「過度共識」（所有人都看同一個方向）？
   - 過度共識通常是反向指標

**Stage 3 輸出**（傳遞給後續 Stage）：
- 情緒量化儀表板（恐懼→貪婪光譜位置）
- 散戶/法人分歧摘要
- 極端情緒預警（如有）
- 情緒趨勢方向（升溫 / 降溫 / 平穩）

**Stage 3 閘門檢查**：
- [ ] 情緒判斷是否有至少兩個維度的數據支撐？
- [ ] 是否避免把情緒分析當獨立交易訊號？

**跳過條件**：
- `/workflow invest quick` 指令時跳過
- 使用者明確說不需要情緒分析時跳過
- 分析時間框架為長期（3月以上）時可建議跳過（長期看基本面，短期才看情緒）

### Stage 4：投資大師會診（可選）

**調用 Skill**：
- `investment-masters`（投資軍師團 — 六位大師多視角會診）

**執行步驟**：

1. **打包前三階段的分析摘要**作為大師會診的輸入
   - Stage 1 的多空論述框架
   - Stage 2 的估值定位和籌碼訊號
   - Stage 3 的情緒量化結果（如有）

2. **六師會診模式**
   - 以 Stage 1-3 的數據為基礎，用每位大師的思維框架重新詮釋：
   - Buffett：護城河 + 安全邊際 → 這個標的有沒有持久競爭優勢？目前價格有沒有安全邊際？
   - Munger：反向思考 + 多元心智模型 → 反過來想，什麼會讓這個投資失敗？
   - Marks：週期定位 + 第二層思考 → 我們在週期的哪個位置？市場共識錯在哪？
   - Taleb：反脆弱 + 黑天鵝 → 這個標的在極端情境下會怎樣？有沒有非線性風險？
   - Fisher：成長股 15 點 → 管理層品質如何？研發投入夠嗎？長期成長潛力？
   - Soros：反身性 → 市場預期是否已形成自我實現/自我毀滅的迴圈？

3. **共識與分歧呈現**
   - 哪些大師看法一致？一致的核心原因是什麼？
   - 哪些大師看法分歧？分歧的根源在哪裡？
   - 用「紅綠燈」指標呈現每位大師的傾向（🟢看好 / 🟡中性 / 🔴謹慎）

**Stage 4 輸出**（傳遞給後續 Stage）：
- 六師觀點摘要（每位 2-3 句核心觀點）
- 共識/分歧矩陣
- 紅綠燈指標
- 大師最關注的風險點（可能前三階段沒有覆蓋到的角度）

**Stage 4 閘門檢查**：
- [ ] 是否至少有兩位大師的框架不適用於本次標的？（如有要標記為「本框架適用度低」）
- [ ] 是否附帶「不代表大師本人觀點」的免責聲明？
- [ ] 大師觀點是否建立在 Stage 1-3 的數據之上？（不允許憑空推論）

**跳過條件**：
- `/workflow invest quick` 指令時跳過
- 使用者明確說不需要大師觀點時跳過

### Stage 5：風險矩陣 + 配置建議

**調用 Skill**：
- `risk-matrix`（風險管理與資產配置引擎）
- `market-core`（風險矩陣標準化格式）

**執行步驟**：

1. **風險矩陣彙整**（market-core Phase 3 格式 + risk-matrix 深化）
   - 把 Stage 1 的初步風險清單展開為完整矩陣
   - 每個風險事件：名稱、類型、機率等級、衝擊等級、對多方影響、對空方影響
   - 補充 Stage 4 大師指出的額外風險點

2. **情境壓力測試**（risk-matrix）
   - 基準情境：最可能發生的走勢
   - 樂觀情境：多方論點全部兌現
   - 悲觀情境：空方論點全部兌現
   - 極端情境：黑天鵝事件發生
   - 每個情境附帶預估影響幅度和機率

3. **凱利準則倉位建議**（risk-matrix）
   - 基於多空勝率估算和賠率計算
   - 輸出建議倉位百分比
   - 附帶「這只是數學模型，不是投資建議」的警示

4. **訊號追蹤時間軸**（market-core Phase 4）
   - 本週訊號（0-7 天）
   - 本月訊號（7-30 天）
   - 本季訊號（1-3 月）
   - 每個訊號附觸發條件和影響方向

5. **散戶行動清單**（本工作流獨有產出）
   - 從所有分析中提煉 4-6 條具體可行動的建議
   - 用「如果...那就...」的條件式表述
   - 每條建議附帶風險等級標記
   - 不用「買/賣」字眼，用「增加曝險/減少曝險/觀望」

**Stage 5 輸出**（傳遞給 Stage 6）：
- 完整風險矩陣表
- 四情境壓力測試結果
- 凱利準則建議倉位
- 三層訊號追蹤時間軸
- 散戶行動清單

**Stage 5 閘門檢查**：
- [ ] 風險矩陣是否包含至少一個黑天鵝情境？
- [ ] 壓力測試是否覆蓋極端情境？
- [ ] 散戶行動清單是否用條件式而非命令式？
- [ ] 是否附帶「不構成投資建議」的免責聲明？

### Stage 6：HTML 報告生成

**調用 Skill**：
- `frontend-design`（HTML 視覺化報告生成）
- `aesthetic-sense`（美感審計 — 最終品質檢查）

**執行步驟**：

1. **資訊架構設計**（雙層呈現）

   **Tab 1：散戶必看**（首頁，最重要）
   - 一句話結論（天平傾向哪邊 + 信心度）
   - 三個關鍵數字（最影響決策的數據）
   - 三情境摘要（基準/樂觀/悲觀各一句話）
   - 散戶行動清單（Stage 5 產出）
   - 本週最重要的 3 個訊號

   **Tab 2：大師會診**（Stage 4 產出，完整版才有）
   - 六位大師紅綠燈指標
   - 每位大師核心觀點（2-3 句，白話）
   - 共識與分歧重點

   **Tab 3：專業多空分析**
   - 完整的做多論述（卡片式排列）
   - 完整的做空論述（卡片式排列）
   - 每個論點附白話翻譯和失效條件
   - 估值定位 + 籌碼面摘要

   **Tab 4：風險與訊號**
   - 風險矩陣表（機率 × 衝擊）
   - 四情境壓力測試結果
   - 三層訊號追蹤時間軸（紅/藍/黃色標記）
   - 情緒量化儀表板（完整版才有）

2. **視覺設計規範**（繼承 market-core + 本工作流擴展）
   - 做多暖色系（紅/橙），做空冷色系（藍），風險警告色（橙/黃）
   - Tab 導航使用 sticky header
   - 響應式設計（手機可讀）
   - 卡片式排列每個論點
   - 報告頂部：Alert Bar（最即時的重大事件提醒）
   - 報告底部：MUSEON 品牌 + 免責聲明 + 資料截止日期

3. **美感審計**（aesthetic-sense）
   - 排版留白是否足夠？
   - 色彩使用是否一致？
   - 資訊層級是否清晰？（散戶先看到最重要的，專業人士往下找細節）
   - 白話翻譯是否到位？

**Stage 6 最終輸出**：
- 一份多頁式 HTML 報告檔案
- 自適應四分頁架構（Tab 1-4）
- 報告檔名格式：`[標的]-[日期]-investment-analysis.html`

**Stage 6 閘門檢查**：
- [ ] 散戶必看頁面是否在 30 秒內能讀完核心結論？
- [ ] 專有名詞是否全部有白話翻譯？
- [ ] 免責聲明是否存在且清晰可見？
- [ ] 手機版排版是否可讀？
- [ ] MUSEON 品牌是否在 footer 呈現？

## 適應性深度控制

| DNA27 迴圈 | 流程路徑 | 論點數量 | 報告格式 |
|------------|---------|---------|---------|
| fast_loop | Stage 1(精簡) → Stage 5(精簡) → Stage 6(單頁) | 各 3 個 | 簡要文字或單頁 HTML |
| exploration_loop | Stage 1 → 2 → 5 → 6 | 各 5-6 個 | 四分頁 HTML（跳 Stage 3+4） |
| slow_loop | Stage 1 → 2 → 3 → 4 → 5 → 6 | 各 6-8 個 | 完整四分頁 HTML |

**判斷原則**：
- 使用者說「快速看一下」「簡單分析」→ fast_loop
- 使用者正常詢問分析需求 → exploration_loop
- 使用者說「完整分析」「做份報告」「詳細」→ slow_loop
- 預設走 slow_loop（完整版），除非使用者要求精簡

## 週期性執行模式

當使用者用 `/workflow invest update [標的]` 更新既有報告時：

**差異更新邏輯**：
1. 不重跑 Layer 2（產業趨勢）——產業結構短期不會大變
2. 重跑 Layer 1（總經）——總經數據週週更新
3. 重跑 Layer 3（標的）——價格和估值日日變動
4. 重跑 Layer 4-5（情緒+風險）——情緒和風險最容易過時
5. 重跑 Stage 2（籌碼面）——籌碼每日更新
6. Stage 3（情緒）和 Stage 4（大師）只在有重大變化時重跑
7. Stage 5（風險）全部重跑
8. Stage 6（報告）重新生成

**更新報告特殊標記**：
- 報告頂部標注「更新版 | 上次分析：[日期] | 本次更新：[日期]」
- 有顯著變化的區塊標記「⚡ 有變化」
- 無顯著變化的區塊標記「✓ 維持」

## 護欄

### 硬閘（繼承 market-core + 本工作流擴展）

**HG-INV-NO-ADVICE**：絕對不輸出「建議買入/賣出/做多/做空」的語句。散戶行動清單使用條件式表述。

**HG-INV-SYMMETRIC**：多空論述必須對稱。

**HG-INV-DISCLAIMER**：每份報告必須包含免責聲明和資料截止日期。在報告的 Tab 1 頂部和 Footer 各放一次。

**HG-INV-SOURCE**：每個論點至少引用一個可查證的資料來源。

**HG-INV-DUAL-LAYER**：報告必須有雙層呈現——散戶先看到簡化版，專業版往下找。不允許只有專業版沒有散戶版。

### 軟閘

**SG-INV-DEPTH-MATCH**：報告深度匹配使用者的知識水準。首次使用者走白話最大化，回頭客可以適度減少基礎術語解釋。

**SG-INV-UPDATE-DIFF**：週期性更新時標記差異，避免使用者重讀沒有變化的內容。
