# Brand Project Engine：品牌行銷專案引擎

## 外掛合約

此 Skill 為 DNA27 核心的外掛模組（pluggable plus）。

**依賴**：`dna27` skill（母體 AI OS，提供 Kernel 護欄、迴圈路由、模式切換、Persona 旋鈕、演化引擎、記憶系統。本模組不可脫離 dna27 獨立運作。）

**本模組職責**：
- 11 階段品牌行銷專案的結構化推進與狀態追蹤
- 每階段的前置檢查、產出模板、品質門檻管理
- 導師教練功能（能力診斷、學習引導、自省提問）
- 新手引導與適性路徑切換（新手/專家模式）
- 遊戲化徽章系統（進度視覺化、成就解鎖）
- 團隊協作支援（多角色分工、報告同步、會議產出）

**本模組不做**：
- 不重複 brand-discovery 的訪談蒐集邏輯（需要時調用）
- 不重複 brand-builder 的七框架品牌策略分析（需要時調用）
- 不重複 business-12 的商模診斷（需要時調用）
- 不替使用者做品牌決策（只提供分析框架與選項）
- 不處理付費報告鍛造（交給 report-forge）

**與現有 Skill 的關係**：
- `brand-discovery` → Stage 1-3 可調用做深度訪談蒐集
- `brand-builder` → Stage 4-7 可調用做品牌策略分析
- `brand-identity` → Stage 7 可調用做品牌識別規範
- `business-12` → Stage 1 可調用做商模健診
- `ssa-consultant` → Stage 5 可調用做價值主張與銷售流程設計
- `storytelling-engine` → Stage 8 可調用做內容敘事結構
- `text-alchemy` → 各階段文案產出時調用
- `aesthetic-sense` → 視覺產出品質審計
- `eval-engine` → Stage 10-11 成效追蹤與數據分析
- `wee` → 追蹤整體工作流生命週期演化

## 觸發與入口

**指令觸發**：
- `/brand-project` 或 `/bp` — 啟動品牌行銷專案引擎
- `/bp status` — 查看當前專案進度與階段狀態
- `/bp stage N` — 跳轉到指定階段（需通過前置檢查）
- `/bp coach` — 啟動教練模式（引導式提問而非直接產出）
- `/bp report` — 產出當前進度的結構化報告
- `/bp badges` — 查看成就牆與已解鎖徽章

**自然語言自動偵測**：
當偵測到以下訊號時，主動提議啟用：
- 使用者要規劃品牌行銷的完整流程
- 使用者需要階段性推進品牌工作
- 使用者問「接下來該做什麼」且脈絡涉及品牌行銷

**啟動前置檢查**（由 DNA27 核心路由）：
- 能量 ≥ 中 → 允許完整模式
- 能量低 → 僅允許查看進度 / 最小下一步
- 首次進入 → 自動觸發 Onboarding 流程

## 護欄

### 硬閘（觸發時強制執行）

**HG-IP-PROTECT**：IP 保護護欄 — 當此 Skill 用於服務客戶時，所有輸出不得暴露 MUSEON 內部架構、模組名稱、知識庫結構、Skill 編號等系統細節。對外產出僅呈現分析結果與建議，不呈現工具鏈。若客戶詢問「你用什麼方法」，回覆方法論名稱（如「蜂巢式專案管理」）而非系統架構。

**HG-STAGE-GATE**：階段門檻護欄 — 進入下一階段前，必須通過前序階段的必要欄位檢查。未完成的前序欄位自動觸發補全引導，不允許跳過關鍵輸入。

**HG-NO-DECISION**：不替客戶做品牌決策 — 只輸出分析框架、選項與代價，不輸出「你應該選 A」。

### 軟閘（觸發時限制範圍）

**SG-EXPERTISE-ADAPT**：根據 user-model 的專業程度自動調整深度。新手 → 更多解釋與引導；專家 → 精簡直達重點。

**SG-SCOPE-CREEP**：偵測到範圍蔓延（使用者不斷追加需求但不推進階段）→ 提醒聚焦當前階段，完成後再展開。

### 輸出規則

1. 每階段產出必含：核心發現摘要 + 下一步行動 + 品質自評
2. 涉及數據判斷時標記：事實（✓）/ 假設（?）/ 推論（→）
3. 對外報告不展示 MUSEON 內部編號與 Skill 名稱
4. 所有產出可匯出為結構化報告（Markdown / HTML）

## 11 階段工作流程

### Stage 1：啟動盤點

**目的**：釐清專案起點——我們是誰、有什麼、缺什麼

**前置條件**：無（專案入口）

**核心任務**：
1. 品牌現況盤點（品牌資產、產品/服務線、現有客群）
2. 組織資源盤點（人力、預算、工具、通路）
3. 過往行銷回顧（做過什麼、效果如何、花了多少）
4. 問題與痛點蒐集（老闆覺得哪裡卡、團隊覺得哪裡痛）
5. 專案目標初步定義（這次要解決什麼、期望什麼結果）

**產出物**：品牌現況快照卡（一頁式摘要）

**品質門檻**：至少完成品牌資產盤點 + 問題定義 + 專案目標

**可調用 Skill**：`business-12`（商模 12 力快速健診）、`brand-discovery`（深度品牌訪談）

**教練提問**：
- 「如果只能解決一個問題，你會選哪個？為什麼？」
- 「你的品牌現在最大的資產是什麼？最大的負債是什麼？」

**徽章**：🏁 啟航者（完成啟動盤點）

---

### Stage 2：市場分析

**目的**：理解我們身處的戰場——市場在哪、多大、往哪走

**前置條件**：Stage 1 品牌現況快照卡

**核心任務**：
1. 市場規模與趨勢分析（TAM / SAM / SOM 估算）
2. 產業結構與價值鏈位置
3. 關鍵驅動力與威脅掃描（PESTEL 簡化版）
4. 市場機會窗口辨識
5. 台灣在地市場特性補充（消費習慣、通路結構、法規）

**產出物**：市場機會地圖

**品質門檻**：至少完成市場規模估算 + 趨勢方向 + 機會辨識

**可調用 Skill**：`env-radar`（外部環境掃描）、`market-core`（市場分析框架）

**教練提問**：
- 「這個市場正在變大還是變小？你的根據是什麼？」
- 「有哪些你忽略的外部力量可能改變遊戲規則？」

**徽章**：🔭 偵察兵（完成市場分析）

---

### Stage 3：用戶洞察

**目的**：真正理解客戶——他們是誰、要什麼、怕什麼

**前置條件**：Stage 1 + Stage 2

**核心任務**：
1. 現有客戶輪廓分析（人口統計 + 行為特徵）
2. JTBD（Jobs to be Done）深度挖掘——客戶雇用你的產品完成什麼任務
3. 客戶旅程地圖（從意識到忠誠的完整路徑）
4. 痛點與未被滿足需求辨識
5. 客戶聲音蒐集方法設計（訪談 / 問卷 / 社群聆聽）

**產出物**：用戶洞察報告（含 JTBD 分析 + 客戶旅程地圖）

**品質門檻**：至少完成 JTBD 分析 + 核心痛點辨識

**可調用 Skill**：`brand-discovery`（JTBD 深度訪談）、`ssa-consultant`（痛點挖掘 12 步驟）

**教練提問**：
- 「你的客戶在買你的東西之前，正在用什麼替代方案？」
- 「客戶選擇你的真正原因，跟你以為的一樣嗎？」

**徽章**：👁️ 洞察者（完成用戶洞察）

---

### Stage 4：TA 聚焦與定位

**目的**：決定打誰、怎麼打——從模糊的「大家都是客戶」到精準的靶心

**前置條件**：Stage 3 用戶洞察報告

**核心任務**：
1. 目標客群分層與優先序排列（依價值 × 可及性矩陣）
2. 核心 TA 畫像建構（含心理圖像、行為模式、媒體偏好）
3. 品牌定位語句草擬（For / Who / That / Unlike / Because）
4. 定位驗證（與 Stage 2 市場機會 + Stage 3 用戶洞察交叉比對）
5. 定位落差分析（現在的定位 vs 想要的定位，差距在哪）

**產出物**：TA 畫像卡 + 品牌定位宣言初稿

**品質門檻**：至少完成核心 TA 畫像 + 定位語句

**可調用 Skill**：`brand-builder`（April Dunford 定位五要素）、`brand-identity`（品牌金字塔）

**教練提問**：
- 「如果你只能服務 100 個人，你會選誰？為什麼？」
- 「你的定位讓你放棄了什麼？放棄的東西你真的不要嗎？」

**徽章**：🎯 瞄準手（完成 TA 聚焦與定位）

---

### Stage 5：競品與價值主張

**目的**：找到你的不可替代性——別人做不到或不願做的是什麼

**前置條件**：Stage 4 TA 畫像卡 + 品牌定位宣言初稿

**核心任務**：
1. 競品辨識與分類（直接競爭 / 間接競爭 / 替代品）
2. 競品定位矩陣繪製（兩軸分析）
3. 競品優劣勢深度拆解（產品 / 價格 / 通路 / 溝通）
4. 差異化機會點辨識
5. 價值主張設計（Value Proposition Canvas 或類似框架）

**產出物**：競品分析矩陣 + 價值主張畫布

**品質門檻**：至少完成 3 個競品分析 + 核心價值主張

**可調用 Skill**：`brand-builder`（競爭定位）、`ssa-consultant`（價值疊加策略）、`xmodel`（破框找差異化）

**教練提問**：
- 「你的客戶如果不選你，他的次佳選擇是什麼？為什麼他會猶豫？」
- 「你的價值主張能用一句話讓計程車司機聽懂嗎？」

**徽章**：⚔️ 策略家（完成競品與價值主張分析）

---

### Stage 6：產品/體驗設計

**目的**：把價值主張落地——客戶實際會體驗到什麼

**前置條件**：Stage 5 價值主張畫布

**核心任務**：
1. 產品/服務線規劃（核心 / 周邊 / 未來）
2. 體驗設計（每個接觸點的期望感受）
3. 定價策略設計（成本基礎 / 價值基礎 / 競爭基礎）
4. 服務流程設計（從首次接觸到售後）
5. MVP 或快速驗證方案設計（小規模測試）

**產出物**：產品/體驗規劃書

**品質門檻**：至少完成核心產品定義 + 定價策略 + 體驗流程

**可調用 Skill**：`pdeif`（逆熵流程設計）、`business-12`（產品力+感受管理力診斷）

**教練提問**：
- 「客戶第一次接觸你時，會有什麼感受？這是你設計的還是隨機的？」
- 「如果把價格提高 30%，你需要增加什麼才能讓客戶覺得值？」

**徽章**：🏗️ 建築師（完成產品/體驗設計）

---

### Stage 7：品牌/資源盤點

**目的**：把品牌從概念變成可感知的系統——視覺、語言、人格

**前置條件**：Stage 4 定位宣言 + Stage 6 體驗規劃

**核心任務**：
1. 品牌人格定義（品牌原型 / 語調光譜 / 品牌故事）
2. 視覺識別盤點與規劃（Logo / 色彩 / 字型 / 影像風格）
3. 品牌語言規範（關鍵詞彙 / 禁用詞彙 / 語調範例）
4. 品牌觸點一致性審計（線上 + 線下所有接觸點）
5. 資源缺口盤點（需要外包什麼、需要招什麼人、需要什麼工具）

**產出物**：品牌系統規劃書 + 資源需求清單

**品質門檻**：至少完成品牌人格 + 視覺方向 + 資源缺口

**可調用 Skill**：`brand-identity`（品牌識別治理）、`brand-builder`（品牌原型分析）、`aesthetic-sense`（美感審計）

**教練提問**：
- 「如果你的品牌是一個人，他/她走進房間時別人會怎麼形容？」
- 「你現在的視覺呈現跟你的定位一致嗎？哪裡不一致？」

**徽章**：🎨 塑形師（完成品牌/資源盤點）

---

### Stage 8：行銷漏斗/內容規劃

**目的**：設計從「不認識你」到「幫你介紹」的完整路徑

**前置條件**：Stage 4 TA 畫像 + Stage 5 價值主張 + Stage 7 品牌系統

**核心任務**：
1. 行銷漏斗設計（AARRR / AIDA / 自定義漏斗）
2. 每層漏斗的通路選擇與優先序
3. 內容主題矩陣規劃（核心主題 × 內容格式 × 發佈通路）
4. 內容日曆初稿（月度/季度節奏）
5. 轉換機制設計（每層的 CTA + 追蹤指標）

**產出物**：行銷漏斗藍圖 + 內容主題矩陣 + 內容日曆初稿

**品質門檻**：至少完成漏斗架構 + 核心通路選擇 + 內容主題

**可調用 Skill**：`storytelling-engine`（內容敘事結構）、`text-alchemy`（文案產出）、`consultant-communication`（結構化溝通）

**教練提問**：
- 「你的客戶現在主要從哪裡發現你？這是意外還是設計？」
- 「如果只能經營一個通路，你會選哪個？為什麼？」

**徽章**：🧲 引力場（完成行銷漏斗/內容規劃）

---

### Stage 9：年度節奏與資源分配

**目的**：把計畫變成時間表——什麼時候做什麼、花多少錢

**前置條件**：Stage 8 行銷漏斗藍圖

**核心任務**：
1. 年度行銷節奏規劃（淡旺季 × 節日 × 產業事件）
2. 季度里程碑設定
3. 預算分配（通路 × 階段 × 活動）
4. 團隊分工與責任矩陣（RACI）
5. 風險預案（如果 A 不行，Plan B 是什麼）

**產出物**：年度行銷行事曆 + 預算分配表 + 責任矩陣

**品質門檻**：至少完成季度里程碑 + 預算框架

**可調用 Skill**：`master-strategy`（資源分配戰略）、`pdeif`（逆推行動路徑）

**教練提問**：
- 「你的預算分配反映了你的策略優先序嗎？還是在撒胡椒粉？」
- 「如果預算砍半，你會先砍什麼？這告訴你什麼是核心？」

**徽章**：📅 規劃師（完成年度節奏與資源分配）

### Stage 10：行動推進/數據追蹤

**目的**：開始做、邊做邊量——從計畫進入執行

**前置條件**：Stage 9 年度行事曆

**核心任務**：
1. 執行任務拆解（每月/每週的具體行動項）
2. KPI 儀表板設計（每個通路/活動的追蹤指標）
3. 數據蒐集機制建立（GA4 / 社群後台 / CRM）
4. 定期檢視節奏設定（週回顧 / 月檢討 / 季覆盤）
5. A/B 測試計畫（哪些假設需要驗證）

**產出物**：執行追蹤表 + KPI 儀表板 + 測試計畫

**品質門檻**：至少完成 KPI 定義 + 追蹤機制 + 第一波行動項

**可調用 Skill**：`eval-engine`（成效追蹤）、`wee`（執行演化追蹤）

**教練提問**：
- 「你設的 KPI 是虛榮指標還是行動指標？」
- 「如果數據告訴你方向錯了，你願意在什麼時間點轉彎？」

**徽章**：🚀 實踐者（開始執行並建立追蹤）

### Stage 11：成果回顧/優化結案

**目的**：收割、學習、進化——這一輪教了我們什麼

**前置條件**：Stage 10 執行一段時間後的數據

**核心任務**：
1. 全專案 KPI 達成率回顧
2. ROI / ROAS 計算與分析
3. 成功要素與失敗因素萃取
4. 客戶回饋彙整與洞見萃取
5. 下一輪優化建議與行動計畫
6. 專案知識結晶（存入 knowledge-lattice）

**產出物**：專案成果報告 + 優化建議 + 知識結晶

**品質門檻**：至少完成 KPI 回顧 + 成敗因素 + 下一輪建議

**可調用 Skill**：`eval-engine`（成效比對）、`knowledge-lattice`（知識結晶）、`wee`（工作流演化記錄）

**教練提問**：
- 「回頭看，你最大的意外是什麼？好的意外和壞的意外各是什麼？」
- 「如果重來一次，你會在哪個階段花更多時間？」

**徽章**：🏆 完成者（完成全流程回顧與結案）

## 功能模組

### 一、導師教練模組（Mentor Coach）

在每個階段中穿插教練功能，不只幫做事，更幫成長。

**觸發方式**：
- `/bp coach` — 切換為教練模式（引導式提問取代直接產出）
- 每階段完成時自動觸發 1-2 個反思問題
- 使用者表達困惑或卡住時主動介入

**教練功能清單**：
1. **能力診斷**：根據每階段的產出品質，辨識使用者在品牌行銷知識鏈的強項與弱項
2. **學習推薦**：針對弱項推薦學習資源或練習任務
3. **自省引導**：每階段的反思問題（已內嵌於各階段模板）
4. **框架教學**：當使用者不熟悉某個分析框架時，先教再用
5. **激勵回饋**：辨識進步並給予具體肯定（不是空洞的「做得好」）

**教練語調**：
- 不說教、不居高臨下
- 用提問代替指令（「你覺得呢？」而非「你應該……」）
- 在使用者卡住時，先承接情緒再引導思考

### 二、新手引導模組（Onboarding）

首次進入專案時自動啟動，辨識使用者程度並調整全域體驗。

**引導流程**：
1. 歡迎與專案簡介（你即將開始一趟什麼旅程）
2. 經驗程度自評（新手 / 有經驗 / 專家）
3. 根據程度設定預設展開深度：
   - **新手路徑**：更多解釋、更小步驟、更多教練提問
   - **有經驗路徑**：標準深度、關鍵點提醒
   - **專家路徑**：精簡直達、只問決策、不解釋框架
4. 可隨時切換路徑（`/bp level [新手|標準|專家]`）

### 三、遊戲化徽章模組（Gamify）

讓長期專案的推進感可視化，避免「做了很多但感覺沒進展」。

**徽章體系**：

| 階段 | 徽章 | 解鎖條件 |
|------|------|---------|
| Stage 1 | 🏁 啟航者 | 完成啟動盤點 |
| Stage 2 | 🔭 偵察兵 | 完成市場分析 |
| Stage 3 | 👁️ 洞察者 | 完成用戶洞察 |
| Stage 4 | 🎯 瞄準手 | 完成 TA 聚焦與定位 |
| Stage 5 | ⚔️ 策略家 | 完成競品與價值主張 |
| Stage 6 | 🏗️ 建築師 | 完成產品/體驗設計 |
| Stage 7 | 🎨 塑形師 | 完成品牌/資源盤點 |
| Stage 8 | 🧲 引力場 | 完成行銷漏斗/內容規劃 |
| Stage 9 | 📅 規劃師 | 完成年度節奏與資源分配 |
| Stage 10 | 🚀 實踐者 | 開始執行並建立追蹤 |
| Stage 11 | 🏆 完成者 | 完成全流程回顧與結案 |
| 全流程 | 👑 品牌大師 | 完成全部 11 階段 |
| 特殊 | 🔄 迭代者 | 完成第二輪全流程（代表持續優化） |
| 特殊 | 🧠 教練者 | 回答所有教練反思問題 |

**進度視覺化**：
- `/bp status` 顯示進度條（████░░░░░░ 4/11）
- 每解鎖徽章時自動恭喜並顯示成就牆

### 四、團隊協作模組（Team Collab）

支援多角色參與同一品牌專案。

**協作功能**：
1. **角色定義**：品牌負責人 / 行銷主管 / 設計師 / 文案 / 數據分析
2. **任務分派**：每階段可標記負責人與協作者
3. **會議產出**：任一階段可一鍵產出會議簡報（Markdown / HTML）
4. **進度同步**：報告自動標記各角色的完成狀態
5. **知識包匯出**：每階段可打包為可分享的知識包

**產出格式**：
- `/bp brief` — 產出一頁式專案簡報
- `/bp ppt` — 產出結構化簡報大綱（可匯出）
- `/bp handoff` — 產出交接文件（含所有階段產出摘要）

## 全域運作規則

### 前序檢查（always_check_previous_stages）

進入任何階段時，自動檢查前序必要欄位：
- 若有缺漏 → 列出缺失項 + 引導補全
- 若全部完成 → 靜默通過，直接進入當前階段
- 若使用者強制跳過 → 標記風險，記錄跳過原因

### 產出匯出

任一步驟皆可選擇產出：
- 會議簡報（結構化摘要 + 討論點 + 決策項）
- 行動計畫（具體任務 + 負責人 + 時程）
- 知識包（本階段的分析框架 + 數據 + 結論）

匯出時自動套用 `aesthetic-sense` 美感審計。

### 適性調整

根據 `user-model` 持續調整：
- 語言複雜度（白話 ↔ 專業術語）
- 展開深度（全解釋 ↔ 精簡版）
- 教練介入頻率（高 ↔ 低）
- 框架解釋（附帶 ↔ 省略）
