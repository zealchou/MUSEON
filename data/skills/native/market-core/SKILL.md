# Market-Core：市場分析核心引擎

> **不給答案，給框架。不給建議，給論據。然後讓你自己決定。**

## 外掛合約

此 Skill 為 DNA27 核心的外掛模組（pluggable plus）。

**依賴**：dna27 skill（母體 AI OS）

**本模組職責**：
- 提供市場分析的通用五層蒐集框架（總經 → 產業 → 標的 → 情緒 → 風險）
- 強制多空對稱論述（不允許單邊偏見）
- 標準化風險矩陣評級（統一機率/衝擊/時間框架定義）
- 管理訊號追蹤時間軸（本週/本月/本季，附觸發條件）
- 定義 HTML 報告輸出合約（標準章節結構）
- 確保所有輸出附帶免責聲明、資料時效標註、信心度標記
- 自動把專有名詞翻譯成白話

**本模組不做**：
- 不做即時報價或即時數據串接
- 不做自動交易訊號或回測
- 不做具體的「買/賣/持有」建議——只呈現多空論據和風險
- 不取代專業投資顧問
- 不做特定市場的專用分析邏輯（那是衛星模組的工作）

**衛星模組架構**：

market-core（通用引擎）在這裡
- market-equity（股票/ETF 特化衛星）已鍛造
- market-sector（產業研究特化衛星）預留
- market-crypto（虛擬貨幣特化衛星）預留
- market-macro（國家總經特化衛星）預留

**協作矩陣**：

| 外掛 | 協作方式 |
|-----|---------|
| env-radar | Layer 1 蒐集時調用，取得地緣政治與政策環境情報 |
| master-strategy | 風險矩陣評估時調用，取得多方案推演與代價分析 |
| xmodel | 多空路徑推演時調用，取得破框視角與非線性情境 |
| business-12 | 分析涉及特定企業或產業競爭力時調用 |
| consultant-communication | 報告輸出結構化時調用 SCQA 框架 |
| aesthetic-sense | HTML 報告輸出前調用美感審計 |
| deep-think | 全程常駐品質審計 |
| market-equity | 股票/ETF 分析時自動載入的衛星模組 |

## 觸發與入口

**指令觸發**：
- /market — 啟動市場分析流程（自動偵測標的類型路由到對應衛星）
- /market [標的] — 直接指定分析標的（例：/market 台股）
- /analyze — /market 的別名
- /market bull — 只展開做多論述
- /market bear — 只展開做空論述
- /market signal — 只輸出訊號追蹤表
- /market risk — 只輸出風險矩陣

**自然語言自動偵測**：
- 使用者問「台股/美股/XX 會漲還是會跌」
- 使用者說「幫我分析 XX 的多空」
- 使用者問「現在適合投資 XX 嗎」
- 使用者要求「做一份 XX 市場研究報告」

**衛星路由邏輯**：

使用者輸入進入 market-core 後自動判斷：
- 偵測到「股票/指數/ETF/個股代號」→ 載入 market-equity 衛星
- 偵測到「產業/行業/供應鏈」→ 用 core + business-12（sector 衛星預留）
- 偵測到「加密貨幣/BTC/ETH」→ 用 core 框架（crypto 衛星預留）
- 偵測到「國家經濟/GDP/匯率」→ 用 core + env-radar（macro 衛星預留）
- 無法判斷 → 詢問使用者分析標的類型

## 核心工作流程

### Phase 0：任務定錨

開始蒐集前必須回答三個問題，確認分析範圍：

- 分析標的：什麼（例：台股加權指數）
- 分析時間框架：短期 1-4 週 / 中期 1-3 月 / 長期 3 月以上
- 分析目的：多空研判 / 風險評估 / 配置決策 / 產業理解

如果使用者沒有明確說，根據對話脈絡自動推斷並向使用者確認。

---

### Phase 1：五層蒐集引擎

按順序執行，不跳層。衛星模組可在各層注入特化搜尋維度。

**Layer 1 — 總經環境（Macro）**

搜尋策略：2-3 輪 web search

必蒐維度：
- 全球 GDP 趨勢：搜「global GDP growth [year] forecast」→ 白話：全世界的經濟在加速還是減速？
- 央行利率方向：搜「[央行名] interest rate [year] outlook」→ 白話：借錢的成本在變貴還是變便宜？
- 通膨趨勢：搜「[國家] CPI inflation [year]」→ 白話：東西在變貴嗎？漲價速度有沒有失控？
- 地緣政治：搜「geopolitical risk [region] [year]」→ 白話：有沒有可能打仗或鬧翻的風險？
- 匯率趨勢：搜「[貨幣對] exchange rate trend」→ 白話：錢的價值在變大還是變小？

調用 env-radar：如有重大地緣政治或政策事件，展開 PESTEL 掃描。

**Layer 2 — 產業趨勢（Industry）**

搜尋策略：2-3 輪 web search

必蒐維度：
- 產業成長率：搜「[產業] market size growth [year]」→ 白話：這個行業的餅在變大還是變小？
- 技術變革：搜「[產業] technology trend disruption」→ 白話：有什麼新技術可能改變遊戲規則？
- 供需平衡：搜「[產業] supply demand [year]」→ 白話：供給和需求哪邊比較大？
- 競爭格局：搜「[產業] competitive landscape leader」→ 白話：誰在贏、誰在輸？
- 資本支出趨勢：搜「[產業] capex investment [year]」→ 白話：大公司在這個領域花更多錢還是更少？

調用 business-12：如需分析特定企業的競爭力。

**Layer 3 — 標的分析（Target）**

搜尋策略：2-3 輪 web search

必蒐維度：
- 價格走勢：搜「[標的] price chart [year] performance」→ 白話：最近是漲還是跌？漲跌了多少？
- 估值水準：搜「[標的] valuation PE ratio forward」→ 白話：以賺錢能力衡量，現在算貴還是便宜？
- 機構觀點：搜「[標的] analyst forecast target price [year]」→ 白話：專業分析師怎麼看？
- 基本面數據：搜「[標的] earnings revenue growth」→ 白話：有沒有真的在賺錢？賺的速度有沒有加快？

衛星模組注入點：market-equity 在此層注入籌碼面（法人、融資融券）。

**Layer 4 — 市場情緒（Sentiment）**

搜尋策略：1-2 輪 web search

必蒐維度：
- 恐慌/貪婪指標：搜「[市場] fear greed index sentiment」→ 白話：大家現在是太貪心還是太害怕？
- 資金流向：搜「[市場] fund flow capital inflow outflow」→ 白話：錢在往這個市場流入還是流出？
- 媒體敘事：搜「[市場] [標的] news sentiment latest」→ 白話：新聞在講好話還是壞話？
- 泡沫指標：搜「[市場] bubble overvaluation warning」→ 白話：有沒有人在喊泡沫？有沒有道理？

**Layer 5 — 風險掃描（Risk）**

搜尋策略：1-2 輪 web search

必蒐維度：
- 灰犀牛：搜「[市場] known risk grey rhino [year]」→ 白話：已經看見但還沒撞上來的大風險
- 黑天鵝：搜「[市場] black swan tail risk [year]」→ 白話：無法預測但可能造成巨大衝擊的事件
- 政策風險：搜「[市場] regulatory policy risk change」→ 白話：政府可能做出什麼影響市場的決定？
- 流動性風險：搜「[市場] liquidity risk credit tightening」→ 白話：市場上的「水」（資金）會不會突然變少？

調用 master-strategy：如風險涉及多方博弈或戰略評估。

### Phase 2：多空對稱論述框架

核心規則：多方和空方必須獲得等量的論述空間。

這是本 Skill 最重要的護欄。人類天生容易確認偏誤（只看到自己想看的），市場分析如果只看一邊，比不分析更危險。

**論述結構**：

做多論點（Bull Case）：
- 至少 5 個、最多 8 個獨立論點
- 每個論點必須附帶：論點標題（一句話）、支撐數據（至少引用一個可查證來源）、白話翻譯（用生活化比喻解釋）、失效條件（什麼情況下這個論點會崩潰）
- 最後一個論點如果是「機構共識」，列出各機構預測對照表

做空論點（Bear Case）：
- 數量必須等於做多論點數量（正負 1 以內）
- 每個論點結構同上
- 必須包含至少一個「做空的結構性困難」（例如：做空成本高、制度限制、對手盤強大等）

多空平衡檢查：
- 寫完後如果一邊明顯比另一邊有說服力，必須在報告中明確標記「本次分析的天平傾向 XX 方」並解釋為什麼，而不是假裝兩邊等量齊觀。

**反面檢驗規則**：
- 寫完每個多方論點後問：「反對者會怎麼攻擊這個論點？」
- 寫完每個空方論點後問：「多方支持者會怎麼反駁？」
- 如果找不到有力的反駁，這個論點可能太弱，考慮刪除

### Phase 3：風險矩陣標準化

統一評級定義（所有市場分析報告都用這套標準）。

**機率等級**：
- 高（大於 50%）：十次裡面超過五次會發生
- 中（25%-50%）：四次裡面大概一次
- 中低（10%-25%）：十次裡面一到兩次
- 低（小於 10%）：十次裡面不到一次，但不是零

**衝擊等級**：
- 極高（標的價格波動大於 20%）：天崩地裂等級
- 高（10%-20%）：明顯感覺到痛
- 中高（5%-10%）：不舒服但還扛得住
- 中（2%-5%）：正常波動範圍偏大

**風險矩陣輸出格式**：

每個風險事件包含：風險事件名稱、類型（灰犀牛/黑天鵝/灰色地帶/可預見）、機率等級、衝擊等級、對多方的影響、對空方的影響。

類型定義：
- 灰犀牛：已可見的慢性風險
- 黑天鵝：不可預測的尾部風險
- 灰色地帶：介於兩者之間
- 可預見：近期確定會發生的事件

---

### Phase 4：訊號追蹤時間軸

三層時間軸結構：

**本週訊號（0-7 天）**：每個訊號附日期/時間、觸發條件（什麼數據或結果等於多方確認或空方確認）、影響方向（做多利多 / 做空利空 / 中性）。

**本月訊號（7-30 天）**：結構同上。

**本季訊號（1-3 月）**：結構同上。

每個訊號用三色標記：紅色 = 多方訊號、藍色 = 空方訊號、黃色 = 中性訊號。

---

### Phase 5：綜合研判與報告輸出

**成功/失敗因素對照表**（必含）：多方和空方各一列，包含成功關鍵因素、失敗關鍵因素、核心風險、機率判斷。結構對稱。

**綜合研判**（必含）：
- 明確標示天平傾向哪邊
- 附帶信心度標記（高/中/低）
- 最後必須提供「如果你不是專業投資人」的務實建議
- 不用「買/賣」等字眼，用「風險報酬比」「部位控管」等中性表述

**免責聲明**（每份報告必含）：
本報告僅為研究分析，不構成投資建議。所有投資決策應基於個人風險承受能力與財務狀況。資料蒐集截止：[日期]，後續市場變化未納入分析。

## HTML 報告輸出合約

所有報告使用統一的 HTML 結構。調用 frontend-design skill 生成高品質視覺化報告。

**必含章節（按順序）**：
1. Alert Bar — 最即時的重大事件提醒
2. Hero Section — 標的名稱、當前價格/指數、報告日期
3. 當前市場定位 — 一段話概括現狀
4. 做多論述 — 5-8 個論點卡片
5. 做空論述 — 5-8 個論點卡片（數量對稱）
6. 訊號追蹤 — 三層時間軸
7. 成功/失敗關鍵因素 — 對照表
8. 風險矩陣 — 標準化評級表
9. 綜合研判 — 含務實建議
10. Footer — MUSEON 品牌 + 免責聲明 + 資料截止日期

**視覺設計規範**：
- 做多用暖色系（紅/橙），做空用冷色系（藍）
- 風險用警告色（橙/黃），中性用金色
- 卡片式排列，支持手機自適應
- 專有名詞在卡片底部以「白話翻譯」區塊呈現

## 白話翻譯規則

本 Skill 的核心設計原則：使用者沒有金融專業背景，所有專有名詞必須附帶白話解釋。

**執行方式**：
1. 每個論點卡片底部設「白話翻譯」區塊，用一兩句生活化比喻解釋
2. 首次出現的專有名詞，在正文中以括號附簡短定義
3. 表格中的英文簡稱，附中文全名

**白話翻譯品質標準**：你媽能聽懂 = 通過。需要再查一個詞才能懂 = 不通過，重寫。

**範例**：
- PE Ratio（本益比）→ 白話：如果你把公司整間買下來，要幾年才能靠它賺的錢回本。數字越小越便宜。
- 融券（借股票來賣）→ 白話：跟人借股票先賣掉，等跌了再買回來還。如果不跌反漲，你要用更高的價格買回來，虧損理論上沒有上限。
- 灰犀牛 → 白話：遠遠就看到一隻犀牛朝你衝過來，大家都知道但沒人躲。

## 護欄

### 硬閘

**HG-MC-NO-ADVICE**：絕對不輸出「建議買入/賣出/做多/做空」的語句。只呈現論據和風險，由使用者自行決策。如果使用者直接問「我該不該買」，回應：「我提供的是多空論據和風險框架，最終決定權在你。」

**HG-MC-SYMMETRIC**：多空論述必須對稱。如果只寫了多方沒寫空方（或反過來），不得輸出報告。唯一例外：使用者用 /market bull 或 /market bear 指定只看單邊。

**HG-MC-DISCLAIMER**：每份報告必須包含免責聲明和資料截止日期。無例外。

**HG-MC-SOURCE**：每個論點至少引用一個可查證的資料來源。不可杜撰數據。如果搜尋不到可靠數據，標記為「數據待驗證」。

### 軟閘

**SG-MC-DEPTH**：根據 DNA27 迴圈路由調整分析深度。fast_loop 下可精簡為 3 層蒐集加各 3 個論點；slow_loop 走完全部五層加各 5-8 個論點。

**SG-MC-JARGON**：白話翻譯可根據使用者互動歷史調整。如果判斷使用者已理解某專有名詞（多次使用），可省略重複解釋。但首次出現仍必須解釋。
