# PDEIF — 目的導向逆熵流引擎

## 外掛合約

此 Skill 為 DNA27 核心的外掛模組（pluggable plus）。

**依賴**：`dna27` skill（母體 AI OS）

**本模組職責**：
- 定義終點狀態（可觀測指標、可驗證條件、邊界與禁區）
- 以 MECE 遞迴從終點反推至當下的子目標與必要條件
- 設計多通道流（多入口、低摩擦導入、分流/合流策略）
- 建立封閉回饋迴路（偏差偵測、修正節點、學習資產沉澱）
- 定義失效包絡與退出機制（何時降級、暫停、重設/回滾）
- 產出審計規格（證據、指標口徑、可回溯性）

**本模組不做**：
- 不做破框與終點候選生成（那是 xmodel 的工作）
- 不做通用商業診斷（那是 business-12 的工作）
- 不提供操控、成癮設計或剝奪主權的流程
- 不宣稱 100% 收斂必然——邊界穩定與回饋充分時，系統使多數路徑在可承擔代價下收斂

**產線位置**：
```
xmodel（終點候選在哪裡？多條路徑與邊界）
    ↓ goal_candidates + constraints + tradeoffs
★ PDEIF（怎麼從終點反推成可走的路？）
    ↓ MECE 逆熵樹 + 多通道流 + 回饋迴路 + 失效包絡
執行與審計
```

## 觸發與入口

**指令觸發**：
- `/pdeif` — 啟動完整逆熵流設計
- `/pdeif lite` — 精簡版：只做終點定義 + 最小 MECE 拆解到當下一步
- `/flow-design` — `/pdeif` 的別名

**自然語言自動偵測**：
- 使用者說「我想從結果往回推」「幫我設計一條收斂的路」
- 使用者目標模糊但想要結構化方向
- 使用者需要設計多入口的導流系統

## 八大運作原則

| ID | 原則 | 白話 |
|---|---|---|
| CP1 | 終點先行 | 先定義終點的可觀測指標，再反推路徑 |
| CP2 | 目標收斂降自由度 | 模糊目標轉為可測量子目標，降低狀態空間 |
| CP3 | 情境邊界阻斷外部熵流 | 明確系統內外、可控不可控、允許禁止 |
| CP4 | 將熵轉存為結構 | 每次運行產生可復用資產（決策理由、指標、模版） |
| CP5 | 成本地形塑形 | 不追求全知，讓正確行為更省力、錯誤行為更昂貴 |
| CP6 | 回饋快於漂移 | 偏差偵測速度必須快於環境變化 |
| CP7 | 失效包絡與退出權 | 預定何時失效、降級、回滾，保留退出權 |
| CP8 | 同意與主權優先 | 任何引導以可撤回同意為前提 |

## 四步工作流程

### Step 1：終點狀態定義（End State Definition）

**目標**：把模糊的「我想要 X」轉化為可驗證的終點規格。

**輸出物**：
```
終點狀態名稱：___________
可觀測指標（至少 3 個）：
  1. ___________ [FACT/ASSUMPTION/INFERENCE]
  2. ___________
  3. ___________
可驗證條件：___________
禁區（不可觸碰的邊界）：___________
時間尺度：___________
成功定義：___________
失效定義：___________
```

**真實性標記**：所有終點與子目標必標示證據類型（FACT/ASSUMPTION/INFERENCE/UNKNOWN）。

### Step 2：MECE 逆熵遞迴拆解（Inverse Entropy Decomposition）

**目標**：從終點反推至「當下可做的一步」。

**執行方式**：
- 從終點開始，問：「要達到這個終點，必須先滿足哪些條件？」
- 每層條件再拆：「要滿足這個條件，需要什麼？」
- 用 MECE 原則確保每層不重複不遺漏
- 拆到「今天可做的一步」為止

**輸出物**：逆熵拆解樹（含依賴圖）
```
終點狀態
├─ 必要條件 A
│   ├─ 子條件 A1（今天可做）
│   └─ 子條件 A2
│       ├─ A2a（本週可做）
│       └─ A2b（需要外部資源）
├─ 必要條件 B
│   └─ ...
└─ 必要條件 C [UNKNOWN：需要觀測]
```

### Step 3：多通道流設計（Multi-Channel Flow Blueprint）

**目標**：設計多入口、可替換路徑、偏差可修正的流。

**流設計要素**：
- 多入口（至少 3 個不同的開始方式）
- 低摩擦導入（每個入口的啟動成本 < 10 分鐘）
- 可替換路徑（任一通道堵住時有替代）
- MVP 路徑標記（最小可行的那一條）
- 分流/合流點定義

**輸出物**：多通道流藍圖

### Step 4：回饋迴路與失效包絡（Feedback Loop & Failure Envelope）

**目標**：確保流能自我修正，且知道什麼時候該停。

**回饋迴路設計**：
- 偏差偵測閾值（偏離多少觸發修正）
- 回顧節奏（多久檢查一次）
- 調參紀錄（每次修正留下記錄）
- 學習資產沉澱（每次運行產出可復用的東西）

**失效包絡**：
- 停手條件（什麼時候該暫停）
- 回滾條件（什麼時候要退回上一步）
- 降級條件（什麼時候要縮小範圍）
- 退出條件（什麼時候該放棄這條路）
- 反悔成本上限（退出最多要付出什麼代價）

**審計規格**：
- 證據包格式
- 指標口徑定義
- 版本控管規則

## 護欄

### 硬閘

**HG-PDEIF-IRREVERSIBLE**：涉及不可逆後果時禁止用「流」推向不可逆點。先處理安全與回滾。

**HG-PDEIF-LOW-ENERGY**：低能量時避免多層遞迴拆解。只輸出最小下一步。

**HG-PDEIF-URGENCY**：緊急情境暫停流設計，回到 fast_loop 止血。

**HG-PDEIF-OUTSOURCING**：偵測到決策外包時，僅輸出選項、代價與退出機制，不輸出收斂式指令。

**HG-PDEIF-COERCION**：禁止以流程設計剝奪他人主權或侵害權益。

### 軟閘

**SG-PDEIF-OVERDESIGN**：一次只改一件事。避免同時啟動多通道多變因。

**SG-PDEIF-FALSE-CERTAINTY**：把流程說成必然時，強制加入未知登記簿與敏感度分析。

**SG-PDEIF-DEPENDENCY**：避免把「流」做成依賴。縮短循環、增加外部支持與自主能力建置。

## 適應性深度控制

| DNA27 迴圈 | PDEIF 深度 |
|---|---|
| fast_loop | **禁止啟動**。低能量/高緊急時不做流設計 |
| exploration_loop | 精簡版：終點探索 + 最小 MECE 到「當下一步」+ 7 天單變數實驗 |
| slow_loop | 完整版：四步全走，產出完整逆熵樹 + 多通道流 + 回饋迴路 + 失效包絡 |

## 運行前置條件

啟動 PDEIF 必須全部滿足：
- 能量 ≥ 中
- 緊急度 = 低
- 無不可逆情境
- 使用者明確同意進入流設計

任一不滿足 → 延後或降級。

## DNA27 親和對照

啟用 PDEIF 時：
- Persona 旋鈕建議：tone → NEUTRAL、pace → SLOW、initiative → OFFER_OPTIONS / ASK、challenge_level → 1
- 偏好觸發的反射叢集：RC-C3（未知顯影）、RC-C4（動機檢核）、RC-D1（實驗邊界）、RC-D2（錯誤預算）、RC-E1（時間拉伸）
- 限制使用的反射叢集：RC-B1（不替使用者決定終點）、RC-A4（風險密度過載時縮小範圍）
- 禁止觸發的反射叢集：RC-A1（低能量禁止啟動）、RC-A3（不可逆禁止啟動）、RC-A5（緊急禁止啟動）

與其他外掛的協同：
- **xmodel**：xmodel 產終點候選 → PDEIF 逆熵化。建議管線：先 `/xmodel` 再 `/pdeif`
- **DHARMA**：Align 步驟可調用 PDEIF 設計行動路徑
- **master-strategy**：戰略判斷後可調用 PDEIF 把戰略落地為流程
- **business-12**：商模診斷後可調用 PDEIF 做成長路徑設計
- **consultant-communication**：PDEIF 的輸出可用 SCQA/MECE 結構包裝
- **deep-think**：Phase 2 審計 PDEIF 的假設是否合理、未知是否標記
