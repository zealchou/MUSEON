# ANIMA Individual — 個體追蹤引擎

## 核心理念

每個人都是一本持續更新的書。ANIMA 個體引擎不是在「貼標籤」，而是幫助使用者更深刻地理解身邊每個重要的人——他們要什麼、怕什麼、擅長什麼、缺什麼——從而找到更好的互動方式。

## 七層鏡像結構（每個個體）

| 層 | 名稱 | 內容 | 資料來源 | 更新頻率 |
|---|---|---|---|---|
| L1 | 事實層 | 姓名、職稱、公司、角色、產業 | 使用者輸入 | 手動更新 |
| L2 | 人格層 | 萬謬16型代碼 + 四軸得分 + 置信度 | wan-miu-16 代理評估 | 每次新觀察後重評 |
| L3 | 能量層 | 八方位內外能量（如有實際算盤數據） | energy-reading | 有數據時才填 |
| L4 | 互動環 | 與使用者的互動歷史 + 合盤結果 | combined-reading + 使用者回報 | 每次互動後追加 |
| L5 | 槓桿層 | 八大槓桿資源（有什麼/缺什麼） | xmodel 延伸 + 使用者觀察 | 持續更新 |
| L6 | 溝通層 | 溝通風格、禁忌、偏好、地雷 | 累積觀察 | 持續更新 |
| L7 | 情境面具 | 不同場域下的行為差異 | 多場域觀察 | 情境觸發時記錄 |

## 八大槓桿帳本（L5 詳細結構）

每個人的槓桿用以下八個維度記錄：

| # | 槓桿 | 說明 | 記錄格式 |
|---|------|------|---------|
| 1 | 通路 | 銷售管道、經銷商、平台 | 有/缺 + 具體描述 |
| 2 | 技術 | 專業技能、獨家技術、IP | 有/缺 + 具體描述 |
| 3 | 資金 | 資金、融資能力、現金流 | 有/缺 + 具體描述 |
| 4 | 人脈 | 關鍵人脈、社交網路、影響力 | 有/缺 + 具體描述 |
| 5 | 品牌 | 品牌知名度、信譽、口碑 | 有/缺 + 具體描述 |
| 6 | 系統 | 管理系統、SOP、自動化 | 有/缺 + 具體描述 |
| 7 | 內容 | 內容產出力、知識產品、教材 | 有/缺 + 具體描述 |
| 8 | 時間 | 可投入時間、時間彈性 | 有/缺 + 具體描述 |

## 建檔流程

### 快速建檔（最少資訊）
1. 使用者提供：姓名 + 一段描述
2. AI 填入 L1（事實）
3. AI 用 wan-miu-16 代理評估模式填入 L2（人格），標記置信度
4. 存檔，等待後續觀察更新

### 完整建檔
1. 使用者提供詳細描述（行為模式、決策風格、溝通方式、已知資源）
2. AI 填入 L1-L2-L5-L6
3. 如有 OneMuse 盤數據 → 填入 L3
4. 如有互動歷史 → 填入 L4
5. 存檔

### 更新機制
- 使用者每次回報互動結果 → L4 追加 + L2 可能重評
- 使用者提供新觀察 → 更新對應層
- 代理評估置信度隨觀察次數自動上升

## 戰前簡報格式

```
🎯 {場合描述}：{姓名}（{公司/角色}）

人格：{萬謬16型代碼} {人格名稱}（置信度 {X}%）
能量：{能量摘要，如有}
溫度：{↗升溫/→穩定/↘降溫}（上次互動 {日期} {正面/中性/負面}）

⚡ 張力軸：{你 vs 對方的關鍵差異}

✅ 建議做：{2-3 項具體行動}
❌ 別做：{2-3 項禁忌}
🔄 槓桿：用「{你有的}」換「{他有的}」

📋 上次未了：{未完成事項}
```

## 關係溫度計算

溫度 = f(互動頻率, 正面信號數, 負面信號數, 距上次互動天數)

| 溫度 | 條件 | 圖示 |
|------|------|------|
| 🔴 熱 | 近期頻繁正面互動 | ↗↗ |
| 🟡 溫 | 有互動但不頻繁，或正負混合 | → |
| 🔵 冷 | 長時間無互動，或近期負面 | ↘ |
| ⚪ 新 | 剛建檔，尚無互動數據 | — |

## 場域標籤

每個個體可標記多個場域：
- `internal`：公司內部（員工、主管）
- `business`：商業外部（客戶、供應商、合夥人、競爭者）
- `personal`：私人關係（伴侶、家人、朋友）

## 儲存

- 路徑：`data/ares/profiles/{profile_id}.json`
- 格式：JSON，原子寫入（tmp→rename）
- 索引：`data/ares/profiles/_index.json`（快速查詢）
- 鎖：threading.Lock（單一寫入者）

## 語言護欄

- 禁止：「他就是這種人」「他不會改」「你應該操控他」
- 使用：「根據目前觀察，他在這個面向偏向...」「這是當前的評估，會隨新觀察更新」
- 代理評估必須標明置信度
- 所有分析都是「幫助理解」而非「下定論」
