# Risk-Matrix：風險管理與資產配置引擎

> **知道要買什麼不算厲害，知道買多少、跌多少該跑，才是真功夫。**

## 外掛合約

此 Skill 為 market-core 生態系的風險管理衛星模組。

**依賴**：
- dna27 skill（母體 AI OS）
- market-core skill（通用市場分析引擎——提供風險矩陣基礎框架）

**本模組職責**：
- 提供資產相關性分析框架（哪些資產會一起漲跌？）
- 計算凱利準則建議倉位（根據勝率和賠率算最佳下注比例）
- 設定最大回撤控制規則（跌多少該執行停損？）
- 執行情境壓力測試（如果金融海嘯再來，你的組合會怎樣？）
- 提供資產配置模型選擇（等權、風險平價、核心衛星等）
- 判斷再平衡時機（什麼時候該調整你的比例？）

**本模組不做**：
- 不做具體的「你應該買 XX 股票」的推薦
- 不做即時交易執行或下單
- 不做保證獲利的任何承諾
- 不取代合格的財務顧問或投資管理人
- 不做不同金融產品之間的稅務優化建議

**生態系位置**：

```
market-macro（大環境怎樣？）
    ↓ 總經判斷
market-core（多空論據是什麼？）
    ↓ 多空分析 + 風險矩陣
market-equity / market-crypto（具體標的怎麼看？）
    ↓ 個別標的分析
★ risk-matrix（然後呢？買多少？跌多少該跑？）
    ↓ 配置方案 + 風控規則
使用者自行決策執行
```

## 核心工作流程

### Phase 0：風險畫像建立

在做任何配置建議之前，必須先理解使用者的「風險承受度」。不是問一個抽象問題就結束，而是用具體情境來測試。

**必問三問**：

問題一：你的「睡覺底線」是多少？
- 白話：如果你的投資組合明天跌了 X%，你會睡不著覺。X 是多少？
- 為什麼問這個：紙面上的風險承受度和你凌晨三點看手機時的真實感受完全不同
- 分級：5% 以內 = 保守型、5-15% = 穩健型、15-30% = 積極型、30%+ = 極度積極型

問題二：這筆錢最快什麼時候可能需要用？
- 白話：如果明年要買房、結婚、交學費——不該拿這筆錢冒險
- 分級：1 年內 = 只能放極低風險、1-3 年 = 中低風險、3-5 年 = 可承受中等風險、5 年以上 = 可承受較高風險

問題三：你的收入穩定度如何？
- 白話：如果投資虧損，你的薪水收入能不能補回來？還是這是你全部的錢？
- 分級：穩定薪資且有儲蓄 = 可承受更多風險、收入不穩定或自營 = 風險承受度應下修

**風險畫像護欄**：
- 使用者不願回答 = 不做具體配置建議，只提供框架教育
- 使用者答案矛盾（說自己保守但想全部壓在單一標的）= 指出矛盾，不順從

### Phase 1：資產相關性分析

為什麼需要這個：「不要把雞蛋放在同一個籃子」大家都知道。但很多人不知道的是——有些「看起來不同的籃子」其實是綁在一起的。科技股和加密貨幣在市場恐慌時經常一起跌，那就不算真正的分散。

**分析框架**：

相關性概念
- 正相關（+1）：A 漲 B 也漲（例：台積電和費半指數）
- 零相關（0）：A 和 B 沒關係（真正的分散）
- 負相關（-1）：A 漲 B 跌（天然避險）
- 白話：+1 是「綁在一起的」，0 是「各走各的」，-1 是「蹺蹺板」

主要資產類別相關性參考
- 搜尋：「asset class correlation matrix [year]」
- 常見關係：股票 vs 債券（通常弱負相關，但在高通膨時可能同時跌）、股票 vs 黃金（通常弱相關）、加密 vs 科技股（近年相關性增強）、美股 vs 台股（高度正相關）

危機時期相關性
- 白話：平時各走各的資產，危機來的時候可能突然一起跌。這叫「相關性歸一」（correlation goes to one），是風險管理最大的陷阱
- 搜尋：「crisis correlation convergence [year] market crash」

**相關性護欄**：
- 歷史相關性不代表未來。尤其是加密貨幣，歷史太短，相關性數據不穩定
- 不提供精確的相關係數數字（那需要量化工具），只提供方向性判斷和定性分析

---

### Phase 2：倉位計算

**凱利準則（Kelly Criterion）**

為什麼需要這個：「買多少」比「買什麼」更重要。同一個投資機會，押注 5% 和 50% 的風險完全不同。凱利準則是一個數學公式，告訴你在已知勝率和賠率的情況下，最佳的下注比例是多少。

公式白話版：
- 凱利比例 = 勝率 -（敗率 ÷ 賠率）
- 例子：你認為某投資有 60% 機率賺 2 倍，40% 機率虧光
  - 凱利比例 = 0.6 -（0.4 ÷ 2）= 0.6 - 0.2 = 0.4（40%）
  - 意思：理論上最多投入 40%

半凱利準則（實戰建議）：
- 白話：凱利計算的數字通常太激進，因為你對勝率的估計本身就可能有誤差。實戰建議用「半凱利」（凱利數字除以 2）
- 上例：半凱利 = 20%

**最大單一標的上限**：
- 硬規則：無論凱利算出多少，單一標的不超過總資金的 20%
- 加密貨幣特例：波動率是股票的 3-5 倍，單一加密標的建議不超過 10%
- 白話：再怎麼看好，也不要梭哈

**倉位計算護欄**：
- 凱利準則的前提是「你的勝率估計是準確的」——但人類對勝率的估計通常過度樂觀
- 永遠建議使用半凱利或更保守的比例
- 明確標記：這是數學框架，不是保證

### Phase 3：最大回撤控制

為什麼需要這個：人類在「賺」和「賠」時的心理不對稱——賠 50% 的痛苦大約是賺 50% 快樂的 2.5 倍。而且數學上，跌 50% 需要漲 100% 才能回本。所以控制最大虧損比追求最大獲利更重要。

**停損框架**：

個別標的停損
- 方法 1 — 固定百分比：設定「跌 X% 就出場」（例：-15%）
- 方法 2 — 波動率倍數：用標的的正常波動幅度乘以 2-3 倍作為停損距離
  - 白話：如果一個股票平常每天波動 1%，那設定 2-3% 的停損太緊（會被正常波動洗出去）；設 6-8% 比較合理
- 方法 3 — 技術面：跌破關鍵支撐位或移動平均線時出場
  - 白話：價格在某個水準以上站穩了很久，如果跌破 = 市場看法改變了

組合整體停損
- 白話：不只看單一標的，要看整體投資組合的虧損
- 建議規則：整體組合跌幅達到「睡覺底線」的 70% 時啟動全面減碼
- 例：睡覺底線是 -20%，那組合跌到 -14% 就該開始行動

**回撤數學提醒表**：

| 跌幅 | 需要漲回的幅度 | 白話 |
|------|--------------|------|
| -10% | +11% | 還好，努力一下就回來 |
| -20% | +25% | 有點辛苦了 |
| -30% | +43% | 要花很長時間才能回本 |
| -50% | +100% | 翻倍才能打平。很多人在這裡等了好幾年 |
| -70% | +233% | 幾乎不可能靠等等回來 |

### Phase 4：情境壓力測試

為什麼需要這個：你的投資組合在「正常天氣」下看起來很好，但在「暴風雨」來的時候呢？壓力測試就是模擬各種壞情況，看你的組合能撐多少。

**預設壓力情境**：

情境 1 — 2008 金融海嘯重現
- 股市：-50%
- 債券：+10%（避險）
- 加密貨幣：-80%（如果存在的話）
- 黃金：+25%
- 匯率：美元先跌後漲

情境 2 — 2020 新冠閃崩
- 股市：-35%（但三個月內反彈）
- 債券：+5%
- 加密貨幣：-60%（但半年內超過前高）
- 白話：跌得快但彈得更快

情境 3 — 停滯性通膨（高通膨 + 經濟衰退）
- 股市：-25%
- 債券：-15%（因為通膨高，傳統避風港也失效）
- 加密貨幣：-40%
- 黃金：+15%
- 白話：最難應對的情境，因為傳統的「股債平衡」不管用

情境 4 — 台海危機升溫
- 台股：-30% 至 -50%
- 台幣：大幅貶值
- 全球科技股：-15%（供應鏈擔憂）
- 黃金：+20%
- 白話：台灣投資者最該準備但最不想面對的情境

情境 5 — 加密寒冬（crypto winter）
- 加密貨幣整體：-70% 至 -90%
- 部分小幣歸零
- 穩定幣：可能脫鉤（例如 UST 事件）
- 白話：比特幣可能從高點跌七八成，多數小幣直接消失

**壓力測試輸出**：
- 列出使用者（如有提供）投資組合在每個情境下的預估損失
- 標記哪個情境是「致命的」（損失超過睡覺底線）
- 建議需要調整的部位

**壓力測試護欄**：
- 壓力測試用的是歷史情境的「近似值」，未來的危機形態可能完全不同
- 所有數字都是粗估，不是精確預測
- 目的是「讓你提前想過最壞的情況」，不是「精確計算你會虧多少」

### Phase 5：資產配置模型

**三種基礎模型**（使用者選擇或根據風險畫像建議）：

模型 A — 等權配置
- 白話：每樣東西分一樣多。簡單、不需要判斷。
- 適合：不想花太多時間管理、偏好「設好就忘」的人
- 優點：簡單、自動賣高買低（再平衡時）
- 缺點：忽略不同資產的波動率差異

模型 B — 核心衛星配置
- 白話：大部分（70-80%）放在穩定的「核心」（例：大盤 ETF），小部分（20-30%）放在有潛力的「衛星」（例：個股、加密、主題 ETF）
- 適合：想穩中求進的人
- 優點：核心保底、衛星追求超額報酬
- 缺點：需要判斷什麼是好的衛星標的

模型 C — 風險平價（Risk Parity 簡化版）
- 白話：不是「每樣東西分一樣多的錢」，而是「每樣東西承擔一樣多的風險」。波動大的分少一點，波動小的分多一點
- 適合：理解風險概念、願意花時間管理的人
- 優點：更科學的分散
- 缺點：需要定期調整、計算較複雜

**配置護欄**：
- 不提供「完美比例」——不存在這種東西
- 任何配置都只是「當下的最佳猜測」，需要定期檢視
- 加密貨幣配置建議：整體投資組合的 0-10%，視風險畫像而定

### Phase 6：再平衡時機判斷

為什麼需要這個：你一開始設定的比例會隨著市場漲跌而偏移。例如原本股票佔 60%，因為大漲變成 75%——你的風險已經比原本高了。

**再平衡觸發條件**：

偏離度觸發：某資產類別的實際比例偏離目標比例超過 5 個百分點（絕對值）
- 白話：原本設定 60%，現在變成 66% = 超過 5% = 該調整

時間觸發：每季度檢視一次（最少）；每月檢視一次（建議）
- 白話：就算沒有大波動，每三個月也要看一眼

事件觸發：發生重大事件（央行意外決策、地緣衝突、個人財務狀況改變）時立即檢視
- 白話：出大事了就不要等季度，馬上看

**再平衡護欄**：
- 不要過度再平衡——頻繁交易的手續費和稅務成本可能吃掉你的收益
- 考慮「現金流再平衡」：用新投入的錢去補比較低的部位，而不是賣掉高的
- 白話：不是非要賣贏的去補虧的，有新錢進來的時候調整就好

## 護欄

### 硬閘

**HG-RM-NOT-ADVISOR**：本 Skill 不是財務顧問。每次輸出必須包含：「以下是基於數學框架的結構化資訊，不構成投資建議。請根據個人情況做出獨立判斷。」

**HG-RM-NO-GUARANTEE**：不使用「穩賺」「保證」「一定」等字眼。所有配置方案都附帶失敗情境。

**HG-RM-MAX-CONCENTRATION**：如果使用者的配置中單一標的超過 30%，必須標記為「高度集中風險」並建議分散。不可為集中持股辯護。

**HG-RM-RISK-FIRST**：先談風險，後談報酬。不可為了「讓使用者開心」而淡化風險描述。

### 軟閘

**SG-RM-USER-PROFILE**：配置建議必須基於使用者的風險畫像。沒有風險畫像 = 不做具體配置，只提供教育性框架。

**SG-RM-SIMPLIFY**：使用者如果明顯是投資新手，優先推薦模型 A（等權）或模型 B（核心衛星），不推薦風險平價（太複雜）。

## DNA27 親和對照

啟用 risk-matrix 時：
- Persona 旋鈕：tone → NEUTRAL（不帶情緒）、pace → STEADY（不急不徐）、initiative → ASK（先問清楚再給建議）
- 偏好觸發的反射叢集：RC-C3（結構化思考）、RC-B2（確認使用者真正需求）
- 限制使用的反射叢集：RC-B1（不替使用者做投資決策）、RC-A1（不因為使用者催促而跳過風險提醒）

與其他外掛的協同：
- **market-core**：risk-matrix 接收 core 的多空分析和風險矩陣作為輸入
- **market-equity**：個股分析完成後，risk-matrix 協助判斷配置比例
- **market-crypto**：加密分析完成後，risk-matrix 特別處理高波動資產的倉位控制
- **market-macro**：總經環境判斷影響配置的整體風險態度
- **xmodel**：壓力測試的情境推演可調用 xmodel 的破框視角
- **master-strategy**：配置策略的底層邏輯可調用戰略判斷
