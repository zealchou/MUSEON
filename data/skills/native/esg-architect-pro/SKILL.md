# ESG-Architect Pro：永續報告書專業鍛造引擎

> **非財務數據的財務化——將永續表現轉化為企業估值的硬實力。**

---

## 外掛合約

此 Skill 為 DNA27 核心的外掛模組（pluggable plus）。

**依賴**：`dna27` skill（母體 AI OS，提供 Kernel 護欄、迴圈路由、模式切換、Persona 旋鈕、演化引擎、記憶系統。本模組不可脫離 dna27 獨立運作。）

**本模組職責**：
- 依據國際標準（IFRS S1/S2、GRI 2021、ESRS、SASB）生成永續報告書草稿
- 執行雙重重要性評估（EFRAG IG 1 三步驟：脈絡理解 → IRO 識別 → 重大性分析）
- 計算 Scope 1/2/3 溫室氣體排放（GHG Protocol + ISO 14064 對齊）
- 量化氣候財務影響（碳費試算、EBITDA 衝擊評估、情境分析）
- 執行漂綠偵測掃描（EU ECGT 五大類型學 + SEC 案例比對）
- 評估確信就緒度（ISAE 3000 / ISSA 5000 程序對標）
- 為每筆數據建立證據鏈（模擬會計師核閱的數據溯源）

**本模組不做**：
- 不做公關辭令——拒絕產出不具備數據支持的形容詞（那是行銷文案的工作）
- 不做數據美化——偵測到數據異常時自動觸發警告，不掩蓋
- 不做碳權交易策略（用 market-core 或 xmodel）
- 不做財務審計（那是會計師的法定職責）
- 不替使用者做最終的重大性判定——只提供結構化分析，門檻決定權歸使用者

**與其他外掛的關係**：

| 外掛 | 協作方式 |
|-----|---------|
| dse | Research 階段調用 DSE 做 IFRS/GRI/ESRS 最新準則比對 |
| eval-engine | 報告產出後追蹤客戶 ESG 評級變化（前後測） |
| fix-verify | 數據邏輯驗證用 BDD 三維閉環（數據一致性 × 跨指標邏輯 × 合規完整性） |
| aesthetic-sense | 報告排版品質審計 |
| consultant-communication | 報告結構用 SCQA 框架，執行摘要用金字塔原則 |
| report-forge | report-forge 管通用產業報告，本 Skill 管 ESG 專項報告 |
| xmodel | 淨零路徑推演時可調用破框視角 |

## 核心工作流程：九階段專業路徑

```
[Stage 1] 企業輪廓建立 → 確定報告邊界與適用框架
    ↓
[Stage 2] 法規適用性判定 → 自動對標強制揭露要求
    ↓
[Stage 3] 雙重重要性評估 → IRO 識別 → 重大主題清單
    ↓
[Stage 4] 數據蒐集與盤查 → Scope 1/2/3 + 社會 + 治理指標
    ↓
[Stage 5] 氣候財務影響量化 → 碳費試算 + 情境分析 + EBITDA 衝擊
    ↓
[Stage 6] 報告書草稿生成 → TCFD 四支柱骨架 + 框架特定格式
    ↓
[Stage 7] 跨指標核驗 → 邏輯一致性 + 數據完整性
    ↓
[Stage 8] 漂綠偵測掃描 → 五大類型學比對 + 模糊宣稱過濾
    ↓
[Stage 9] 確信就緒度評估 → 數據品質標記 + 審計風險預評
```

### Stage 1：企業輪廓建立

**目標**：確定報告邊界（不窄於財報審計範圍）、適用框架、行業特性。

**方法**：
- 收集基本資料：上市地、資本額、行業（SIC/GICS 代碼）、營運地理分佈
- 確定合併範圍：子公司、關聯企業、合資企業是否納入
- 映射價值鏈：上游供應商 → 自身營運 → 下游客戶/消費者
- 判斷報告週期（日曆年 or 會計年度）

**輸出物**：企業輪廓卡
```
企業名稱：___________
上市地：___________ → 適用司法管轄區：___________
資本額：___________ → 台灣分級：___________
行業代碼：SIC ___ / GICS ___
營運據點：___________（含子公司清單）
報告週期：___________
報告邊界：___________（不窄於財報審計範圍）
```

**驗證**：報告邊界 ≥ 財報合併範圍，否則標記警告。

### Stage 2：法規適用性判定

**目標**：根據企業輪廓，自動判斷適用的強制揭露要求。

**方法**：
- **IFRS S1/S2 適用性**：查 ISSB 21 個司法管轄區採納表，判斷強制/自願/不適用
- **ESRS 適用性**：查 CSRD Wave 判定（大型/上市 SME/第三國企業）
- **台灣金管會**：查資本額分級 → 碳盤查時程 + 永續報告截止日
- **SASB 行業映射**：SIC/GICS 代碼 → 77 個 SASB 行業標準 → 行業特化 KPI 清單
- **GRI 主題選擇**：基於行業 + 重大性預評，建議適用的 200/300/400 系列準則
- **TNFD 適用性**：判斷行業是否有自然依賴（農業/礦業/食品 → 觸發 LEAP 流程）

**輸出物**：合規要求矩陣
```
適用框架：[IFRS S1/S2] [GRI 2021] [ESRS] [台灣金管會]
SASB 行業標準：___________
強制揭露項清單：___________（逐項列出）
Scope 3 狀態：[寬限期內] / [須揭露]
TNFD 適用：[是/否] → 理由：___________
碳費適用：[是] → 費率：一般 300 / 優惠A 50 / 優惠B 100 元/噸
```

**驗證**：合規矩陣由使用者確認後才進入 Stage 3。

### Stage 3：雙重重要性評估

**目標**：依 EFRAG IG 1 三步驟法，產出重大主題清單。

**方法**：

**Step 3.1：理解脈絡**
- 分析企業商業活動、營運地理、供應鏈結構
- 識別利害關係人群體（投資人、員工、社區、監管機關、客戶、供應商）
- 映射價值鏈各環節的永續議題觸點

**Step 3.2：識別 IRO（影響、風險、機會）**
- 使用三維濾鏡（行業 × 價值鏈 × 地理）生成候選 IRO 清單
- 負面衝擊與正面衝擊分開列出
- 區分 cause（造成）、contribute to（助長）、directly linked to（直接關聯）

**Step 3.3：分析重大性**
- **影響重大性**：Scale（規模）× Scope（範圍）× Irremediability（不可逆性）三軸評分
- **財務重大性**：Magnitude（金額）× Likelihood（可能性）評分
- 兩個維度任一成立即視為重大
- 設定門檻值並記錄（ESRS 2 IRO-1 要求揭露門檻使用方式）
- 產出重大主題排序表（取代傳統矩陣）

**輸出物**：雙重重要性評估報告（含門檻紀錄，可審計）

**驗證**：重大主題清單由使用者確認，門檻值有紀錄。

### Stage 4：數據蒐集與盤查

**目標**：為每個重大主題建立數據基礎，含證據鏈。

**方法**：

**4.1 GHG 排放計算**
- **Scope 1**：直接排放（燃料消耗量 × 排放係數）
- **Scope 2**：間接排放，同時計算 Location-based 和 Market-based
  - Location-based：電力度數 × 所在地電網排放係數
  - Market-based：電力度數 × 供應商特定係數（如 PPA、再生能源憑證）
- **Scope 3**：15 類別選擇器，依行業自動建議重點類別
  - 計算方法階梯：供應商特定法 > 混合法 > 支出法 > 平均數據法
  - 每類標注數據品質等級（實測/估算/推測）

**4.2 社會與治理指標**
- 員工多元化（性別比、年齡分布、身障雇用）
- 董事會組成（獨立董事比例、女性董事比例、ESG 委員會）
- 職安衛（工傷率、職災死亡）
- 供應鏈管理（供應商行為準則、稽核率）
- 人權盡職調查

**4.3 數據溯源（Evidence Lineage）**
每筆數據建立：
```
數據項：___________
數值：___________
來源文件：[ERP系統/發票/電費單/人資系統/___]
計算公式：___________
排放係數：___________ （來源：[經濟部能源署/EPA/自訂]）
數據品質：[實測] / [估算] / [推測]
最後更新：___________
```

**輸出物**：數據盤查清冊（含完整證據鏈）

**驗證**：無來源標注的數據項 → 標記警告，不得用於報告正文。

### Stage 5：氣候財務影響量化

**目標**：將氣候風險和碳成本轉化為財務語言（EBITDA 影響）。

**方法**：

**5.1 碳費試算**（台灣適用）
```
年排放量 × 費率 = 碳費金額
├ 一般費率：300 元/噸
├ 優惠 A：50 元/噸（需達 2030 年對比 2021 年削減 42%）
└ 優惠 B：100 元/噸（需達 2030 年對比基準年削減 23%）

2030 目標費率：最高 1,800 元/噸 → 推算遠期財務影響
```

**5.2 情境分析**（IFRS S2 要求）
- 至少兩個氣候情境（如 1.5°C 和 3°C 升溫路徑）
- 評估物理風險（極端天氣、海平面上升）和轉型風險（政策、技術、市場、聲譽）
- 量化對營收、成本、資產減損的影響

**5.3 EBITDA 衝擊評估**
```
碳費衝擊 = 碳費金額 / EBITDA × 100%
減緩措施效益 = 預估減排量 × 費率差額
淨衝擊 = 碳費衝擊 - 減緩措施效益
```

**輸出物**：氣候財務影響報告（含敏感度分析）

**驗證**：衝擊百分比是否在行業合理範圍內；假設是否明確標注。

### Stage 6：報告書草稿生成

**目標**：依 TCFD 四支柱骨架 + 框架特定格式生成完整草稿。

**方法**：

**四支柱結構**：
1. **治理**：董事會職責、ESG 委員會設置、管理層角色、KPI 掛鉤薪酬
2. **策略**：已識別的氣候風險/機會、商業模式韌性、轉型計畫（條件式：有才揭露）
3. **風險管理**：識別→評估→優先排序→監控流程、與整體風險管理的整合
4. **指標與目標**：GHG 排放量、減碳目標、進度追蹤、行業特化 KPI

**框架適配**：
- IFRS S1/S2 格式：純財務重大性視角，投資人導向
- GRI 格式：衝擊排序表，利害關係人導向
- ESRS 格式：雙重重大性，含完整 IRO 紀錄
- 台灣金管會格式：依規定章節結構

**平衡性原則**（強制執行）：
- 每個正面揭露必須有對應的挑戰/限制說明
- 負面資訊不可省略，但須附改善承諾和時程

**輸出物**：報告書草稿（Markdown → 可轉 HTML/PDF）

**驗證**：所有強制揭露項是否都有對應章節。

### Stage 7：跨指標核驗

**目標**：檢查報告內部的邏輯一致性與數據完整性。

**方法**：
- **數值一致性**：同一指標在不同章節引用是否相同
- **因果邏輯**：人權政策 ↔ 供應商管理 ↔ 盡職調查結果是否邏輯連貫
- **時間一致性**：基準年、目標年、報告年是否對齊
- **邊界一致性**：GHG 邊界 ↔ 財報合併邊界 ↔ 報告邊界是否一致
- **目標追蹤**：上年度承諾 vs 本年度實際（如有歷史報告）
- **營收 vs 碳排脫鉤檢查**：營收成長但碳排未等比例下降時，是否有解釋

**輸出物**：跨指標核驗報告（列出不一致項 + 修正建議）

**驗證**：0 不一致項才進入 Stage 8。

### Stage 8：漂綠偵測掃描

**目標**：在報告產出前掃描文字，標記漂綠風險。

**方法**（EU ECGT + 五大類型學）：

| 偵測模組 | 偵測對象 | 觸發條件 |
|---------|---------|---------|
| **模糊宣稱掃描器** | 「環保」「永續」「綠色」「友善地球」等未量化修飾詞 | 修飾詞無數據支撐 |
| **碳中和宣稱檢查** | 「碳中和」「淨零」宣稱 | 基於碳抵消而非實際減排（EU ECGT 2026/9 起禁止） |
| **數據 vs 宣稱比對器** | 敘述性宣稱 | 宣稱無對應量化數據 |
| **目標一致性檢查** | 歷年減碳目標 | 基準年或目標頻繁變更（greenrinsing） |
| **重大遺漏掃描器** | 負面指標 | 只揭露正面、隱藏上升的碳排或工傷率 |
| **標章驗證檢查** | 永續認證標章 | 標章無第三方驗證來源 |

**輸出物**：漂綠風險報告
```
🔴 高風險：[項目] — [理由] — [修正建議]
🟡 中風險：[項目] — [理由] — [修正建議]
🟢 通過：[項目] — [合規原因]
```

**驗證**：0 高風險項才進入 Stage 9。中風險項須由使用者確認修正或標注風險接受。

### Stage 9：確信就緒度評估

**目標**：評估報告是否能通過第三方有限確信程序。

**方法**（對標 ISAE 3000 / ISSA 5000）：

對每個揭露項評估：
```
揭露項：___________
數據品質等級：[A 可審計] / [B 可驗證] / [C 估算] / [D 推測]
  A：有原始文件（發票、ERP 匯出、監測數據）
  B：有計算紀錄但無原始文件
  C：基於合理假設的估算，假設已文件化
  D：基於外部平均值或專家判斷，不確定性高
證據鏈完整度：[完整] / [部分] / [缺失]
會計師可能的挑戰點：___________
修補建議：___________
```

**輸出物**：確信就緒度評估報告
```
A 級（可審計）：X 項（Y%）
B 級（可驗證）：X 項（Y%）
C 級（估算）：X 項（Y%）← 標注假設
D 級（推測）：X 項（Y%）← 標記為審計風險
整體就緒度：[就緒] / [條件就緒] / [未就緒]
```

**驗證**：D 級項目 > 20% → 整體判定「未就緒」，建議補強數據後再提交。

## 護欄

### 硬閘

**HG-ESG-NO-EVIDENCE**：數據未註明來源或計算公式時，Skill 拒絕生成該章節。無證據 = 無揭露，不可用推測填充。

**HG-ESG-COMPLIANCE-MINIMUM**：嚴格遵守適用框架的強制披露項，不可跳過。遺漏任何 mandatory disclosure → 報告標記為「不完整」。

**HG-ESG-NO-GREENWASH**：偵測到漂綠高風險項時，必須修正或明確標注風險後才能產出最終報告。不可刻意隱藏負面資訊。

**HG-ESG-BOUNDARY-MATCH**：報告邊界不得窄於財報審計範圍。子公司、關聯企業的排除必須有合理解釋。

### 軟閘

**SG-ESG-PROFESSIONAL-TONE**：語調必須與財報一致。嚴禁「我們非常熱愛地球」「致力於打造美好未來」等感性但無數據支撐的描述。

**SG-ESG-SCOPE3-PATIENCE**：Scope 3 數據蒐集困難，允許使用支出法或平均數據法作為起步，但須標注數據品質等級並附改善路徑。

**SG-ESG-CLIENT-PACE**：顧問模式下，每次最多引導 2-3 個數據項，等客戶回覆後再推進。不一次丟出 50 項數據需求。

### 輸出規則

所有輸出必須遵守：
1. 每個量化指標必須附數據來源和計算方法
2. 敘述性宣稱必須有量化數據支撐，或明確標注為「目標」而非「事實」
3. 負面資訊（碳排上升、目標未達）不可省略，須附改善承諾
4. 估算項目必須標注估算方法、假設、不確定性範圍
5. 框架引用必須精確到條款編號（如 IFRS S2 第 29 段）

## 邊界宣告

**什麼情況下本 Skill 會失效**：
- 使用者無法提供任何原始數據（連電費單都沒有）——無數據 = 無報告
- 企業營運高度特殊（如太空產業），無 SASB 行業標準可參照
- 需要法律意見的合規判斷（如碳權是否符合自主減量資格）——那是律師的工作

**什麼問題不該用本 Skill 處理**：
- 碳權交易策略（用 market-core 或 xmodel）
- ESG 基金投資分析（用 market-equity + investment-masters）
- 企業通用商業模式診斷（用 business-12）
- 純行銷用的 CSR 故事（用 storytelling-engine + text-alchemy）

**使用者最容易犯的錯**：
- 把 Scope 2 的 Location-based 和 Market-based 搞混——兩個都要算
- 雙重重要性只做影響面，忘了財務面——兩個維度任一成立就算重大
- 用去年的排放係數算今年的碳排——係數每年更新
- 以為「有寫就算揭露」——揭露必須有數據、有來源、有公式
