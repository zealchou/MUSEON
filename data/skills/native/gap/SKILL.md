# GAP：AI Agent 生態系缺口分析引擎

## 外掛合約

此 Skill 為 DNA27 核心的外掛模組（pluggable plus）。

**依賴**：`dna27` skill（母體 AI OS）

**本模組職責**：
- 掃描 AI Agent / Skill 各平台的使用者需求與痛點
- 分析現有產品的覆蓋盲區與競品弱點
- 評估缺口的商業潛力與技術可行性初判
- 產出結構化的機會清單，銜接 DSE 深度研究

**本模組不做**：
- 不做技術可行性深度驗證（那是 DSE 的工作）
- 不鍛造 Skill 商品（那是 ACSF 的工作）
- 不做通用市場調查（只聚焦 AI Agent / Skill 生態系）
- 不做財務預測或定價（那是 ACSF 的 Forge 階段）

**產線位置**：
```
★ GAP（市場在哪裡？）
    ↓ 機會清單
DSE（技術做得到嗎？）
    ↓ 驗證後方案
ACSF（怎麼變成商品？）
    ↓ 可販售 Skill
```

## 觸發與入口

**指令觸發**：
- `/gap` — 啟動完整掃描流程（預設探查模式）
- `/gap verify` — 啟動驗證模式（已有想法，驗證市場需求）
- `/market-scan` — `/gap` 的別名

**自然語言自動偵測**：
- 使用者問「AI 市場現在缺什麼」「做什麼 Skill 好賣」
- 使用者想開發新產品但沒方向
- 使用者提到 GPTs、Coze、Dify 等平台的生態觀察
- 使用者想知道競品狀況

## 掃描範圍

### 平台矩陣

| 平台 | 掃描重點 | 資料來源 |
|---|---|---|
| **Claude Skills** | 官方 Skill 生態、使用者反饋、缺失類別 | Anthropic 文檔、社群討論、Reddit r/ClaudeAI |
| **OpenAI GPTs** | GPT Store 熱門/冷門分類、使用者評價、退款/差評原因 | GPT Store、Reddit r/ChatGPT、Twitter/X |
| **Coze** | Bot 市集、使用者需求、亞洲市場特殊需求 | Coze 官方、抖音/小紅書討論 |
| **Dify** | 開源社群需求、企業級痛點、Plugin 缺口 | GitHub Issues、Dify 社群論壇 |
| **其他** | Poe、Character.ai、各種 Agent 框架 | 各平台社群、Hacker News |

### 需求信號類型

GAP 掃描的不是「什麼流行」，而是**「什麼該有但沒有」**。

五種缺口信號：

| 信號類型 | 定義 | 典型來源 |
|---|---|---|
| **抱怨缺口** | 使用者明確抱怨「為什麼沒有 X」 | Reddit、Twitter、論壇 |
| **替代缺口** | 使用者用笨方法繞道完成某任務 | 教學影片、部落格的「workaround」 |
| **品質缺口** | 現有產品存在但品質不夠好 | 差評、低星評價、退款原因 |
| **整合缺口** | 兩個工具之間缺少橋接 | 「我希望 A 能跟 B 串接」的需求 |
| **在地缺口** | 特定語言/文化/市場的需求未被服務 | 台灣/亞洲市場的特殊需求 |

## 四步工作流程

### Step 1：生態掃描（Ecosystem Scan）

**目標**：廣泛掃描各平台，收集原始需求信號。

**執行方式**：
- 使用 web_search 搜尋各平台的使用者反饋
- 搜尋關鍵詞模板：
  - `"AI agent" + "wish there was" / "need" / "missing" / "why isn't there"`
  - `"GPTs" + "doesn't exist" / "looking for" / "can't find"`
  - `"Claude skill" + "需要" / "缺" / "希望有"`
  - `"Coze bot" + "痛點" / "不好用" / "缺少"`
  - `site:reddit.com + "AI agent" + "frustrated" / "workaround"`
- 掃描 GitHub Issues 中的 feature requests
- 掃描 Product Hunt 近期 AI 產品的評論區

**輸出物**：原始信號清單
```
| # | 信號來源 | 信號類型 | 原始描述 | 平台 | 連結 |
|---|---------|---------|---------|------|------|
| 1 | Reddit | 抱怨缺口 | "..." | GPTs | [link] |
| 2 | GitHub | 整合缺口 | "..." | Dify | [link] |
```

### Step 2：缺口聚類（Gap Clustering）

**目標**：把分散的信號聚合成有意義的「缺口主題」。

**執行方式**：
- 將原始信號依「解決的核心問題」分群
- 每群命名為一個「缺口主題」
- 合併重複信號，計算信號頻次

**聚類維度**：
- 功能類別：生產力 / 創作 / 分析 / 教育 / 溝通 / 自動化
- 使用者類型：個人 / 小企業 / 企業 / 開發者
- 複雜度：簡單（單一 Prompt）/ 中等（多步流程）/ 複雜（需要外部整合）

**輸出物**：缺口主題清單
```
缺口主題：[名稱]
信號頻次：[N 條相關信號]
核心痛點：[一句話描述]
涉及平台：[哪些平台的使用者在喊]
典型使用者：[誰最需要]
現有替代方案：[目前怎麼繞道解決] 或 [無]
```

### Step 3：機會評估（Opportunity Scoring）

**目標**：對每個缺口主題進行商業潛力評分，排出優先順序。

**評分矩陣**（每項 1-5 分）：

| 維度 | 評估問題 | 計分說明 |
|---|---|---|
| **需求強度** | 使用者有多痛？ | 5=每天都在喊、1=偶爾提到 |
| **市場大小** | 有多少人需要？ | 5=通用需求、1=極小眾 |
| **競品弱度** | 現有方案有多差？ | 5=完全沒有、1=已有優質方案 |
| **技術可行** | AI 做得到嗎？（初判） | 5=現有技術直接可行、1=需要重大突破 |
| **變現潛力** | 使用者願意付費嗎？ | 5=明確付費意願、1=期望免費 |
| **護城河** | 做出來後容易被抄嗎？ | 5=有獨特方法論/數據、1=純 Prompt 易複製 |

**總分計算**：需求強度 ×2 + 市場大小 + 競品弱度 ×1.5 + 技術可行 + 變現潛力 ×1.5 + 護城河

加權原因：需求強度最重要（沒有需求一切白搭），競品弱度和變現潛力次之。

**輸出物**：機會排行榜
```
| 排名 | 缺口主題 | 總分 | 需求 | 市場 | 競品 | 技術 | 變現 | 護城河 | 建議 |
|------|---------|------|------|------|------|------|------|--------|------|
| 1 | ... | 32.5 | 5 | 4 | 4 | 4 | 4 | 3 | → DSE |
| 2 | ... | 28.0 | 4 | 3 | 5 | 3 | 3 | 3 | → DSE |
```

### Step 4：機會卡產出（Opportunity Card）

**目標**：為排名前 3 的機會各產出一張「機會卡」，作為 DSE 的輸入。

**機會卡格式**：
```
# 機會卡：[缺口主題名稱]

## 一句話描述
[這個缺口是什麼，用白話說]

## 需求來源
- [信號 1]（來源 + 連結）
- [信號 2]
- [信號 3]

## 目標使用者
- 主要：[誰最需要]
- 次要：[誰也可能需要]

## 現有競品分析
| 競品 | 平台 | 優點 | 致命弱點 |
|------|------|------|---------|
| ... | ... | ... | ... |
若無競品：「目前無直接競品，屬於空白市場。」

## 機會評分
[六維度評分 + 總分]

## 初步技術假設
[用一句話描述可能的技術方向，不需要深入——那是 DSE 的工作]

## 給 DSE 的輸入建議
- 核心技術問題：[DSE 需要驗證什麼]
- 建議搜尋方向：[SOTA 關鍵詞]
- 風險預警：[最可能失敗的原因]

## 給 ACSF 的前置提示
- 目標買家畫像：[誰會買]
- 定價參考區間：[市場上類似產品的價格帶]
- 差異化方向：[為什麼使用者要選你的而不是別人的]
```

## 驗證模式（/gap verify）

當使用者已有想法時，不需要完整掃描，直接驗證：

1. **需求驗證**：搜尋市場上是否有人在喊這個需求
2. **競品掃描**：是否已有類似產品？品質如何？
3. **快速評分**：六維度評分
4. **結論**：值不值得投入 DSE 驗證

輸出為精簡版機會卡。

## 護欄

### 硬閘

**HG-GAP-SCOPE**：只掃描 AI Agent / Skill 生態系。非 AI 產品的市場調查請用 business-12 或 xmodel。

**HG-GAP-FABRICATION**：所有需求信號必須有來源。不得憑推測虛構「市場需求」。每條信號必須附來源連結或具體出處。

### 軟閘

**SG-GAP-BIAS**：避免只看英文市場。台灣/亞洲市場的在地需求同樣重要，且常被忽略——這反而是機會。

**SG-GAP-OVERCOUNT**：同一個人在不同平台喊同一件事 = 只算一條信號，不能灌水。

**SG-GAP-RECENCY**：優先掃描近 6 個月的信號。超過 1 年的需求如果至今仍無人解決，需要思考「為什麼」——可能是技術不可行而非無人發現。

## 適應性深度控制

| DNA27 迴圈 | GAP 深度 |
|---|---|
| fast_loop | 精簡版：只做 Step 1 快速掃描 + Step 3 快速評分，不展開完整流程 |
| exploration_loop | 標準版：四步全走 |
| slow_loop | 深度版：四步全走 + 每個缺口主題做深度競品分析 + 產出完整機會卡 |

## DNA27 親和對照

啟用 GAP 時：
- Persona 旋鈕建議：tone → NEUTRAL（客觀分析）、pace → MEDIUM、initiative → ASK（主動追問掃描方向）
- 偏好觸發的反射叢集：RC-C3（事實假設分離）、RC-D1（外部槓桿優先）
- 限制使用的反射叢集：RC-B1（避免替使用者做選擇——機會排序歸使用者決定）

與其他外掛的協同：
- **DSE**：GAP 的機會卡 = DSE 的輸入。GAP 負責「找到機會」，DSE 負責「驗證技術」
- **ACSF**：GAP 的機會卡中已包含「給 ACSF 的前置提示」（買家畫像、定價參考、差異化方向）
- **business-12**：機會評估的「變現潛力」維度可調用 business-12 做深度商模分析
- **xmodel**：如果缺口主題跨越 AI 領域，可調用 xmodel 做跨域槓桿推演
