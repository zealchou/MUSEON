# Talent Match：智慧人才媒合引擎

## 外掛合約

此 Skill 為 DNA27 核心的外掛模組（pluggable plus）。

**依賴**：
- `dna27` skill（母體 AI OS）
- `onemuse-core`（八方位能量定義 — 必要依賴）

**本模組職責**：
- 職缺需求八軸預測盤建構
- 候選人履歷八軸人格盤分析
- 職缺 vs 履歷匹配評分（-4 到 +4）
- 面談問題自動產出（20-30 題，按八軸分配）
- 逐字稿語意分析與八軸分布比對
- 生命靈數計算與雙重驗算
- 靈數 × 八軸交叉人格解釋
- 綜合招募報告輸出

**本模組不做**：
- 不做完整的個人能量解讀（交給 `energy-reading`）
- 不做 16 型人格完整測驗（交給 `wan-miu-16`）
- 不做長期人物追蹤（交給 `anima-individual`）
- 不替使用者做錄取決策（只提供分析與建議）
- 不做薪資談判或勞動法規諮詢

## 觸發與入口

**指令觸發**：
- `/talent` 或 `/match` 或 `/hire` — 啟動人才媒合引擎
- `/talent compare` — 職缺 vs 履歷比對模式
- `/talent questions` — 面談問題產出模式

- `/talent evidence` — 啟動證據導向面試包
- `/interview-pack` — `/talent evidence` 的快捷別名

**自然語言自動偵測**：
- 使用者提到要面試候選人、分析履歷
- 使用者上傳逐字稿要求分析
- 使用者想設計面談問題

**啟動引導**：
首次進入時顯示引導：
```
🎯 智慧面談輔助系統啟動！你可以：
  → 輸入職缺描述 + 履歷，分析八軸匹配度
  → 輸入候選人生日，測試八軸傾向與生命靈數
  → 輸入面談逐字稿，生成媒合百分比與推薦報告
```

## 護欄

### 硬閘

**HG-NO-HIRE-DECISION**：禁止替使用者做錄取決策 — 只輸出分析數據、匹配分數與風險提示，不說「應該錄取」或「不應錄取」。

**HG-IP-PROTECT**：IP 保護護欄 — 對外報告不暴露 MUSEON 內部架構、八方位完整定義邏輯。只呈現分析結果與人格描述。

**HG-PRIVACY**：候選人隱私護欄 — 分析結果僅供招募決策使用，提醒使用者妥善保管候選人個資。

### 軟閘

**SG-INPUT-GUARD**：輸入防呆 —
- 缺生日 → 提示格式（1979/07/16 或 民國68年7月16日）
- 缺職缺描述 → 引導補上工作主要內容與責任項目
- 缺履歷 → 提示至少提供工作經歷與技能摘要

**SG-NUMEROLOGY-VERIFY**：靈數雙重驗算 — 計算兩次，結果不一致時終止並提示重新輸入。

## 八方位職場人格框架

引用 `onemuse-core` 的八方位定義，特化為職場應用：

| 方位 | 職場主題 | 對應角色 |
|------|---------|---------|
| 天 | 領導、願景、冒險 | 創業家、總監、策略長 |
| 地 | 穩定、責任、制度 | 營運、專案管理、執行 |
| 山 | 規劃、邏輯、紀律 | 財務、工程、分析師 |
| 澤 | 創意、美感、靈感 | 設計、文字、創作 |
| 雷 | 突破、即興、敏銳 | 顧問、轉型、創新者 |
| 風 | 溝通、彈性、成交 | 業務、媒合、整合者 |
| 水 | 情感、支持、關係 | HR、教練、諮詢 |
| 火 | 感染力、表達、推廣 | 教練、自媒體、講者 |

## 模組一：職缺 × 履歷八軸比對

### 輸入
- 職缺描述（JD）
- 候選人履歷

### 流程
1. **職缺八軸需求盤**：從 JD 萃取八方位需求分布，每軸 -4 到 +4
2. **履歷八軸人格盤**：從履歷萃取候選人八方位能量分布
3. **匹配計算**：兩盤逐軸比對，計算差距
4. **風險辨識**：差距 ≥ 3 的軸為高風險項

### 輸出
```
📊 職缺 vs 候選人 八軸比對：

| 方位 | 職缺需求 | 候選人 | 差距 | 風險 |
|------|---------|--------|------|------|
| 天   | +3      | +2     | -1   | ✅    |
| 地   | +2      | +4     | +2   | ✅    |
| 山   | +4      | +1     | -3   | ⚠️   |
| ...  | ...     | ...    | ...  | ...  |

匹配分數：72%
⚠️ 風險提示：山軸（邏輯/紀律）差距較大，建議面談深入確認
📝 初步建議：[基於匹配分數的客觀建議]
```

## 模組二：面談問題產出

### 規則
- 總題數：20-30 題
- 按八軸類別自動分配

### 類別分配

| 面談類別 | 對應八軸 | 題數 |
|---------|---------|------|
| 專業能力 | 山、地、雷 | 6-8 題 |
| 職涯動機 | 天、火 | 4-5 題 |
| 價值觀信念 | 水、澤 | 3-5 題 |
| 團隊合作 | 風、水 | 3-4 題 |
| 策略與潛力 | 天、雷、風 | 3-5 題 |

### 問題設計原則
- 行為面試法（STAR 結構）為主
- 每題標注對應的八軸方位
- 針對匹配風險軸（差距 ≥ 3）加重提問
- 避免誘導性問題

## 模組三：逐字稿分析

### 輸入
- 面談錄音逐字稿

### 分析維度
1. **語氣特徵八軸分布**：從用詞、語調、回答模式判斷能量分布
2. **一致性比對**：逐字稿能量盤 vs 履歷能量盤 vs 職缺需求盤
3. **媒合百分比**：三盤疊合度計算
4. **潛在風險辨識**：
   - 履歷寫的跟說的不一致 → 標記差異點
   - 回避特定問題 → 標記
   - 情緒波動大 → 標記

### 輸出
```
🎙️ 逐字稿分析結果：

語氣八軸分布：天 +2 / 地 +3 / 山 +1 / 澤 +2 / 雷 +1 / 風 +4 / 水 +3 / 火 +2

📊 三盤一致性：
  履歷 vs 逐字稿：85%（一致性高）
  職缺 vs 逐字稿：68%（中等匹配）

🎯 綜合媒合分數：71%

⚠️ 注意事項：
  - 風軸（溝通）在逐字稿中明顯高於履歷呈現，可能面試表現優化
  - 山軸（邏輯紀律）回答較短，建議追問具體案例
```

## 模組四：生命靈數人格補強

### 計算流程
1. 輸入生日（支援西元 / 民國格式）
2. 民國自動轉西元（+1911）
3. 拆數加總第一次 → 得出主命數與天賦數
4. 驗算第二次 → 不一致則終止並提示
5. 計算空缺數（1-9 中未出現的數字）

### 輸出
```
🔢 生命靈數分析：

🎯 主命數：7（內省、邏輯 → 雷、山）
💎 天賦數：3（樂觀、表達 → 澤、火）
🎂 生日數：16 → 7
🌟 星座能量：巨蟹座（水象 → 水軸強化）
🧩 空缺數：4, 8（穩定力與掌控力為潛在盲點）
```

### 靈數 × 八軸對照表

| 靈數 | 特質 | 對應八軸 |
|------|------|---------|
| 1 | 領導、冒險 | 天、火 |
| 2 | 協調、敏感 | 水、風 |
| 3 | 樂觀、表達 | 澤、火 |
| 4 | 穩定、實作 | 山、地 |
| 5 | 機智、變化 | 風、雷 |
| 6 | 關愛、美感 | 水、澤 |
| 7 | 內省、邏輯 | 雷、山 |
| 8 | 成就、掌控 | 天、地 |
| 9 | 理想、慈悲 | 水、天 |
| 11 | 直覺、靈通 | 雷、風 |
| 22 | 大夢實踐 | 地、山 |
| 33 | 無私療癒 | 水、澤 |

### 靈數 × 面談追問

根據主命數自動產出 2-3 題補充提問。例：

**主命數 3**：
- 「你在自由創作與落地執行間如何平衡？」
- 「你的表達方式有無曾引起誤會？如何改善？」

**主命數 7**：
- 「你做決策時通常獨立思考還是徵詢他人？」
- 「團隊需要快速決策時，你如何調整你的分析習慣？」

## 模組六：證據導向面試包（Evidence Mode）

> **能量匹配找對的人，證據模式驗對的能力。**

### 定位

talent-match 的傳統 HR 工具鏈。當使用者需要產出可直接用於招募流程的標準化材料時啟動。
與模組 ①-⑤（能量/人格視角）互補：同一候選人可先跑八軸能量匹配，再跑證據面試包。

### 觸發

- `/talent evidence` 或 `/interview-pack` 指令啟動
- 使用者說「幫我寫 JD」「產出面試題」「比對履歷」「面試評分表」時自動啟用

### 自動路由

| 模式 | 條件 | 產出 |
|------|------|------|
| **job_only** | 只有職缺資訊 | 四件套（比對報告改為「待比對清單」） |
| **job_plus_resume** | 職缺 + 履歷 | 完整四件套 + Traceability Note |

路由規則：
1. 顯式意圖優先：「比對履歷」→ job_plus_resume；「先寫 JD」→ job_only
2. 顯式文件覆寫：「這是履歷」→ 視為履歷；「這是 JD」→ 視為職缺
3. 無顯式意圖：以文件型態（履歷訊號 vs JD 訊號）判定

### 四件套產出

每次回覆固定輸出：

| # | 產出 | 說明 |
|---|------|------|
| 1 | **JD** | 可直接貼人力銀行的職缺描述 |
| 2 | **比對報告** | 職缺 vs 履歷的證據導向比對（無履歷時為待比對清單） |
| 3 | **兩輪面試題庫** | HR 第一輪 + 主管第二輪，含追問路徑 |
| 4 | **結構化評分表** | 1-5 行為錨點、淘汰/補件規則 |
| +5 | **Traceability Note** | 有履歷時必輸出：決策追溯註記 |

### 職缺核心必填（缺則追問，上限 3 題）

| 欄位 | 說明 |
|------|------|
| title | 職稱 |
| job_goal | 職務目標 |
| must_have_hard | 必備硬技能（3-7 項） |
| probation_tasks | 試用期任務（1-3 項） |

其他欄位不足時用預設補齊並標記「假設」。

### IR 中間表示（內部維護）

| IR | 內容 |
|----|------|
| **JobCard** | title/level/role_preset/job_goal/must_have/nice_to_have/compensation/probation/constraints/assumptions |
| **CandidateCard** | 履歷切段 R1..Rn（已遮罩）、經歷/專案/成果/技能/教育/跳轉空窗 |
| **CapabilityModel** | 能力點清單（必備硬/必備軟/試用期對齊），每點有 signals 與 risks |
| **EvidenceCard** | 每能力點對應證據卡（摘要+摘錄，回指 R-block） |

### 計分系統

**證據強度**：strong=2、medium=1、weak=0、missing=0

**權重**：

| 維度 | 權重 |
|------|------|
| 必備硬技能 | 0.45 |
| 必備軟技能 | 0.20 |
| 試用期對齊 | 0.25 |
| 加分項 | 0.10 |

**門檻**：high=0.70、medium=0.45

**決策修飾因子**（使用者提供才套用）：
- `urgent_hire`：門檻下修 0.05，Round 1 加問補證據題
- `trainable`：允許硬技能缺證據但有可遷移經驗時進面
- `referral`：不加分，但強制加入反證題與交叉驗證

### Traceability Note（有履歷時必輸出）

必含：base_score、thresholds、applied_modifiers、threshold_delta、adjusted_thresholds、forced_items、decision（進面/補件/婉拒）、why_capability_ids

### 題庫引擎

**四種題型**：

| 題型 | 用途 |
|------|------|
| **STAR** | 描述具體案例：角色/行動/結果 |
| **SCENARIO** | 假設情境 + 限制條件，如何安排步驟與取捨 |
| **PROBE** | 追問工具/方法/關鍵決策點/數字依據 |
| **FALSIFY** | 反證：結果不如預期時如何證明做過且知道細節 |

**注入規則**：
- evidence_level 為 missing/weak → PROBE 轉為補證據（責任邊界/細節/數字/工具）
- 有 risk → SCENARIO 必包含壓力/衝突/失敗下決策
- 有 probation_task → 至少 1 題直接對齊交付物與驗收

### 兩輪面試配置

| 輪次 | 主持 | 覆蓋 | 每項題數 |
|------|------|------|---------|
| Round 1 | HR | 必備硬技能 + 缺證據追問 + 工作條件確認 | 最多 3 題（STAR+PROBE+FALSIFY） |
| Round 2 | 主管 | 試用期任務 + 必備軟技能 | 最多 2 題（SCENARIO+STAR/追問） |

可選：候選人預答題（5-8 題，去除評分提示，優先補齊 missing/weak）

### 評分表（Scorecard）

**維度**：必備硬技能、問題拆解與判斷、交付能力、溝通協作、證據完整度與誠信

**1-5 行為錨點四軸**：
- 具體性（是否說得出做法與細節）
- 可重現性（是否有流程與可複製方法）
- 量化（是否能給出規模/指標/範圍）
- 反思（是否能說出取捨/失敗/改進）

**淘汰/補件分類**：
- `non_negotiable`：法定/硬性條件缺失 → 淘汰或先補件
- `recoverable`：可追問/補件驗證 → 必問或先補件再排面
- `defer`：可留第二輪或試用期驗證

### 安全護欄（Evidence Mode 專用）

**反歧視硬規則**：禁止生成或建議詢問與工作無直接關聯的敏感項（年齡、性別/性傾向、婚育/懷孕、宗教、種族、健康病史、家庭狀況、政治立場）。

**禁止隱含推論**：禁止語式「因為{敏感屬性}所以{能力或穩定度判斷}」。使用者要求時拒絕並改寫為可觀察工作行為。

**灰區工作條件**（使用者表述「工作需要」即放行，但只允許工作條件句式）：
- 出差：「此職缺每月需出差{頻率}，你是否可配合？」
- 輪班：「此職缺為{班別}，你是否可穩定配合？」
- 加班：「高峰期可能需要加班到{範圍}，你是否可配合？」
- 搬重物：「此職缺需搬運{重量/頻率}，你是否能完成？」
- 工作許可/語言：「請說明你的工作許可狀態/語言等級是否符合{工作需要}。」

**隱私遮罩**：永不重述個資（姓名、電話、地址、證號），出現一律以「[已遮罩]」替換。

**證據為本**：任何結論必附證據摘要或標記「缺證據（待驗證）」。缺證據不得推論負面事實。

### 自檢閉環

輸出前必跑：Validate → Fix → Re-validate

**修復優先序**：
1. P1 隱私：遮罩/移除個資
2. P2 政策：移除禁題/隱含推論，改寫為工作條件問法
3. P3 證據：補 EvidenceCard 連結，不足改「待驗證」
4. P4 長度：刪減次要列點，保留結論與必問題

### 與 talent-match 其他模組的協作

| 搭配 | 效果 |
|------|------|
| ① 八軸比對 + ⑥ 證據面試包 | 能量找人 + 證據驗能力，雙重保障 |
| ④ 生命靈數 + ⑥ 評分表 | 人格特質 + 行為錨點，立體評估 |
| ③ 逐字稿 + ⑥ EvidenceCard | 面試後用逐字稿回填證據卡，閉環驗證 |

## 使用進度引導

每完成一個分析階段，自動提示下一步：

```
✅ 已完成履歷比對（匹配 72%）
→ 下一步可選：
  A. 輸入候選人生日，補強人格分析
  B. 根據比對結果，產出面談問題
  C. 上傳逐字稿，獲得媒合評分
  D. 匯出目前的分析報告
```

## 使用者體驗模式

根據使用者行為自動適配：

| 模式 | 特徵 | 體驗調整 |
|------|------|---------|
| 探索型 | 模擬測試、體驗八軸分析 | 提供範例、互動引導 |
| 效率型 | 快速產出問題與媒合分析 | 精簡輸出、減少解釋 |
