## 老闆脫鉤

**核心洞見**（EX_SSA01 D-Axis）：主權在於老闆與執行層的「脫鉤」，讓系統具備自運轉能力。目標：老闆在「沙灘」時系統不崩潰。

**執行算子**（EX_SSA01 S-Axis）：
1. 盤點重複性工作 → 列出老闆目前親手做的所有事
2. 轉譯為 SOP → 把可標準化的工作寫成步驟
3. 委派 → 找到對的人接手（不一定是員工，可以是外包/AI/自動化）
4. 監控異常 → 只管例外，不管例行

**診斷指標**（EX_SSA01 P-Axis）：
- 人治依賴度：若系統運作高度依賴單一自然人（老闆），判定為主權喪失
- 老闆每週花在例行工作的時數 > 60% → 脫鉤嚴重不足
- 老闆休假 3 天以上系統是否正常運轉 → 終極測試

**常見阻力與對策**：
- 「只有我做得好」→ 這是最大的系統風險，不是優勢
- 「教人太花時間」→ 教一次 vs 做一輩子，哪個貴？
- 「找不到人」→ 先拆分任務顆粒度，降低對人才的依賴門檻

## 子系統穩態

**核心洞見**（EX_SSA03 D-Axis）：系統包含多個子系統（供應、生產、銷售、服務、財務），缺一不可。

**三腳支撐算子**（EX_SSA03 S-Axis）：
- 確保至少有三種獲客管道：產品漏斗 × 社群漏斗 × 行銷漏斗
- 任一管道佔比 > 70% 即為系統不穩，需分散風險

**子系統失靈偵測**（EX_SSA03 P-Axis）：
- 供應鏈不穩 → 產品交付延遲/品質波動
- 銷售管道單一 → 一旦斷裂，營收歸零
- 服務體驗差 → 客戶流失率上升
- 財務不透明 → 賺錢但不知道賺在哪/虧在哪

**弱點分析算子**（EX_SSA12 S-Axis）：
- 朱蘭品質管理法：分析現有環節弱點 → 針對薄弱點優化 → 提升整體效能
- 挑戰看似不可能的目標（如 9000萬），從中逼出系統弱點
- 監控演化停滯：若相似問題重複發生，進入深度優化

## SOP 與合規治理

**核心洞見**（EX_SSA09 D-Axis）：法治勝於人治。規矩的一致性是系統權威的來源。

**SOP 斷路算子**（EX_SSA09 S-Axis）：
1. 定義不可觸碰紅線
2. 零容忍執行
3. 反饋修正

**治理診斷**（EX_SSA09 P-Axis）：
- 內部熵值：人為例外是否過多？
- 若例外 > 規則 → 治理失敗，啟動法度重塑
- 新人能否在不問老闆的情況下做出正確決定？

## 全人演化與八方位

**核心洞見**（EX_SSA18 D-Axis）：系統不只是商業，更是生命。事業成功但身體崩潰不是演化，是失衡。

**八方位體檢算子**（EX_SSA18 S-Axis）：

| 方位 | 對應 | 檢查重點 |
|---|---|---|
| 天 | 願景/使命 | 方向是否清晰？ |
| 風 | 行動/執行 | 行動是否在推進？ |
| 水 | 財務/資源 | 現金流是否健康？ |
| 山 | 穩定/累積 | 資產是否在增長？ |
| 地 | 身體/健康 | 體能是否支撐得住？ |
| 雷 | 能量/動力 | 動力是否充足？ |
| 火 | 關係/連結 | 核心關係是否穩定？ |
| 澤 | 喜悅/創造 | 是否還有樂趣？ |

**全人卡點偵測**（EX_SSA18 P-Axis）：
- 若事業（天/風/水）成功但身體（地/雷）卡住 → 演化失衡，強制修復
- 多維失衡時，先處理最低的方位（能量最低處）

**80/20 策略**（EX_SSA15）：
- 核心 20% 的 VIP 客戶是公司的突觸，給予極端優先的能量補給
- 若將大量精力花在 80% 的低價值客戶身上 → 資源誤配，啟動權重修正

## 系統呼吸與 PCT 閉環

**核心洞見**（EX_SSA27 D-Axis）：讓系統自動回正。維持一張一弛的節律。

**PCT 閉環算子**（EX_SSA27 S-Axis）：
1. 每週檢視 → 這週最重要的一件事做了嗎？
2. 反饋回路 → 哪裡偏離了？為什麼？
3. 自動回正 → 調整下週的行動

**過熱偵測**（EX_SSA27 P-Axis）：
- 系統連續運轉無呼吸 → 強制啟動休息
- 過熱訊號：決策品質下降、情緒反應過度、身體不適增加
- 回正儀式：晨間呼吸 + 願景設定

**傳承閉環**（EX_SSA25）：
- 教練結晶算子：我是誰 → 信仰核心 → 觀察洞察 → 成功個案
- 確保學員能將知識轉化為自身的肌肉記憶
- 監控傳承失真：輸出內容是否能體現品牌立場
