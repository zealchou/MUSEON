Section 05

SWOT 分析

針對 1-5 人微企市場的優劣勢全景。

💪 Strengths
  • 47 Skill 協作矩陣 — OpenClaw 13,729 個 Skill 寬但淺，MUSEON 47 個窄但深到骨髓
  • 工作流會自己長大 — Morphenix + 聯邦學習，越多客戶用，所有 WF 都變更好
  • 台灣繁中深度在地化 — 讀 PTT / Mobile01 語境，SSA 用台灣顧問業語言
  • 顧問方法論即產品 — SSA 27 染色體、business-12 是實戰經驗結晶，不是通用 prompt
  • 對話式零門檻 — 不需要拖拉節點，開口說問題就好
🔴 Weaknesses
  • 無視覺化介面 — 客戶看不見系統在幹嘛，信任建立慢
  • 自動化執行層弱 — 能想策略，但不能自動發 IG、排 LINE、接 CRM
  • 無社群生態 — OpenClaw 215K stars + Discord，MUSEON 目前 0 社群
  • 核心團隊精簡 — vs OpenClaw 社群 + OpenAI 資源
  • Web SaaS 版尚未建立 — 目前只有 CLI + Telegram + Electron
🟢 Opportunities
  • 台灣中小企 AI 導入潮 — 政府補助推動，79% 卡在試點，需要「顧問」不是「工具」
  • OpenClaw 安全信任危機 — 12% 惡意 Skill、Meta 研究員事件，MUSEON 的治理護欄是賣點
  • GPT Store 失敗後的空白 — 「賣模板」不行，「賣深度服務」的需求仍在
  • 合作顧問網路 — 台灣大量獨立顧問/教練需要 AI 提升產能
  • Dify/n8n 用戶的「最後一哩路」 — 建完流程不知道「然後呢」
🟠 Threats
  • OpenClaw 社群速度 — 每月 1,000+ 新 Skill，可能出現商業顧問類
  • Claude 自身持續進化 — 客戶可能覺得「直接用 Claude 就好」
  • Dify Marketplace 擴張 — 若出現高品質中文商業模板，搶低端市場
  • 定價壓力 — 競品都有免費方案，訂閱制需很強的價值感

Section 08

開發優先級

主力是 Web SaaS，所有優先級圍繞「讓小美用瀏覽器打開就能用」。

最大風險不是競品，而是：
Web UI 做不出來 → SaaS 無法上線 → 1-5 人微企進不來 → 所有策略都是空談。
P0

Web UI（客戶端介面）

沒有這個就沒有 SaaS 模式。對話式互動 + 工作流結果呈現 + 帳號管理。

P0

多租戶資料隔離

多個客戶同時使用母體，記憶/資產/會話必須完全隔離。已有 session_id 基礎，需擴展到資料層。

P0

Adobe 授權驗證系統

訂閱狀態管理 + 停付鎖定 + 方案層級控制。Web SaaS 版天然整合，子機版需心跳驗證。

P1

工作流進度視覺化

讓客戶看到「系統正在分析你的品牌…」的即時進度。建立信任、降低焦慮。

P1

結果美化輸出

HTML 策略報告、PDF 下載。讓客戶覺得「值這個錢」。已有 WF-INV-01 的 HTML 報告基礎。

P2

LINE / IG API 串接

台灣微企最常用的通道。補上「幫你做」的自動化執行能力。

P2

合作顧問管理後台

Tier 3 顧問通路的管理介面：授權、分潤、品質追蹤。Phase 2 才需要。

結論

MUSEON 的戰場不是「AI 工具平台」。
戰場是「AI 顧問訂閱服務」。

OpenClaw 是免費的萬用助手 — 寬而淺、安全存疑。
n8n 是自動化工具 — 需技術、不幫你想。
Dify 是 AI 應用建構器 — 需自建、通用無深度。

MUSEON 是月付制商業軍師 — 零門檻、深度方法論、越用越聰明。

護城河不在「能建工作流」—— 所有人都能做。
