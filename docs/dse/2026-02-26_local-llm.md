# DSE #2: Local LLM for MUSEON (MLX + Qwen3-8B)

**Date**: 2026-02-26 (previous session)
**Trigger**: User's "CPU 能做的就絕對不讓 GPU 做" philosophy. Need local inference for skill routing and simple tasks to reduce API token cost.

---

## Research Targets

| Framework | Platform | Quantization | Speed (tok/s) |
|-----------|----------|-------------|---------------|
| MLX | Apple Silicon only | 4-bit GGUF | ~40-80 tok/s on M-series |
| llama.cpp | Cross-platform | GGUF | ~30-60 tok/s on M-series |
| Ollama | Cross-platform (wraps llama.cpp) | GGUF | Similar to llama.cpp |
| vLLM | GPU (CUDA) | Various | Not suitable for Mac |

## Key Findings

### 1. MLX is the Best Choice for Apple Silicon

- Native Metal GPU acceleration
- Python-first API (fits MUSEON's Python backend)
- Active development by Apple ML team
- 4-bit quantized models fit in 8GB RAM

### 2. Qwen3-8B is the Recommended Model

| Model | Size (4-bit) | Quality | Speed | Multilingual |
|-------|-------------|---------|-------|-------------|
| Qwen3-8B | ~4.5 GB | Excellent for 8B | Fast | Chinese + English |
| Llama 3.1-8B | ~4.5 GB | Good | Fast | Primarily English |
| Mistral-7B | ~4.0 GB | Good | Fast | Primarily English |

Qwen3-8B wins because:
- Excellent Traditional Chinese support (critical for MUSEON)
- Strong instruction following
- Good at classification/routing tasks

### 3. Phased Integration Strategy

**Phase 1 (Recommended first step)**:
- Use local LLM only for **skill routing** (classify user intent → which skill to invoke)
- Simple classification task, doesn't need full conversation capability
- Reduces API calls by ~30-40% for routine queries

**Phase 2**:
- Local LLM handles simple Q&A and memory retrieval
- Claude API reserved for complex reasoning, creative tasks, and evaluation

**Phase 3**:
- Full hybrid routing: local for fast/simple, API for deep/complex
- Dynamic routing based on query complexity estimation

## Architecture Proposal

```
User message
  ↓
Local LLM (Qwen3-8B via MLX)
  ├→ Skill routing classification → Invoke matching skill
  ├→ Simple Q&A → Direct response
  └→ Complex/creative → Forward to Claude API
```
