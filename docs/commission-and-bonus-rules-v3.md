# MUSEON 業務獎金計算制度 v3.0（積分制）

## 文件概述

本文檔定義 MUSEON 業務獎金的「雙軸積分制」——同時獎勵「簽新客」和「養老客」。
適用於三層業務架構（經銷商 > 業務主管 > 業務）的獎金計算、升降級、結算與防禦機制。

**適用對象**：所有 MUSEON 業務人員、業務主管、經銷商
**最後更新**：2026-03-24
**版本**：v3.0

---

## 1. 產品定價基礎

| 項目 | 金額 | 說明 |
|------|------|------|
| 設定費 | NT$ 24,000（或根據市場調整） | 一次性，新客簽約時收取 |
| 月費 | NT$ 6,000/月（或根據市場調整） | 持續性，每月向客戶收取 |

**百分比基準**：無論售價如何調整，獎金始終以百分比計算，確保相對性原則。

---

## 2. 核心概念：雙軸積分

### 2.1 積分公式

```
月積分 = 當月新簽案件數 x 1 + 當月活躍客戶數 x 0.3
```

- **新簽案件**：當月內完成簽約且收到設定費的客戶，每案 1 分
- **活躍客戶**：當月有繳月費的既有客戶，每位 0.3 分
- 積分每月重新計算（不跨月累積）

### 2.2 白話解釋

> **簽一個新客戶 = 1 分；養活一個老客戶 = 0.3 分。分數越高，等級越高，抽成越多。**

### 2.3 積分計算範例

| 情境 | 當月新案 | 活躍客戶 | 月積分 | 等級 |
|------|---------|---------|--------|------|
| 新人第一個月，簽了 2 案 | 2 | 0 | 2.0 | 銅牌 |
| 做了半年，每月簽 4 案 | 4 | 24 | 4 + 7.2 = 11.2 | 金牌 |
| 養客高手，只簽 2 案但養 30 個老客 | 2 | 30 | 2 + 9 = 11.0 | 金牌 |
| 衝量型，月簽 12 案但只剩 5 個老客 | 12 | 5 | 12 + 1.5 = 13.5 | 金牌 |
| 資深經銷商，月簽 5 案 + 養 60 客 | 5 | 60 | 5 + 18 = 23.0 | 鑽石 |

## 3. 等級與獎金對照表

### 3.1 等級門檻

| 等級 | 月積分門檻 | 白話說明 |
|------|-----------|---------|
| 銅牌 | 0 – 3.9 分 | 新人起步期 |
| 銀牌 | 4.0 – 8.9 分 | 穩定出單 or 養了一批客戶 |
| 金牌 | 9.0 – 20.9 分 | 高產量 or 大量穩定客戶 |
| 鑽石 | 21.0 分以上 | 需具備經銷商資格 |

### 3.2 每案一次性獎金（百分比制，基於設定費計算）

| 等級 | 即時獎金 | 季度留任金 | 額外獎金 | 每案總分出 | 公司保留 |
|------|---------|-----------|---------|-----------|---------|
| 銅牌 | 設定費 × 25% | 設定費 × 12.5% | — | 設定費 × 37.5% | 設定費 × 62.5% |
| 銀牌 | 設定費 × 25% | 設定費 × 12.5% | 設定費 × 12.5% | 設定費 × 50% | 設定費 × 50% |
| 金牌 | 設定費 × 25% | 設定費 × 12.5% | 設定費 × 16.67% | 設定費 × 54.17% | 設定費 × 45.83% |
| 鑽石 | 設定費 × 25% | 設定費 × 12.5% | 設定費 × 25% | 設定費 × 62.5% | 設定費 × 37.5% |

發放規則：
- **即時獎金（25%）**：簽約收款後 7 天內發放，已發不追回
- **季度留任金（12.5%）**：每季末統一發放（3/6/9/12 月底），該季內離職則未發部分取消
- **額外獎金（12.5% ~ 25%）**：隨即時獎金一併發放

**舉例**：若設定費調整至 NT$30,000，銀牌業務每案獲得 $30,000 × 50% = $15,000（而非固定 $12,000）

### 3.3 月費分潤（百分比制，基於月費計算）

| 等級 | 每位客戶每月分給業務 | 公司保留 |
|------|-------------------|---------|
| 銅牌 | 月費 × 3.33% | 月費 × 96.67% |
| 銀牌 | 月費 × 8.33% | 月費 × 91.67% |
| 金牌 | 月費 × 16.67% | 月費 × 83.33% |
| 鑽石 | 月費 × 25% | 月費 × 75% |

**重點**：銅牌也有月租分潤（月費 × 3.33%）。只要你有活躍客戶就有被動收入，不管你是什麼等級。

**舉例**：若月費調整至 NT$8,000，銀牌每位客戶每月獲得 $8,000 × 8.33% = $664（而非固定 $500）

### 3.4 月費分潤發放時序

```
客戶月費繳入時間          業務分潤發放時間
---------------------    -----------------
次月 5 日前收到           -> 次月 25 日發放
次月 5 日後收到           -> 再次月 25 日發放（遞延一個月）
未收到                   -> 不發放（該客戶當月不計入活躍客戶）
```

## 4. 組織架構與三層分潤

### 4.1 三層架構

```
經銷商（需培訓認證，鑽石級，可招募並培訓業務團隊）
  |-- 業務主管（銀牌以上，管理業務團隊）
      |-- 業務人員（直接面對客戶）
```

### 4.2 三層月費分潤

三層為**獨立等級，各自抽成**，不疊加。每層依自己的等級對應月費分潤金額。

| 層級 | 分潤來源 | 說明 |
|------|---------|------|
| 業務 | 自己簽的客戶 | 依自己等級抽 |
| 業務主管 | 旗下業務簽的客戶 | 依主管自己等級抽 |
| 經銷商 | 旗下所有客戶 | 依經銷商自己等級抽 |

同一客戶最多被三層各抽一次。

### 4.3 業務主管晉升

- 月積分達銀牌（>= 4.0）即可晉升業務主管
- 適用升級規則（見第 5 節）

## 5. 升降級規則

### 5.1 積分計算週期

- 積分每月重新計算
- 新案件：月底前系統收到客戶設定費才計入當月
- 活躍客戶：當月有繳月費（或在免月租優惠期內）的客戶

### 5.2 升級

**當月積分達到更高等級門檻 -> 立即生效。**

不需等待多個月。積分本身已包含留存因子（養客 = 積分），不需要額外的時間窗口來平滑。

升級後，**所有名下客戶的月費分潤立即適用新等級**。

### 5.3 降級

**連續 2 個月**積分低於當前等級最低門檻 -> 第 3 個月降一級。

| 當前等級 | 連續 2 個月積分低於 | 降至 |
|---------|-------------------|------|
| 銀牌 | < 4.0 | 銅牌 |
| 金牌 | < 9.0 | 銀牌 |
| 鑽石 | < 21.0 | 金牌 |

**設計理念**：快升慢降。升級是即時獎勵（心理正面），降級有 2 個月緩衝（減少焦慮）。

## 6. 客戶流失與追回

### 6.1 流失處理

- 客戶不續約 -> 該客戶當月起不計入「活躍客戶」
- 月費分潤立即停止，無緩衝期
- 不區分流失原因（產品問題或業務問題）
- **流失自動影響積分**：少一個活躍客戶 = 少 0.3 分，可能影響等級

### 6.2 客戶追回

公司設有**客戶關係追回客服人員**：

| 步驟 | 說明 |
|------|------|
| 1. 調單確認 | 客服確認流失原因與追回可能性 |
| 2. 追回成功 | 客戶重新啟用，**免設定費** |
| 3. 分潤轉移 | 追回客戶的月費分潤歸該客服人員，原業務不再享有 |

**防禦規則**：追回客服不得為原負責業務（防止故意放流失再自己追回的套利）。

## 7. 客戶轉介（引薦）機制

### 7.1 引薦獎勵

| 對象 | 獎勵 |
|------|------|
| 引薦客戶（現有客戶） | 免 1 個月月租費（公司吸收） |
| 負責業務 | 一次性引薦獎金（設定費 × 12.5%）+ 正常新案獎金 + 月費分潤 |

### 7.2 引薦防套利規則

- 引薦獎金 $3,000 為一次性額外獎勵，**不計入積分**
- 引薦帶來的新案件本身照常計入案件積分（+1 分）
- 引薦客戶的免月租期間，業務的月費分潤照常發放（公司吸收）
- 引薦需主管書面確認（防止假引薦）

### 7.3 計算範例（假設設定費 $24,000、月費 $6,000）

```
客戶 A 引薦客戶 B，業務小美負責（當前銀牌）：

小美獲得：
  - 引薦獎金：$24,000 × 12.5% = $3,000（一次性，不算積分）
  - 新案即時獎金：$24,000 × 25% = $6,000（7 天內發）
  - 新案季度留任：$24,000 × 12.5% = $3,000（季末發）
  - 額外獎金：$24,000 × 12.5% = $3,000（銀牌，隨即時獎金一併發）
  - 客戶 B 月費分潤：$6,000 × 8.33% = $498/月（銀牌，每月持續）
  - 客戶 B 計入積分：+1 分（新案）+ 次月起 +0.3 分（活躍客戶）

客戶 A 獲得：
  - 下個月免繳 $6,000 月租
  - 小美在客戶 A 免租期間的分潤由公司吸收

---

**若售價調整為設定費 $30,000、月費 $7,500**，同樣引薦情況下小美獲得：
  - 引薦獎金：$30,000 × 12.5% = $3,750
  - 新案即時獎金：$30,000 × 25% = $7,500
  - 新案季度留任：$30,000 × 12.5% = $3,750
  - 額外獎金：$30,000 × 12.5% = $3,750
  - 客戶 B 月費分潤：$7,500 × 8.33% = $623/月
```

## 11. 結算流程

### 11.1 簽約獎金

```
客戶簽約並繳設定費
    |
    v（7 天內）
發放即時獎金 $6,000 + 額外獎金
    |
    v（每季末：3/6/9/12 月底）
發放季度留任金 $3,000（需在職）
```

### 11.2 月費分潤

```
每月 1 日 ~ 月末    客戶月費繳入期間
次月 5 日           截止確認日（5 日前到帳才算當期）
次月 25 日          發放對應業務的月費分潤
```

### 11.3 案件計入規則

| 情境 | 處理 |
|------|------|
| 3/20 簽約，3/25 收設定費 | 計入 3 月新案（+1 分） |
| 3/20 簽約，4/2 收設定費 | 遞延至 4 月新案（+1 分） |
| 簽約後 7 日內退約 | 不計案件，獎金不發 |
| 同統編 90 天內重複簽約 | 第二案不計入新案積分 |

## 13. 系統實裝規格（給開發者）

### 13.1 積分計算引擎（每月執行一次）

```python
def calculate_monthly_score(sales_id: str, year_month: str) -> float:
    """
    計算業務員的月積分。

    Args:
        sales_id: 業務員 ID
        year_month: 結算月份，格式 "2026-03"

    Returns:
        月積分（float，保留一位小數）
    """
    # 1. 計算當月新簽案件數
    #    條件：簽約月份 == year_month AND 設定費已到帳 AND 非 7 天內退約
    #    防拆單：同統編 90 天內只算第一案
    new_cases = count_new_cases(sales_id, year_month)

    # 2. 計算當月活躍客戶數
    #    條件：客戶狀態 == active AND 當月月費已繳（或在免月租優惠期內）
    active_customers = count_active_customers(sales_id, year_month)

    # 3. 積分公式
    score = (new_cases * 1.0) + (active_customers * 0.3)

    return round(score, 1)
```

### 13.2 等級判定引擎

```python
TIER_THRESHOLDS = [
    {"tier": "diamond", "min_score": 21.0, "requires": ["dealer_certification"]},
    {"tier": "gold",    "min_score": 9.0},
    {"tier": "silver",  "min_score": 4.0},
    {"tier": "bronze",  "min_score": 0.0},
]

def determine_tier(score: float, certifications: list[str]) -> str:
    """
    根據積分判定等級。從高到低比對，第一個符合的就是。
    鑽石需要額外的經銷商認證。
    """
    for t in TIER_THRESHOLDS:
        if score >= t["min_score"]:
            if "requires" in t:
                if all(r in certifications for r in t["requires"]):
                    return t["tier"]
            else:
                return t["tier"]
    return "bronze"
```

### 13.3 升降級引擎

```python
def evaluate_tier_change(sales_id: str, current_tier: str, year_month: str) -> str:
    """
    升降級判定。

    升級：當月積分達更高等級 -> 立即升級
    降級：連續 2 個月積分低於當前等級門檻 -> 降一級

    Returns:
        "promote" | "demote" | "maintain"
    """
    current_score = get_monthly_score(sales_id, year_month)
    new_tier = determine_tier(current_score, get_certifications(sales_id))

    # 升級：即時生效
    if tier_rank(new_tier) > tier_rank(current_tier):
        return "promote"

    # 降級：需連續 2 個月低於門檻
    if tier_rank(new_tier) < tier_rank(current_tier):
        prev_month = get_previous_month(year_month)
        prev_score = get_monthly_score(sales_id, prev_month)
        prev_tier = determine_tier(prev_score, get_certifications(sales_id))

        if tier_rank(prev_tier) < tier_rank(current_tier):
            # 連續 2 個月低於門檻，降級
            return "demote"

    return "maintain"
```

### 13.4 獎金計算引擎

```python
# 百分比制常數
SIGNING_BONUS_PERCENTAGES = {

MONTHLY_COMMISSION_PERCENTAGES = {
    "bronze":  0.0333,
    "silver":  0.0833,
    "gold":    0.1667,
    "diamond": 0.25,
}

REFERRAL_BONUS_PERCENTAGE = 0.125  # 12.5% 的設定費

def calculate_signing_bonus(tier: str, setup_fee: int) -> dict:
    """每案簽約獎金（百分比制，基於 setup_fee）。"""
    p = SIGNING_BONUS_PERCENTAGES[tier]
    immediate = int(setup_fee * (p["immediate"] + p["extra"]))
    quarterly = int(setup_fee * p["quarterly"])
    return {
        "immediate": immediate,  # 7 天內發放
        "quarterly": quarterly,   # 季末發放
        "total": immediate + quarterly,
    }

def calculate_monthly_commission(sales_id: str, tier: str, year_month: str, monthly_fee: int) -> int:
    """月費分潤總額 = 活躍客戶數 × 等級對應百分比 × 月費。"""
    active = count_active_customers(sales_id, year_month)
    percentage = MONTHLY_COMMISSION_PERCENTAGES[tier]
    per_customer = int(monthly_fee * percentage)
    return active * per_customer
```

### 13.5 反套利檢查引擎

```python
def anti_fraud_check(sales_id: str, customer_tax_id: str, year_month: str) -> dict:
    """
    簽約前的反套利檢查。

    Returns:
        {"allowed": bool, "reason": str}
    """
    # 1. 同統編 90 天內是否已有案件
    recent = get_cases_by_tax_id(customer_tax_id, days=90)
    if recent:
        return {"allowed": False, "reason": f"同統編 {customer_tax_id} 在 90 天內已有案件（{recent[0]['date']}）"}

    # 2. 跨業務轉單檢查
    original_sales = get_original_sales(customer_tax_id)
    if original_sales and original_sales != sales_id:
        if not has_manager_approval(sales_id, customer_tax_id):
            return {"allowed": False, "reason": "跨業務轉單需主管書面確認"}

    return {"allowed": True, "reason": "通過"}
```

## 14. 常見問題 (FAQ)

### Q1：積分怎麼算？
**A**：每月重新算。公式：`當月新簽案件 x 1 + 當月活躍客戶 x 0.3`。簽 1 個新客得 1 分，養 1 個老客得 0.3 分。

### Q2：我只簽了 2 案但養了 30 個客戶，等級會很低嗎？
**A**：不會。你的積分 = 2 + 30 x 0.3 = 11 分，是金牌。養客也能升級。

### Q3：升級後，之前的客戶月費分潤也跟著升嗎？
**A**：是的。全客戶跟著等級走。你從銀牌升金牌，所有客戶的月費分潤從 $500 跳到 $1,000。

### Q4：客戶跑了會怎樣？
**A**：(1) 該客戶月費分潤立即停止。(2) 你的活躍客戶數減少 -> 積分下降 -> 可能影響等級。流失有自然懲罰。

### Q5：離職後獎金怎麼算？
**A**：已發的即時獎金不追回。未發的季度留任金取消。月費分潤離職次月停止。

### Q6：引薦客戶算不算我的案件？
**A**：引薦帶來的新客戶照常計入你的新案積分（+1 分）。額外的引薦獎金 $3,000 是一次性的，不計入積分。引薦需主管確認。

### Q7：同一個客戶可以重複簽約嗎？
**A**：同一統編 90 天內只算 1 次新案。90 天後續約或重簽可再計 1 次。

### Q8：降級會很快嗎？
**A**：不會。需要**連續 2 個月**積分低於門檻才降一級。偶爾一個月表現差不會降級。

### Q9：銅牌也有月租分潤嗎？
**A**：是的，$200/客戶/月。只要你有活躍客戶就有被動收入，不管你是什麼等級。

### Q10：加掛系統或客製化開發，獎金怎麼算？
**A**：如需加購系統或客製化系統，相關獎金制度另外計算，與 MUSEON 完全分開，不疊加。
