## 1. Skill 概述

**名稱**：ScreenplayForge（劇本鍛造局）
**用途**：將客户需求轉化為結構化、可製作的劇本
**目標使用者**：廣告創意總監、短片製片人、品牌故事編劇
**交付成果**：劇本文本 + 製作清單 + 情緒線可視化

---

## 2. 輸入規格

### 必須欄位（Required）

```json
{
  "project_name": "string",           // 專案名稱，例："品牌微劇本 #咖啡日常"
  "genre": "enum",                    // 劇本類型：短篇|廣告|微劇本|話劇
  "target_duration": "string",        // 目標時長：30秒|1分鐘|3分鐘|5分鐘|10分鐘
  "theme": "string",                  // 主題/核心概念，例："一個媽媽的日常抉擇"
  "num_characters": "integer",        // 角色數：1-8（超過8個需要群演設定）
  "emotional_tone": "enum"            // 基調：溫馨|懸疑|喜劇|激勵|感傷|驚悚
}
```

### 選填欄位（Optional）

```json
{
  "target_audience": "string",        // 目標觀眾：年輕上班族|中高層主管|家庭主婦
  "setting": "string",                // 場景限制：辦公室|家裡|戶外|混合
  "key_props": "array[string]",       // 必須包含的道具清單：["咖啡杯", "手機"]
  "brand_values": "array[string]",    // 品牌價值觀：["創新", "人文", "簡約"]
  "special_requirements": "string",   // 特殊要求：音樂要求|無對白|多語言等
  "budget_tier": "enum"               // 製作預算：low|medium|high
}
```

### 輸入驗證
- target_duration 必須是標準值，生成時自動計算場景數推薦值
- num_characters 若 > 8，自動提示「建議加入群演設定」
- emotional_tone 與 genre 的組合合理性檢查（例：驚悚+喜劇需要特殊處理）

## 3. 處理流程（Processing Pipeline）

### 階段 1：需求理解與結構化（1 分鐘）

**輸入**：上述 JSON 結構

**流程**：
```
1. 驗證輸入完整性
2. 根據 target_duration 推薦 scene count
3. 初步角色分型（主角、對立角、陪襯角色、群演）
4. 確認結構：自動採用「三幕劇」（劇本標準）
   - 第一幕：10-15% 時長（介紹 + 衝突起因）
   - 第二幕：50-70% 時長（衝突深化 + 多個轉折點）
   - 第三幕：20-30% 時長（高潮 + 結局）
```

**輸出**：Screenplay Brief（結構框架、場景數、角色清單）

---

### 階段 2：角色檔案建立（3 分鐘）

**流程**：
```
對每個角色生成：
  a) 背景與身份（年齡、職業、社會階級）
  b) 核心欲望（想得到什麼）
  c) 內在恐懼（害怕失去什麼）
  d) 關鍵傷口（過去的創傷/遺憾）
  e) 角色弧線（起點情緒 → 轉折點 → 終點蛻變）
  f) 說話風格（用詞、語速、口頭禪、方言）
  g) 與其他角色的關係（衝突點在哪）
```

**驗收點**：
- [ ] 主角應有清晰的「成長/改變」
- [ ] 主角 vs 對立角之間有「值得衝突」的原因（不是無故對抗）
- [ ] 每個角色都能用「一句話」概括其動機

---

### 階段 3：情緒線與張力設計（3 分鐘）

**流程**：
```
生成情緒表，列出每場景的「目的」與「預期觀眾情感」：

| 場景 | 情節概要 | 主角狀態 | 預期觀眾情感 | 張力來源 |
|------|--------|--------|----------|--------|
| 1    | 介紹日常 | 平靜   | 親切感   | 無（建立信任） |
| 2    | 衝突出現 | 困惑   | 懸念     | 外部威脅 |
| 3    | 主角掙扎 | 焦慮   | 同情     | 內在困境 |
| ... | ... | ... | ... | ... |
| N    | 蛻變後的新日常 | 寧靜 | 釋放感   | 衝突解決、人物改變 |

目標：情感曲線應呈「波浪狀」而非「平坦」或「無限上升」
```

**驗收點**：
- [ ] 每場景都有「為什麼觀眾應該在乎」的理由
- [ ] 內在衝突（角色 vs 自己）出現次數 ≥ 2 次
- [ ] 至少有 1 次「出乎意料的轉折」（推翻觀眾預期）
- [ ] 第三幕的情感高潮不是「突然發生」，而是「前兩幕鋪墊充分」

---

### 階段 4：場景大綱與道具設計（5 分鐘）

**流程**：

A) **場景大綱** — 每場景生成：
```
【場景 1：早晨的廚房】
  時間：6:30 AM
  地點：現代簡約廚房（陽光透過大窗）
  角色：媽媽（準備早餐）、女兒（還在睡夢中）
  場景目的：建立主角的日常+隱喻其「為愛付出卻感孤獨」

  視覺敘事：
    - 廚房乾淨但冷清（象徵秩序中的寂寞）
    - 媽媽快速、機械化的動作（習慣、無意識）
    - 早晨的冷色光線（隱喻心理狀態）

  轉折點：[如有] 例如：發現冰箱空了，女兒最愛的牛奶沒了

  場景時長：60 秒
```

B) **道具清單** — 生成完整清單：
```json
{
  "scene_1_kitchen": [
    {
      "name": "牛奶瓶（空）",
      "purpose": "視覺象徵 - 匱乏、遺憾",
      "appears_in_scenes": [1, 3, 5],
      "emotional_weight": "high",
      "cost_tier": "free",
      "notes": "應選用常見品牌，增進親近感"
    },
    {
      "name": "懷舊相框",
      "purpose": "敘事道具 - 揭示角色過去",
      "appears_in_scenes": [2, 4],
      "emotional_weight": "medium",
      "cost_tier": "low",
      "notes": "照片應該模糊、泛黃，暗示時間流逝"
    }
  ]
}
```

**驗收點**：
- [ ] 每場景有明確的「視覺敘事意圖」，不只是地點交代
- [ ] 道具清單涵蓋「敘事」+「象徵」+「製作成本」三個維度
- [ ] 場景數與時長匹配（例：3 分鐘 = 6-9 場景）
- [ ] 道具在多個場景反覆出現（強化象徵意義）

---

### 階段 5：初稿對白與舞台指示（10-15 分鐘）

**流程**：
```
生成標準格式劇本（Fountain 格式）：

INT. KITCHEN - MORNING

A modern, minimalist kitchen. SUNLIGHT streams through a large window.
MOM (40s), in pajamas, moves with mechanical precision.

She opens the fridge. It's mostly empty except for...
vegetables, a bottle of jam, no milk.

Her expression darkens.

MOM
(to herself)
又沒了。

[She reaches for the phone, hesitates]
...

```

**對白設計原則**：
- [ ] 每人物有不同的說話風格（年齡、階級、教育背景）
- [ ] 避免「說教」（角色不應解釋自己的情感）
- [ ] 對白應「推動劇情」或「暴露角色」，絕不閒聊
- [ ] 本土化方言自然混入（不生硬）

**舞台指示**：
- [ ] 寫出「角色如何做」而非「角色為什麼這樣做」（讓觀眾推理）
- [ ] 動作應有「情感符號意義」（例：緊握拳頭 = 內在衝突）
- [ ] 不過度描寫，保留給導演/演員詮釋的空間

---

### 階段 6：情緒線驗收與修改建議（3 分鐘）

**自動檢查清單**：

```
✓ 三幕完整性
  - 第一幕有「衝突引爆點」(Inciting Incident) 嗎？ → [ ]
  - 第二幕有「中點轉折」(Midpoint) 嗎？ → [ ]
  - 第三幕有「最後衝刺」(Climax) 嗎？ → [ ]

✓ 情緒連貫性
  - 主角的「外在目標」和「內在目標」是否衝突？ → [ ]
  - 觀眾的情感曲線是否有「釋放點」？ → [ ]

✓ 角色蛻變
  - 主角在結尾是否「與開頭不同」？ → [ ]
  - 蛻變是「自然推導」還是「突然出現」？ → 自然推導 / 突然出現

✓ 敘事效率
  - 是否有「無故場景」（不推動情節、不暴露角色）？ → 有 / 無
  - 對白中是否有「冗長獨白」(> 4 行)？ → 有 / 無

✓ 製作可行性
  - 場景數是否合理（時長 3 分鐘應該 6-9 場景）？ → [ ]
  - 道具清單是否超出「低預算」限制？ → [ ]
  - 角色數是否在「可招募」範圍？ → [ ]
```

**修改建議**：自動生成「建議修改點」清單，例如：
```
⚠️ 場景 3 的對白過長（7 行）→ 建議縮短到 3-4 行
💡 牛奶瓶只在場景 1 出現 → 建議在第三幕再次出現，強化象徵
🎯 主角的「成長」不夠明確 → 建議在結尾加一句「新的決定」來證明蛻變
```

## 5. 品質標準

### Tier 1：結構完整性
- [ ] 三幕劇結構清晰（起承轉合明確）
- [ ] 衝突引爆點在第一幕 10-15% 位置
- [ ] 中點轉折在 50% 位置，改變劇情方向
- [ ] 高潮在第三幕 85-95% 位置
- [ ] 解決方案在 95-100% 之間（不超過 3 場景）

### Tier 2：情感設計
- [ ] 情緒曲線呈波浪狀，≥ 3 個「小高潮」
- [ ] 內在衝突（角色 vs 自己）≥ 2 處
- [ ] 主角有「以為一件事，實際另一件事」的發現
- [ ] 結尾釋放張力，觀眾感到「滿足」或「警醒」

### Tier 3：敘事效率
- [ ] 所有場景都「推動情節」或「暴露角色」
- [ ] 對白無冗長獨白（最長 4 行）
- [ ] 每場景有一個「單一焦點」（不混淆觀眾）
- [ ] 無重複訊息（同一情感/訊息不重複表達）

### Tier 4：製作可行性
- [ ] 場景數與時長合理（時長 min = 場景 count / 2）
- [ ] 道具清單有「象徵意義」標註
- [ ] 佈景設計可 DIY 或低成本搭建
- [ ] 角色數 ≤ 預算允許的招募數
- [ ] 特殊效果 / 聲效清單獨立列舉

## 6. 使用案例與期望輸出

### 案例 1：30 秒廣告短片（咖啡品牌）

**輸入**：
```json
{
  "project_name": "品牌微劇本 #咖啡日常",
  "genre": "廣告",
  "target_duration": "30秒",
  "theme": "一個媽媽在忙碌中找到片刻寧靜",
  "num_characters": 2,
  "emotional_tone": "溫馨",
  "key_props": ["咖啡杯", "手機"],
  "budget_tier": "low"
}
```

**預期輸出**：
- 劇本文本（1 分鐘短篇，4-5 場景）
- 製作清單（場景簡單、道具易取）
- 情緒線表（4-5 個情感轉折點）
- 驗收報告（PASS）
- 預製時間：3-5 天

### 案例 2：10 分鐘話劇片段（教育題材）

**輸入**：
```json
{
  "project_name": "教育微劇本 #勇氣",
  "genre": "短篇",
  "target_duration": "10分鐘",
  "theme": "少女在朋友壓力中堅持自我",
  "num_characters": 5,
  "emotional_tone": "激勵",
  "target_audience": "國中生",
  "budget_tier": "medium"
```

**預期輸出**：
- 劇本文本（10 分鐘，8-12 場景）

- 詳細情緒線表（多層衝突）
- 製作成本估算
- 驗收報告（附改進建議）
- 預製時間：5-7 天

## 7. 技術實作備註

### Prompt 工程策略
1. **結構化 CoT (Chain of Thought)**：逐步引導 Claude 完成「理解 → 設計 → 驗收」
2. **角色扮演**：讓 Claude 扮演「資深劇本編劇」，啟動專業知識
3. **範例引導 (Few-shot)**：提供 1-2 個好劇本範例，提升品質
4. **檢查清單強制執行**：每階段完成前必須通過檢查清單

### 記憶與版本管理
- 每個劇本版本用 Git 追蹤
- 修改建議與實際修改分開記錄
- 製作清單與最終劇本綁定（日期戳記）

### 輸出格式相容性
- **Markdown**：易讀易編輯（MUSEON 標準）
- **Fountain**：通用劇本格式，可轉 PDF / 專業軟體
- **JSON**：結構化數據，便於工具整合
- **HTML**：可視化呈現（圖表、檢查清單）

---

## 8. 風險與降級方案

| 風險 | 指標 | 降級方案 |
|------|------|--------|
| **劇本過長** | 場景數 > 20 | 自動提示「建議拆成兩部分」+ 簡化非必要場景 |
| **情緒線平坦** | 轉折點 < 2 | 提示「衝突不足」+ 建議加入「角色發現」 |
| **角色過多** | num_characters > 10 | 提示「招募困難」+ 建議群演設定 |
| **製作成本超預算** | prop_cost > 預算 × 2 | 自動生成「替代方案」(用日常物件代替) |
