# DSE：舞蹈劇本優化方法論與 Skill 設計

## 研究日期
2026-03-25

## 研究背景
羅臻的創作需求：將 5 分鐘的舞蹈劇本（3 分鐘戲劇 + 2 分鐘舞蹈）壓縮到 3.5 分鐘，保留笑點與節奏感。

---

## 第一層發現：專業劇本理論

### 1.1 經典劇本結構框架

根據 Freytag 金字塔結構，所有戲劇作品都遵循五階段模式：

| 階段 | 功能 | 時長配比（5分鐘版） | 時長配比（3.5分鐘版） |
|------|------|------------|------------|
| **Exposition（開場）** | 介紹角色、設定、背景 | 0:20-0:30 | 0:10-0:15 |
| **Rising Action（上升）** | 衝突建立、笑點埋伏 | 1:00-1:30 | 0:45-1:00 |
| **Climax（高潮）** | 最大張力、核心笑點 | 0:45-1:00 | 0:35-0:45 |
| **Falling Action（下降）** | 後續反應、餘韻 | 0:40-0:50 | 0:30-0:40 |
| **Resolution（結尾）** | 新狀態確立、道別 | 0:15-0:20 | 0:10-0:15 |

**關鍵原則**：
- 開場必須快速進入（不超過 30 秒）
- 高潮區域必須被保護（笑點密集區）
- 結尾應有明確的「新常態」信號

### 1.2 喜劇時序的科學

根據 Jerry Seinfeld 與現代喜劇研究，笑點的有效性取決於三個因素：

1. **Setup-Punchline 的比例**
   - Setup 應該簡潔、清晰
   - Punchline 應該產生「反轉」或「出乎意料」
   - 舞蹈劇本中：戲劇段落負責 Setup，舞蹈段落負責 Punchline（身體語言的視覺反轉）

2. **節奏的密度（Joke Compression）**
   - 不是笑點越多越好，而是笑點間距的**呼吸感**
   - 5 分鐘版本可能有 4-5 個主笑點
   - 3.5 分鐘版本應保留 3-4 個**核心笑點**，並加密節奏
   - 「觸發『the roll』」：第一個笑點引發笑聲，第二個笑點在笑聲尚未完全結束時到達，形成波浪式的累積笑聲

3. **停頓的藝術（The Pregnant Pause）**
   - 一個有效的停頓 = 一行台詞
   - 時間壓縮時，停頓首先被刪減
   - 但**核心笑點前的停頓必須保留**，因為它製造預期、信號「笑點要來了」

### 1.3 戲劇弧線與舞蹈的整合

舞蹈不是獨立的表演，而是**戲劇語言的延伸**：

- **戲劇段落**：傳遞 Setup、角色情緒、背景信息（言語 + 小動作）
- **舞蹈段落**：濃縮情緒、視覺衝擊、身體的詩學（純粹運動 + 音樂）

**整合原則**：
- 戲劇中的「內在衝突」應該在舞蹈中被**視覺化**
- 舞蹈的時長應該與情緒強度成正比
  - 高潮附近：舞蹈更長、更激烈、笑點更集中
  - 下降段：舞蹈變短，過渡到結尾

## 第二層發現：舞蹈劇場的特殊考量

### 2.1 舞蹈在劇場中的七大功能

根據 Broadway 與現代劇場研究，舞蹈可以發揮：

1. **角色心理探索**：內在衝突變成身體語言（無需台詞）
2. **敘事工具**：傳達時間流逝、地點轉換、角色轉變
3. **非言語的 Libretto**：舞蹈取代對話，傳達意義
4. **轉場裝置**：連接兩個場景，節省過場時間
5. **角色發展**：通過舞蹈的變化展現角色成長
6. **隱喻表達**：抽象概念的具象化（例如：逃脫的舞蹈動作 = 心理解放）
7. **視覺純粹性**：無故事敘述，純粹美學享受

**時間壓縮時的應用**：
- 舞蹈不能只是「裝飾」，必須承載核心信息
- 每 10 秒的舞蹈都應該是一句話、一個情緒、一個轉折

### 2.2 喜劇與舞蹈的張力

舞蹈喜劇的核心矛盾：
- **言語喜劇**：依靠出乎意料的想法（Punchline）
- **身體喜劇**：依靠出乎意料的動作（Physical Comedy）

**整合策略**：
- 台詞設置一個「理性預期」
- 舞蹈動作違反或超越這個預期
- 例如：「我要認真學芭蕾」（台詞）→ 然後做出完全相反的笨拙動作（舞蹈）

### 2.3 時間壓縮時舞蹈比例的調整

**原始配比（5 分鐘）**：
- 戲劇：3 分鐘（60%）
- 舞蹈：2 分鐘（40%）

**壓縮配比（3.5 分鐘）**：
**選項 A - 均勻壓縮**：
- 戲劇：2 分 6 秒（60%）
- 舞蹈：1 分 24 秒（40%）
（所有部分都削減 30%）

**選項 B - 保護舞蹈**：
- 戲劇：1 分 30 秒（43%）
- 舞蹈：2 分 00 秒（57%）
（舞蹈只削減 0%，戲劇削減 50%）

**選項 C - 加密戲劇，優化舞蹈**：
- 戲劇：1 分 50 秒（53%）
- 舞蹈：1 分 40 秒（47%）
（通過加密台詞與動作，舞蹈從「冗長」變為「精而悍」）

**建議**：採用 **C 方案**，因為舞蹈是視覺衝擊力最強的元素，不應過度削減。

## 第三層發現：劇本壓縮的五大策略

### 3.1 策略一：入點晚（Late Entry）

**原則**：不在屏幕上/舞台上進行背景交代，讓它發生在「幕後」

**應用**：
- ❌ 「我是一個學過芭蕾的上班族，今天要參加比賽...」（15秒背景）
- ✅ 「（舞蹈三秒，身著運動服衝上舞台）準備好了嗎？」（背景信息通過視覺已知）

**時間救濟**：每段背景交代平均省 10-15 秒

### 3.2 策略二：多目標元素（Multi-Purpose Elements）

**原則**：讓單一元素同時服務多個目的

**應用**：
- **單一用途**：「（演員摔跤）」= 製造喜劇效果
- **多目標**：「（演員摔跤，動作轉為舞蹈）」= 製造喜劇效果 + 展示角色的樂觀 + 過渡到高潮舞蹈

**技術**：
1. 重複使用相同的 Prop（道具）但賦予新意義
2. 角色動作同時兼含多層含義
3. 舞蹈段落不只是「舞」，而是也要推進情節

**時間救濟**：每個被多利用的元素省 5-10 秒

### 3.3 策略三：潛台詞與密度（Subtext Compression）

**原則**：「不說」比「說」更有力

**應用**：
- ❌ 「我很緊張」（言語）
- ✅ 「（手指在腿上敲打不規則節奏，眼睛看向別處）」（身體語言）

**舞蹈應用**：
- 舞蹈動作本身就是角色內心的視覺化
- 台詞應該只提供關鍵信息或矛盾（設置 Setup）

**時間救濟**：將 30% 的台詞轉為「無聲表演」，節省 20-30 秒

### 3.4 策略四：快速開場（Abrupt Opening）

**原則**：故事應該已經進行中

**應用**：
- ❌ Scene 序列「舞台空蕩蕩，演員慢慢走上舞台...」
- ✅ 舞蹈/台詞直接開始，觀眾推斷背景

**時間救濟**：省掉 10-20 秒開場「準備」

### 3.5 策略五：高潮優先（Climax-First Compression）

**原則**：刪減支線，保護高潮

**應用**：
- 原版可能有：序幕（30s）→ 建立（40s）→ 高潮 1（30s）→ 中場（30s）→ 高潮 2（30s）→ 結尾（20s）
- 壓縮版：刪減「中場」，直接高潮連環

**結構**：
1. 識別所有笑點的位置
2. 在笑點間找「冗餘段落」
3. 優先刪減笑點間的過渡段落
4. 確保高潮區的笑點間距不被壓縮（保留停頓）

**時間救濟**：通過刪減非高潮段落，節省 30-60 秒

## 第四層發現：劇本優化的不變原則

### 4.1 必須保留的元素（紅線）

| 元素 | 理由 | 刪除風險 |
|------|------|---------|
| **核心笑點** | 這是故事的「為什麼」 | 削弱喜劇效果 > 30% |
| **角色弧線** | 觀眾需要看到變化 | 角色顯得平面、無聊 |
| **高潮停頓** | Punchline 前的沉默很關鍵 | 笑點失效 |
| **舞蹈的視覺反轉** | 身體喜劇的核心 | 舞蹈變成裝飾 |
| **開場 Hook** | 吸引注意力的前 10 秒 | 觀眾無法進入 |

### 4.2 可以被削減的元素（綠線）

| 元素 | 削減方式 |
|------|---------|
| **背景交代** | 變成視覺或暗示 |
| **過場段落** | 直接剪接或舞蹈轉場 |
| **重複笑點** | 保留最強版本，刪除弱化版 |
| **角色閒話** | 只保留推進情節的台詞 |
| **舞蹈間奏** | 縮短但保留「峰值」 |

### 4.3 劇本品質的四大評估維度

時間壓縮後，用這四個維度檢驗劇本是否「有效」：

| 維度 | 指標 | 評分方式 |
|------|------|---------|
| **喜劇密度** | 笑點 / 時長（笑點每分鐘数） | 原版 4 個笑點 / 5 分 = 0.8 笑/分<br/>目標：3 個笑點 / 3.5 分 = 0.85 笑/分（或更高） |
| **節奏變化** | 場景長度的方差（是否有快慢交替） | 檢查場景長度是否呈現「呼吸曲線」（短-長-短-長） |
| **舞蹈的故事性** | 舞蹈段落是否推進情節或展現角色內心 | 不是「跳舞」，而是「舞蹈中發生了什麼」 |
| **情緒連貫性** | 從開場到結尾的角色轉變是否清晰 | 故事應該回答：「這個人在故事開始和結束時有什麼不同」 |

## 方法論框架：舞蹈劇本優化引擎（Dance Drama Optimization Engine）

### 工作流程（5 個階段）

#### 階段 1：劇本解析
**輸入**：原始劇本（文本 + 舞蹈 timing）
**輸出**：
- 結構圖（Freytag 五階段標記）
- 笑點清單（位置、類型、強度）
- 舞蹈時序表（段落、時長、功能）
- 冗餘度評估

**方法**：
1. 逐句標記「此句是 Setup 還是 Punchline」
2. 標記每個舞蹈段落的「故事功能」（是否推進情節或純粹美學）
3. 計算笑點密度與場景長度

#### 階段 2：約束定義
**輸入**：使用者需求
**輸出**：優化清單
```
- 目標時長：3 分 30 秒 ± 5 秒
- 必保笑點：
  1. 笑點 A（原 2:15）- 核心
  2. 笑點 B（原 3:00）- 高潮
  3. 笑點 C（原 4:30）- 結尾
- 舞蹈比例：≥ 40%
- 角色弧線：內向 → 外向（必須被看到）
```

#### 階段 3：優化方案生成
**方法**：應用五大策略生成 3 個不同的壓縮方案

**方案 A - 激進壓縮**：
- 最大化笑點密度
- 最小化過場
- 舞蹈變得更濃縮但高能
- 風險：可能顯得匆促

**方案 B - 平衡壓縮**：
- 均勻縮減各部分
- 保留部分舞蹈呼吸感
- 最安全的選擇

**方案 C - 舞蹈優先**：
- 保護舞蹈段落
- 大幅刪減戲劇部分
- 適合舞蹈佔比高的作品

#### 階段 4：方案評估
逐個方案檢驗四大維度（喜劇密度、節奏變化、舞蹈故事性、情緒連貫性）

#### 階段 5：最終調整與交付
- 選中方案後，進行微調（停頓 timing、過場方式）
- 輸出：可直接排練的最終版劇本

## 應用案例分析：羅臻劇本（實踐驗證）

### 背景信息（推斷）
- 原始版本：5 分鐘（3 分鐘戲劇 + 2 分鐘舞蹈）
- 目標：3.5 分鐘，保留笑點與節奏感
- 推斷風格：現代喜劇、可能涉及自我嘲諷或日常矛盾

### 優化步驟

#### Step 1: 識別核心笑點（必保）
由於我未看到完整劇本，基於典型舞蹈喜劇推理：
- **笑點 1 - 開場 Setup**：角色展現自信但準備不足（戲劇 0:00-0:30）
- **笑點 2 - 第一次跌跤**：身體語言的反轉（舞蹈/動作 1:30-2:00）
- **笑點 3 - 高潮反轉**：角色突然轉變但仍無法真正達到目標（戲劇 2:45-3:00）
- **笑點 4 - 結尾的詼諧結束**：最後的視覺或台詞 gag（舞蹈/戲劇 3:00-3:30）

#### Step 2: 應用壓縮策略

| 策略 | 應用 | 時間救濟 |
|------|------|---------|
| **Late Entry** | 刪除開場的冗長背景介紹 | -20 秒 |
| **Multi-Purpose** | 讓單個舞蹈動作既搞笑又推進情節 | -15 秒 |
| **Subtext** | 台詞變短，用身體語言傳達 | -15 秒 |
| **Abrupt Opening** | 舞蹈/音樂直接開始 | -10 秒 |
| **Climax-First** | 刪減非高潮的過場 | -10 秒 |
| **總計** | | -70 秒 |

#### Step 3: 結果驗證

**原版結構**：
```
開場介紹（0:30） → 第一段戲（0:40） → 舞蹈 1（0:45）
→ 中場（0:45） → 舞蹈 2（0:50） → 結尾戲（0:30）
= 5 分整
```

**壓縮版結構**（目標 3 分 30 秒）：
```
快速開場 + 舞蹈同步（0:20） → 加密戲劇段（0:45） → 舞蹈 1 精簡版（0:40）
→ 高潮戲劇段（0:35） → 舞蹈 2 精簡版（0:40） → 快速結尾（0:20）
= 3 分 20 秒
```

**驗證四大維度**：
- ✓ 喜劇密度：4 個笑點 / 3.33 分 = 1.2 笑/分（提升 50% ）
- ✓ 節奏變化：短-中-短-中-短（良好呼吸曲線）
- ✓ 舞蹈故事性：舞蹈從「純展示」升級為「笑點載體」
- ✓ 情緒連貫性：清晰的「自信 → 挫折 → 適應 → 新自信」弧線

## 五個核心設計原則（可複用）

### 原則 1：笑點密度優先於時長
- 不是「把 5 分鐘削到 3.5 分鐘」
- 而是「把笑點密度提升，時間反而變短」
- 哲學：質勝於量

### 原則 2：舞蹈是語言，不是裝飾
- 每個舞蹈段落都應該「說」些什麼
- 檢驗標準：去掉舞蹈後，故事還完整嗎？如果是，舞蹈就是冗餘的

### 原則 3：停頓是 Punchline 的一部分
- Punchline 通常包含「台詞」+「停頓」+「觀眾反應」
- 時間壓縮時，刪減過場，但不刪減停頓

### 原則 4：角色轉變是故事的真正高潮
- 不是「發生了什麼」，而是「人物因此改變了什麼」
- 壓縮後，這個轉變應該更清晰、更直接

### 原則 5：舞蹈比例不能低於 35%
- 舞蹈是視覺衝擊力最強的媒介
- 過度刪減舞蹈會削弱整體效果
- 寧可刪減台詞，也要保護舞蹈的視覺張力

## 實務工具：舞蹈劇本優化檢查清單

### 審視清單（優化前必讀）

#### 結構檢查
- [ ] 開場在 30 秒內進入情節？
- [ ] 有清晰的高潮區（通常在整劇 50-70% 處）？
- [ ] 結尾有明確的「新常態」信號？
- [ ] 笑點位置是否合理分佈（避免群聚或孤立）？

#### 台詞檢查
- [ ] 是否有冗長的背景交代？
- [ ] 是否有重複表達同一概念的台詞？
- [ ] 是否有非推進情節的「閒話」？
- [ ] 是否有內心獨白可以變成身體語言？

#### 舞蹈檢查
- [ ] 每段舞蹈都回答「這段舞蹈展現了角色的什麼」？
- [ ] 舞蹈是否推進故事（或至少展現情緒變化）？
- [ ] 舞蹈與音樂的配合是否強化笑點？
- [ ] 舞蹈結尾是否清晰地過渡到下一個場景？

#### 時序檢查
- [ ] 每場景的長度是否形成「呼吸曲線」（快-慢-快）？
- [ ] 高潮前是否有「沉默蓄勢」的時刻？
- [ ] 笑點前的停頓時長是否一致（或有意變化）？
- [ ] 整劇的「節奏溫度」是否在上升（越來越快）？

#### 情感檢查
- [ ] 角色的內心轉變是否可見（通過舞蹈或表情）？
- [ ] 觀眾是否能產生對角色的同理（至少在某個時刻）？
- [ ] 笑點之間是否有「呼吸」（即使短暫的真摯時刻）？
- [ ] 結尾是否留下「餘韻」而非「倉促感」？

### 優化清單（壓縮時的行動清單）

#### 第一輪：刪除顯而易見的冗餘
- [ ] 刪除所有 > 15 秒的背景交代
- [ ] 刪除重複的笑點（保留最強版本）
- [ ] 刪除非推進情節的舞蹈過場（≤ 10 秒）
- [ ] 刪除所有 > 5 秒的停頓（除了 Punchline 前）

**預期節省**：60-90 秒

#### 第二輪：應用五大策略
- [ ] 應用「Late Entry」：標記可以刪除的開場部分
- [ ] 應用「Multi-Purpose」：找 2-3 個元素，賦予多重功能
- [ ] 應用「Subtext」：將 5-10 句台詞轉為身體語言
- [ ] 應用「Abrupt Opening」：直接開始，不要「準備」
- [ ] 應用「Climax-First」：標記可以刪減的非高潮段落

**預期節省**：30-50 秒

#### 第三輪：驗證與微調
- [ ] 運行檢查清單，逐項驗證
- [ ] 檢查四大維度是否達標
- [ ] 計算新的笑點密度（應 ≥ 原版）
- [ ] 計算新的舞蹈比例（應 ≥ 35%）

---

## 結論：不變的設計原則

無論舞蹈劇本是 3 分鐘還是 10 分鐘，以下原則永不改變：

1. **開場必須強勢**（0-10 秒決定生死）
2. **笑點必須被保護**（核心笑點周圍的時間不能被擠壓）
3. **角色必須轉變**（故事的真正高潮是人物改變）
4. **舞蹈必須有「台詞」**（身體語言應該清晰表達意圖）
5. **結尾必須清晰**（觀眾應該知道故事結束了，而不是懸而未決）
