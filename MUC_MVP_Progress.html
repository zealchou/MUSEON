系統範圍（SYSTEM_SCOPE）

MUC MVP 第一階段的系統定義——聚焦「收費閉環 + 設備授權閉環 + 最低防破解基線」

✅ 包含範圍（In Scope）

模組 核心功能 關鍵要素
帳號與身分 註冊、登入、帳號管理 使用者可查看授權與設備
Billing Profile 付款資料維護 支援人工對帳與開通
License 序號 序號生成 / 綁定 / 啟用停用 序號作為計費單位
Plan 方案 訂閱 / 續約分類 / 優惠機制 300 點/月，超額 USD 0.1/點
Subscription 訂閱週期 ACTIVE / GRACE / SUSPENDED 以確認收款日為週期起算
Payment 收款 人工記錄 / 確認 / 作廢 DRAFT / CONFIRMED / VOID 狀態
Device Pairing 配對碼 / 首次綁定 / Token 發行 一序號一 ACTIVE 設備
Device Token 授權輪詢 / Token Revoke 高熵亂數、只存 hash、可驗章
Pre-Job Check 接單前授權驗證 離線超 15 分鐘停止接新任務
Alert 告警 事件通知 / 分級 / ACK 安全事件、到期、停用等
Audit Log 所有狀態變更記錄 Append-only，16 個 action 追蹤

❌ 不包含範圍（Out of Scope）

✗ 自動金流串接 — 第一階段使用人工匯款 / 人工對帳
✗ 自動退款 — 不處理
✗ 推薦分潤 / 介紹人折抵 — Phase 16 規劃
✗ 經銷商 / 代理商 — 暫不支援
✗ 複雜企業組織與多層審批 — 簡化版本
✗ 深度反逆向 / 反 Hook — 最低防破解基線只

🔒 最低安全基線

✓ 序號不能直接當授權憑證
✓ Pair Code 一次性、短時效、只存 hash
✓ Device Token 高熵亂數、只存 hash、可 revoke
✓ Token 綁 device_id + fingerprint 檢查
✓ Entitlement response 必須簽章（Ed25519）
✓ Agent 內建公鑰驗章
✓ 每次接新任務前必查授權
✓ 離線超 15 分鐘停止接新任務
